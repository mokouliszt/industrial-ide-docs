# 付7 他形式プロジェクトからの置換え

他形式プロジェクトをGX Works3形式プロジェクトとして使用する場合、置換えが必要となる命令、デバイス、およびプログラムがあります。本節の内容を確認し、修正してください。

GX Works2形式プロジェクトをGX Works3形式プロジェクトとして使用するために、データを置き換える方法を説明します。

FXCPUのGX Works2形式プロジェクトをGX Works3形式プロジェクトとして使用する場合は、下記を参照してください。

MELSEC FX3G/FX3U/FX3UCシリーズからMELSEC iQ-Fシリーズへの置換えの手引きなお、各データの変換処理の詳細は、下記を参照してください。

### パラメータの修正

GX Works2形式プロジェクトに設定されている自動割付デバイス設定の内容は、GX Works3形式プロジェクトの"CPUパラメータ"の"ラベルエリア容量"、"ラッチラベルエリア容量"、および"ラッチ型ラベルのラッチ種類設定"に引き継がれます。

ただし、GX Works2形式プロジェクトの種類によっては、引き継がれない場合があります。

その場合は、"出力"ウィンドウに表示される変更点を確認し、プロジェクトのラベル容量に応じて"CPUパラメータ"を設定してください。

### 部品の自動的な置換え(GX Works3 Version 1.047Z以降)

GX Works2形式プロジェクトで使用している部品(命令、汎用ファンクション、および汎用ファンクションブロック)の多くは、自動的にGX Works3で使用できる部品に置き換えられます。

それらのうち一部の部品は、GX Works2とGX Works3で引数のデータ型と順序が異なります。その場合、プログラムを変換するとエラーが発生するため、引数のデータ型と順序を修正する必要があります。(→ 命令の置換え(GX

Works3 Version 1.040S以降))

また、MELSEC iQ-RシリーズまたはMELSEC iQ-Lシリーズユニットで対応していない部品は、GX Works3で使用できる部品に置き換えられません。(→ MELSEC iQ-RシリーズまたはMELSEC iQ-Lシリーズのユニットで対応していない部品
