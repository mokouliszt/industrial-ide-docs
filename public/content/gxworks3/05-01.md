# 5.1 ラベルについて

ラベルには、ラベルエディタで登録するラベル(グローバルラベル/ローカルラベル)と、ユニットの専用ラベルとして用意されているユニットラベル(グローバルラベル)、およびMELSOFT Navigatorと連携するシステムラベルがあります。

また、グローバルラベルはシステムラベルとして登録できます。

| 種類 | 内容 | 作成方法 | 作成可能ラベル数 | 参照 |
|---|---|---|---|---|
| ローカルラベル | プログラムごとに使用するラベルです。 | ローカルラベルエディタで作成します。 | / 1プログラムファイル内に最大5120個 | → ラベルの登録 |
| グローバルラベル | プロジェクト内のすべてのプログラムで使用できるラベルです。 一般プログラム、または安全プログラムでのみ使用可能なグローバルラベルと、 どちらでも使用可能な一般/安全共有ラベルがあります。 | グローバルラベルエディタで作成します。 | / 1ファイル内に最大20480個 / 1プロジェクト内に最大16384000個 | → ラベルの登録 |
| システムラベル | iQ Works対応製品との間で共有可能なラベルです。MELSOFT Navigatorにより管理されます。 | グローバルラベルエディタ上で、一般グローバルラベルをシステムラベルとして登録します。 | / 1ファイル内に最大20480個 / 1プロジェクト内に最大16384000個 | → システムラベルの登録 |
| ユニットラベル | 使用するユニットの入出力信号やバッファメモリなどが、あらかじめ定義されているラベルです。 ユニットラベルを使用すると、ユニットの内部アドレスを意識せずに、再利用しやすいプログラムを作成できます。 一般プログラムでのみ使用可能です。 | ユニット情報を追加する際にユニットラベルを追加します。 ユニットラベルは、一般グローバルラベルとして作成されます。 | / 1ファイル内に最大20480個 / 1プロジェクト内に最大16384000個 | → ユニットラベルの登録 |
| ネットワークラベル*1 | ネットワーク上のデバイス局に対して入出力のデータを送受信するラベルです。 ネットワークラベルを使用すると、リンクデバイスの割付けを意識せずにプログラムを作成できます。 ネットワークラベルはグローバルラベルとして登録されます。 | "ネットワークラベル設定"画面で生成します。 | / 1ファイル内に最大20480個 / 1プロジェクト内に最大61440個 | → ネットワークラベル設定 |

*1MXコントローラでのみ対応しています。

### ユニットラベルの自動リフレッシュとダイレクトアクセス

ユニットラベルは自動リフレッシュ用とダイレクトアクセス用があり、ダイレクトアクセス用はラベル名に"_D"が追加されます。

自動リフレッシュとダイレクトアクセスでは、下記のような差異があります。

| 種類 | 内容 | アクセスタイミング |
|---|---|---|
| 自動リフレッシュ | ユニットラベルに書込みおよび読み出した値が自動リフレッシュ時に一括でユニットに反映されます。 プログラムの実行時間を短縮できます。 自動リフレッシュを使用する場合、ユニットパラメータのリフレッシュ設定にて、"リフレッシュ先"に"ユニットラベル"を選択する必要があります。 | 自動リフレッシュ時 |
| ダイレクトアクセス | ユニットラベルに書込みおよび読み出した値が即時ユニットに反映されます。 自動リフレッシュよりもプログラムの実行時間は延びますが、応答性が高くなります。 ダイレクトアクセスを使用する場合、ユニットパラメータのリフレッシュ設定にて、"リフレッシュ先"に"ユニットラベル"以外を選択してください。 "リフレッシュ先"に"ユニットラベル"を選択した場合、END処理実行時またはリフレッシュタイミングを指定したプログラムの実行時に、ユニットラベルの値がリフレッシュにより上書きされます。 | ユニットラベルへの書込み時、 または読出し時 |

### MXコントローラのプロジェクトのグローバルラベル

MXコントローラのプロジェクトには、プロジェクトの新規作成時、モーション設定の設定時、およびアドオンの追加時に下記のグローバルラベルが自動で追加されます。

グローバルラベルにひもづく構造体およびラベルも自動で追加されます。

| グローバルラベル | 構造体/ラベル | 対応する設定 | 参照 |
|---|---|---|---|
| Addon+Global | ADDON SYS _ | アドオンパラメータ | → アドオンパラメータのシステム変数 |
| Adv+Global | / ADV INPUT _ / ADV OUTPUT | アドバンスト同期制御 | → アドバンスト同期制御の変数 |
| Ax+Global | _ / AXIS REAL _ / AXIS ENCODER _ / AXIS VIRTUAL _ / AXIS VIRTUAL ENCODER / AXIS VIRTUAL LINK   | 軸 | → 軸変数 |
| En+Global | 列挙型定数 | ― | ― |
| Gr+Global | AXES GROUP _ | 軸グループ | → 軸グループ変数 |
| IO+Global | / MC INPUT REF / MC TRIGGER REF / MC OUTPUT REF / TARGET REF _ / MC CAMSWITCH SOURCE REF   _ | モーション入出力データ | → モーション入出力データの変数 |
| MotionSys+Global | MT SYSTEM _ | モーションシステム設定 | → モーションシステム設定のシステム変数 |
| / NW+Global1 / NW+Global2 / NW+Global3 | 生成したネットワークラベルおよび構造体 | ネットワークラベル設定 | → ネットワークラベル設定 |
| Prf+Global | / MC CAM REF / MC CAMSWITCH REF / MC POSITIONING DATA REF   _ | 演算プロファイル | → 演算プロファイルの変数 |

- 構造体定義MXコントローラのプロジェクトには、プロジェクトの新規作成時にモーション制御用の構造体定義が追加されます。

**注意事項**

上記のデータはラベルエディタおよび構造体エディタで編集できません。

CSVファイルまたはXMLファイルのデータをラベルエディタにインポートすることもできません。

ただし、一部のデータの"外部機器からのアクセス"のチェック状態は変更できます。(→ MXコントローラのプロジェクトのグローバルラベル)

### ラベルの初期値

GX Works3で設定できるラベルの初期値を示します。

初期値の設定可否は、ラベルのデータ型またはクラスにより異なります。

初期値の表記方法は、ラベルの定数の表記方法と同じです。

**初期値の設定可否および初期値の表記方法(定数の表記方法)の関連項目**

→ MELSEC MXコントローラMX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラMX-Fモデルユーザーズマニュアル

→ MELSEC iQ-RシーケンサCPUユニットユーザーズマニュアル

→ MELSEC iQ-RプロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R SIL2プロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

FX5CPUでは対応していません。

- 基本データ型ラベルの初期値基本データ型のラベルに対して初期値を設定できます。

基本データ型ラベルの初期値は、ラベルエディタで設定します。(→ 初期値)

なお、ポインタ型、タイマ型、積算タイマ型、ロングタイマ型、ロング積算タイマ型、カウンタ型、およびロングカウンタ型のラベルには初期値を設定できません。

配列型ラベルの配列要素にも初期値を設定できます。

**配列要素の初期値の関連項目**

→ 配列要素の初期値
