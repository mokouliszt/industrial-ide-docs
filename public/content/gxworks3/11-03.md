# 11.3 システムシミュレーションの実行

ほかのCPUユニットやシンプルモーションユニットと連携して、シミュレーションを行う場合に使用します。

SDメモリカードを使用する機能をシミュレーションすることもできます。(→ SDメモリカード)

MXコントローラでは対応していません。

**画面の開き方**

[デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [起動]を選択します。

![図(元マニュアル p.645)](images/gxworks3/11-03-01.webp)

(1)パネル：ユニットの動作状態を表示し、ユニットへの操作を行います。

**操作手順**

**システムシミュレーションの操作手順の関連項目**

- マルチCPU: → マルチCPUシステム

- シーケンサCPUとモーションCPU: → シーケンサCPUとモーションCPU

- CPUユニットとシンプルモーションユニット: → シンプルモーションユニット/モーションユニット

- サイクリック伝送: → ネットワークユニット

### シミュレーション接続先設定

"シミュレーション接続先設定"画面では、接続先CPU(GX Simulator3)を設定します。

**画面の開き方**

[デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [接続先設定]を選択します。

![図(元マニュアル p.647)](images/gxworks3/11-03-02.webp)

**画面項目**

| 項目 | 内容 |
|---|---|
| (1)システム構成情報 | GX Simulator3画面に追加されているシステムが一覧で表示されます。 |
| (2)接続対象のシステムを表示する | チェックを入れると、GX Works3で開いているプロジェクトのシステム構成と一致するシステムのみがシステム構成情報に表示されます。 チェックをはずすと、プロジェクトのシステム構成と異なるシステムも表示されます。 |
| (3)接続対象のユニットを表示する | チェックを入れると、接続可能なCPUユニット、そのCPUユニットが設置されているベースユニット、およびシステムのみがシステム構成情報に表示されます。 |

**操作手順**

**"シミュレーション接続先設定"画面の操作方法の関連項目**

- マルチCPU: → マルチCPUシステム

- シーケンサCPUとモーションCPU: → シーケンサCPUとモーションCPU

- CPUユニットとシンプルモーションユニット: → シンプルモーションユニット/モーションユニット

- サイクリック伝送: → ネットワークユニット

- システム構成情報の表示内容"接続対象のシステムを表示する"および"接続対象のユニットを表示する"のチェック状態によって、システム構成情報に表示される内容が切り替わります。

## マルチCPUシステム

マルチCPUシステムをシミュレーションする操作について、下記のシステムを例に説明します。

![図(元マニュアル p.649)](images/gxworks3/11-03-04.webp)

**操作手順**

#### GX Simulator3の起動

1. GX Simulator3を起動します。(→ システムシミュレーションの実行)

![図(元マニュアル p.649)](images/gxworks3/11-03-05.webp)

2. GX Simulator3画面の[システム追加]ボタンをクリックします。

3. "GX Simulator3システム追加"画面で、プロジェクト名とシステム番号を設定します。

![図(元マニュアル p.649)](images/gxworks3/11-03-06.webp)

4. GX Simulator3画面で、シミュレーションを実行するCPUユニットにチェックを入れ、[開始]ボタンをクリックします。

![図(元マニュアル p.649)](images/gxworks3/11-03-07.webp)

**シミュレーションできるCPUユニットの関連項目**

→ GX Simulator3(システムシミュレーション)がサポートするユニット

#### 1号機への接続とデータの書込み

1. [デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [接続先設定]を選択します。

![図(元マニュアル p.650)](images/gxworks3/11-03-08.webp)

2. "シミュレーション接続先設定"画面で、1号機のCPUユニットを選択し、[接続]ボタンをクリックします。

ステータスバーに"シミュレーション((システム番号).(号機番号))"が表示されます。

3. [オンライン] ⇒ [シーケンサへの書込み]を選択します。

![図(元マニュアル p.650)](images/gxworks3/11-03-09.webp)

パラメータおよびプログラムをGX Simulator3に書き込みます。

#### 2号機への接続とデータの書込み

1. 別のGX Works3を起動し、2号機のプロジェクトを開きます。

![図(元マニュアル p.650)](images/gxworks3/11-03-10.webp)

2. [デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [接続先設定]を選択します。

3. "シミュレーション接続先設定"画面で、2号機のCPUユニットを選択し、[接続]ボタンをクリックします。

GX Works3のステータスバーに"シミュレーション((システム番号).(号機番号))"が表示されます。

4. [オンライン] ⇒ [シーケンサへの書込み]を選択します。

![図(元マニュアル p.651)](images/gxworks3/11-03-11.webp)

パラメータおよびプログラムをGX Simulator3に書き込みます。

#### シミュレーションの実行

1. GX Simulator3画面で1号機の[RESET]ボタンをクリックします。

![図(元マニュアル p.651)](images/gxworks3/11-03-12.webp)

2. 各号機の"RUN"を選択します。

3. [オンライン] ⇒ [モニタ] ⇒ [モニタ開始]を選択します。

システムシミュレーションが開始されます。

2. GX Simulator3画面の[システム追加]ボタンをクリックします。

![図(元マニュアル p.652)](images/gxworks3/11-03-15.webp)

4. GX Simulator3画面で、シミュレーションを実行するシーケンサCPUとモーションCPUにチェックを入れます。

![図(元マニュアル p.652)](images/gxworks3/11-03-16.webp)

![図(元マニュアル p.653)](images/gxworks3/11-03-17.webp)

#### シーケンサCPUへの接続とデータの書込み

1. [デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [接続先設定]を選択します。

![図(元マニュアル p.653)](images/gxworks3/11-03-18.webp)

2. "シミュレーション接続先設定"画面で、シーケンサCPUを選択し、[接続]ボタンをクリックします。

GX Works3のステータスバーに"シミュレーション((システム番号).(号機番号))"が表示されます。

3. [オンライン] ⇒ [シーケンサへの書込み]を選択します。

![図(元マニュアル p.653)](images/gxworks3/11-03-19.webp)

パラメータおよびプログラムをGX Simulator3に書き込みます。

#### モーションCPUへの接続とデータの書込み

1. MT Developer2を起動し、シミュレーションを実行するモーションCPUのプロジェクトを開きます。

![図(元マニュアル p.654)](images/gxworks3/11-03-20.webp)

2. [オンライン] ⇒ [接続先設定]を選択します。

3. "接続先設定"画面の"パソコン側I/F"で"シミュレータ"をダブルクリックします。

4. "システムNo.設定"画面の"システムNo."に、手順3で設定したシステム番号を指定します。

5. "接続先設定"画面の"マルチCPU指定"に、モーション

CPUの号機を指定します。

6. "接続先設定"画面の[OK]ボタンをクリックします。

すでに起動しているMT Simulator2に接続されます。

7. [オンライン] ⇒ [モーションへの書込み]を選択します。

![図(元マニュアル p.654)](images/gxworks3/11-03-21.webp)

パラメータおよびプログラムをMT Simulator2に書き込みます。

#### シミュレーションの実行

1. GX Simulator3画面でシーケンサCPUの[RESET]ボタンをクリックします。

![図(元マニュアル p.654)](images/gxworks3/11-03-22.webp)

2. GX Simulator3画面でシーケンサCPUの"RUN"を選択します。

3. "MT Simulator2"画面で"RUN"を選択します。

![図(元マニュアル p.654)](images/gxworks3/11-03-23.webp)

4. GX Works3で[オンライン] ⇒ [モニタ] ⇒ [モニタ開始]を選択します。

![図(元マニュアル p.655)](images/gxworks3/11-03-24.webp)

システムシミュレーションが開始されます。

![図(元マニュアル p.660)](images/gxworks3/11-03-41.webp)

4. 手順5のGX Works3で[デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [接続先設定]を選択します。

![図(元マニュアル p.660)](images/gxworks3/11-03-42.webp)

GX Works3のステータスバーに"シミュレーション((システム番号).(号機番号))"が表示されます。

![図(元マニュアル p.661)](images/gxworks3/11-03-43.webp)

#### シミュレーションの実行

1. GX Simulator3画面でローカル局のCPUユニットの[RESET]ボタンをクリックします。

![図(元マニュアル p.661)](images/gxworks3/11-03-44.webp)

2. GX Simulator3画面でローカル局のCPUユニットの"RUN"を選択します。

3. GX Simulator3画面でマスタ局のCPUユニットの[RESET]ボタンをクリックします。

4. GX Simulator3画面でマスタ局のCPUユニットの"RUN"を選択します。

5. マスタ局のプロジェクトを開いているGX Works3と、ローカル局のプロジェクトを開いているGX Works3で、[オンライン] ⇒ [モニタ] ⇒ [モニタ開始]を選択します。

システムシミュレーションが開始されます。

## SDメモリカード

「仮想SDメモリカードファイル(*.gxsc)」を作成することで、SDメモリカードを使用する機能をシミュレーションできます。

仮想SDメモリカードファイルとは、シミュレーション中にSDメモリカードとして使用するファイル(フォルダー)です。

SDメモリカードおよび任意のフォルダーに保存されているデータを仮想SDメモリカードファイルにコピーできます。シミュレーションする機能に合わせて、必要なファイルをコピーしてください。

**操作手順**

#### 仮想SDメモリカードファイルを新規作成する

1. GX Simulator3を起動します。(→ システムシミュレーションの実行)

![図(元マニュアル p.662)](images/gxworks3/11-03-45.webp)

2. 仮想SDメモリカードファイルを使用するCPUユニットの[SD]ボタンをクリックします。

3. "仮想SDメモリカードファイル設定"画面の"仮想SDメモリカードファイルを使用する"を選択します。

![図(元マニュアル p.662)](images/gxworks3/11-03-46.webp)

4. [新規]ボタンをクリックします。

5. "仮想SDメモリカードファイル新規作成"画面の各項目を設定します。

![図(元マニュアル p.662)](images/gxworks3/11-03-47.webp)

6. [作成]ボタンをクリックします。

7. "仮想SDメモリカードファイル設定"画面に、仮想SDメモリカードファイルのファイルパスが表示されていることを確認します。

![図(元マニュアル p.662)](images/gxworks3/11-03-48.webp)

8. [OK]ボタンをクリックします。

9. GX Simulator3画面で、シミュレーションを実行するCPUユニットにチェックを入れ、[開始]ボタンをクリックします。

![図(元マニュアル p.663)](images/gxworks3/11-03-49.webp)

#### 作成済みの仮想SDメモリカードファイルを使用する

1. GX Simulator3を起動します。(→ システムシミュレーションの実行)

![図(元マニュアル p.663)](images/gxworks3/11-03-50.webp)

2. 仮想SDメモリカードファイルを使用するCPUユニットの[SD]ボタンをクリックします。

3. "仮想SDメモリカードファイル設定"画面の"仮想SDメモリカードファイルを使用する"を選択します。

![図(元マニュアル p.663)](images/gxworks3/11-03-51.webp)

4. [参照]ボタンをクリックします。

5. 表示された画面でファイルを選択し、[開く]ボタンをクリックします。

![図(元マニュアル p.663)](images/gxworks3/11-03-52.webp)

6. "仮想SDメモリカードファイル設定"画面に、仮想SDメモリカードファイルのファイルパスが表示されていることを確認します。

![図(元マニュアル p.663)](images/gxworks3/11-03-53.webp)

7. [OK]ボタンをクリックします。

8. GX Simulator3画面で、シミュレーションを実行するCPUユニットにチェックを入れ、[開始]ボタンをクリックします。

![図(元マニュアル p.664)](images/gxworks3/11-03-54.webp)

Windowsのバージョンとディスクのファイルシステムの組み合わせによって、作成できる仮想SDメモリカードファイルの最大のサイズが決まっています。

そのため、ディスクの空き容量にかかわらず、仮想SDメモリカードファイルの生成に失敗する場合があります。

"仮想SDメモリカードファイル設定"画面の[エクスプローラで仮想SDメモリカードファイルを開く]ボタンをクリックすると、仮想SDメモリカードファイルをエクスプローラー上で開くことができます。

開いたファイルには、"オンラインデータ操作"画面で書き込む、または読み出すことができないデータ(例:

Webサーバ機能を使用して作成したユーザWebページ)を保存できます。

## シミュレーション環境ファイルを開く

保存したシミュレーション環境ファイルを開くことで、シミュレーション環境を復元できます。

**画面の開き方**

GX Simulator3画面で[ファイル] ⇒ [シミュレーション環境を開く]を選択します。

![図(元マニュアル p.666)](images/gxworks3/11-03-57.webp)

**操作手順**

シミュレーション環境ファイルを選択し、[開く]ボタンをクリックします。

**注意事項**

復元したシミュレーション環境と接続するプロジェクトの内容が一致しない場合、GX Simulator3でエラーが発生する可能性があります。

シミュレーション環境ファイルを開いたあと、「シーケンサとの照合」でGX Simulator3とプロジェクトのデータが一致していることを確認してください。

照合結果が不一致の場合は、「シーケンサからの読出し」により、GX Simulator3とプロジェクトのデータを一致させることができます。

## システムシミュレーションの終了

**操作手順**

GX Simulator3画面の[閉じる]ボタンをクリックします。

GX Simulator3画面は閉じずに、仮想システムとの接続を切る場合は、下記の操作を行ってください。

- [デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [接続先解除]

システムシミュレーションを再開するときは、下記の操作を行ってください。

- [デバッグ] ⇒ [シミュレーション] ⇒ [システムシミュレーション] ⇒ [接続先設定]
