# 9.8 デバイス/ラベルの影響範囲の表示

データフロー解析とは、指定したデバイス/ラベルを解析対象(基点)として、その影響範囲をデータフロー図で表示する機能です。

指定したデバイス/ラベルの影響範囲を視覚的に確認でき、異常が発生した場合にエラーの原因を特定するのに便利です。

データフロー解析の機能を示します。

| 機能 | 内容 | 参照 |
|---|---|---|
| デバイス/ラベルの解析 | プログラムで使用しているデバイス/ラベルの影響範囲をデータフロー図に表示します。 | → デバイス/ラベルの解析 |
| 装置動作のAI解析 | 装置動作によるデバイス間の物理的な関係性を推測し、データフロー図に表示します。 | → 装置動作のAI解析 |
| デバイス/ラベルのモニタ | データフロー図上でデバイス/ラベルをモニタし、現在値の変化を確認します。 | → デバイス/ラベルのモニタ |

## デバイス/ラベルの解析

デバイス/ラベルを解析し、データフロー図を表示する手順を示します。

**操作手順**

1.プログラムエディタなどの画面上で、解析するデバイス/ラベルを選択します。

2.右クリック ⇒ ショートカットメニュー[データフロー解析]を選択します。

解析結果がデータフロー図として、"データフロー解析"ウィンドウに表示されます。

- デバイス/ラベルを選択後、[Ctrl]+[Shift]+を押すことでもデータフロー図を表示できます。

- データフロー図に表示されるブロックからデバイス/ラベルを解析することもできます。

下記のブロックを選択後、右クリック ⇒ ショートカットメニュー[データフロー解析]を選択してください。

- デバイス/ラベルブロック

- プロセスブロック(ファンクションブロックのみ対応)

注意事項

解析対象に指定したデバイス/ラベルに関連する項目がない場合、データフロー図は表示されません。

## 装置動作のAI解析

AI解析とは、収集したシーケンサのデータを解析して装置動作によるデバイス間の物理的な関係性を推測し、データフロー図に表示する機能です。推測されたデバイス間の物理的な関係性を確認することで、異常が発生した場合のエラー原因の特定を容易にします。

R00CPU、RnPSFCPU、およびリモートヘッドユニットでは対応していません。

この場合、Y1とX2に物理的な関係性があります。

れません。

**操作手順**

データフロー図上で右クリック ⇒ ショートカットメニュー[モニタ] ⇒ [デバイスのデータ型設定] ⇒ [ファイルのインポート]を選択します。

インポート時のCSVファイルは、下記のフォーマットに従ってください。

- ヘッダー部(1)はなくてもインポートできます。

- 項目間の区切りは「¥t」(タブ)を使用してください。

- インポートしない行には、1列目の先頭に「//」(スラッシュ)を追加してください。

- "データ型"列の表記方法は下記のとおりです。

| 設定するデータ表現形式 | 表記方法 |
|---|---|
| 16ビット整数[符号付き] | INT |
| 16ビット整数[符号なし] | WORD |
| 32ビット整数[符号付き] | DINT |
| 32ビット整数[符号なし] | DWORD |
| 64ビット整数[符号付き]*1 | QINT |
| 64ビット整数[符号なし]*1 | QWORD |
| 単精度実数 | REAL |
| 倍精度実数*1 | LREAL |
| 文字列 | STRING |
| 文字列[Unicode] | WSTRING |

*1：FX5CPUではインポートできません。なお、インポートできるデバイスは下記のとおりです。

デバイス

T*1、ST*1、LT*1,*2、LST*1,*2、C*1、LC*1、D、W、SW、SD、J□¥W*2、J□¥SW*2、U□¥G、U3E□¥G*2、U3E□¥HG*2、Z、LZ、R、ZR*2、RD*2

*1：現在値の情報としてインポートされます。*2：FX5CPUでは対応していません。

注意事項

- データが10,000行を超えたCSVファイルはインポートできません。

- 1つのデバイスに対して異なるデータ型を設定した行が複数存在する場合、CSVファイルの最も上にある行の設定がインポートされます。
