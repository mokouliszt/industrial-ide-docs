# 15.5 CPUユニットのプロジェクトへの不正なアクセスの防止(ユーザ認証)

権限がないユーザによる安全プロジェクトへの不正なアクセスを防止するには、プロジェクトのユーザ認証機能を使用します。

RnPSFCPUおよびRnSFCPUで対応しています。

## プロジェクトのユーザ認証機能について

プロジェクトに対して、ユーザ名/パスワードを登録することで操作を制限できます。

制限できる操作: 下記の操作を制限できます。

プロジェクトのユーザ認証プロジェクトへアクセスするためには、登録されているユーザ情報(ユーザ名/パスワード)でのログオンが必要になります。

![図(元マニュアル p.843)](images/gxworks3/15-05-01.webp)

### アクセスレベル

アクセスレベルとは、プロジェクトおよびCPUユニットにログオンするユーザに与えられる操作権限です。

アクセスレベルに応じて操作できる内容を示します。

| アクセスレベル | | 操作権限 |
|---|---|---|
| 高低 | Administrators (管理者レベル) | プロジェクトのユーザ管理、CPUユニットのユーザ管理などを含むすべての操作 |
| 高低 | Developers (開発者レベル) | ユーザ管理、セキュリティの設定以外の操作一般プログラムおよび安全プログラムに関連するデータのCPUユニットへの書込み |
| 高低 | Assistant Developers (開発者レベル) | ユーザ管理、セキュリティの設定、安全プログラムの書込み、およびパラメータの書込み以外の操作一般プログラムに関連するデータのみのCPUユニットへの書込み |
| 高低 | Users (オペレータレベル) | プロジェクトの参照、CPUユニットのモニタ |

### ユーザ認証が必要な機能

アクセスレベルに応じてプロジェクト内で使用できる機能が制限されます。

制限される機能とそのアクセスレベルを示します。

○: 使用可、×: 使用不可

| 機能 | アクセスレベル | | | |
|---|---|---|---|---|
| 機能 | Administrators | Developers | Assistant Developers | Users |
| 上書き保存 | ○ | ○ | ○ | × |
| ユーザの管理 | ○ | × | × | × |
| プロジェクトのユーザ情報削除 | ○ | × | × | × |
| プロジェクト変更履歴 | ○ | ○ | ○ | × |
| 機種/動作モード変更 | ○ | × | × | × |
| 変換+RUN中書込み | ○ | ○ | ○*1 | × |
| シーケンサへの書込み | ○ | ○ | ○*1 | × |

*1一般プログラムに関連するデータのみCPUユニットに書き込めます。(→ 操作)

### GUESTユーザ

GUESTユーザとは"ユーザ認証"画面で、ユーザ名およびパスワードの入力を省略してプロジェクトおよびCPUユニットへログオンできる権限を持つユーザです。(アクセスレベルはAssistant DevelopersまたはUsersです。)

GUESTユーザとしてログオンするには、"新しいユーザの追加"画面でGUESTユーザを追加する、または"ユーザ管理"画面で

GUESTユーザを有効にする必要があります。

**関連項目**

→ 新しいユーザの追加

→ ユーザの管理ログオン後、このユーザは一般プログラムのみを読み出したり編集したりできます。

## プロジェクトのユーザ認証機能の使用手順

1. 安全プロジェクトを作成し、プロジェクトにAdministratorsのユーザを追加します。(→ 新規作成する)

2. プロジェクトの操作を許可するユーザを追加します。(→ ユーザの管理)

3. プロジェクトを保存します。(→ 保存)

以降、プロジェクトへアクセスする際には、ログオンが必要になります。

## プロジェクトへのログオン

ユーザ情報が登録されているプロジェクトは、プロジェクトを開くなどの操作をする際にログオンする必要があります。

プロジェクトに登録されているユーザ名、パスワードを入力しログオンしてください。

GUESTユーザとしてログオンする場合は、"ユーザ認証"画面の[GUESTでログオン]ボタンをクリックしてください。

## ログオンユーザのパスワードの変更

プロジェクトにログオンしているユーザのパスワードを変更します。

**操作手順**

[プロジェクト] ⇒ [セキュリティ] ⇒ [ユーザパスワード変更]を選択します。

## ユーザの管理

ユーザの登録状態を管理します。

Administratorsのユーザは、すべてのユーザのパスワードを変更できます。

**画面の開き方**

[プロジェクト] ⇒ [セキュリティ] ⇒ [ユーザ管理]を選択します。

![図(元マニュアル p.845)](images/gxworks3/15-05-02.webp)

### 追加/変更

プロジェクトにユーザ情報(ユーザ名/パスワード)を追加/変更します。

**操作手順**

1. [追加]/[変更]ボタンをクリックします。

2. "新しいユーザの追加"画面/"ユーザ変更"画面で各項目を入力し、[OK]ボタンをクリックします。

### 削除

プロジェクトに登録されているユーザ情報を削除します。

**操作手順**

削除するユーザ名を選択し、[削除]ボタンをクリックします。

### GUESTユーザを有効にする

"GUESTユーザを有効にする"にチェックを入れると、GUESTユーザのユーザ情報が一覧に表示されます。

GUESTユーザのユーザ情報を変更できるかどうかを示します。

○: 変更可、×: 変更不可

| ユーザ情報 | | 変更 |
|---|---|---|
| 項目 | 内容 | |
| ユーザ名 | GUEST | × |
| アクセスレベル | Assistant Developers | ○*1 |
| パスワード | GUEST1234 | × |

*1Assistant DevelopersとUsers以外のアクセスレベルには変更できません。
