# 17.2 データロギング機能

MELSEC MXコントローラ、MELSEC iQ-Rシリーズ、MELSEC iQ-Lシリーズ、およびMELSEC iQ-FシリーズのCPUユニットのデータロギング機能を使用して、指定したタイミングでデータを収集します。

データロギング機能の詳細は、下記を参照してください。

→ MELSEC MXコントローラ MX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラ MX-Fモデルユーザーズマニュアル

→ MELSEC iQ-R シーケンサCPUユニットユーザーズマニュアル

→ MELSEC iQ-R プロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

→ MELSEC iQ-F FX5ユーザーズマニュアル(応用編)

R00CPUおよびRnPSFCPUでは対応していません。

## データロギング機能の使用手順

1. データロギング機能で使用するCPUユニットの内部バッファ容量を設定します。(→ データロギング用内部バッファ容量の設定)

2. CPUユニットロギング設定ツールでデータロギングの設定を行い、CPUユニットに書き込みます。

3. CPUユニットロギング設定ツールからデータロギングを開始します。

4. データロギング結果(ロギングデータ)をGX LogViewerで確認します。

CPUユニットロギング設定ツールの使用方法は、下記を参照してください。

→ CPUユニットロギング設定ツール Version 1 オペレーティングマニュアル

→ MELSEC MXコントローラ MX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラ MX-Fモデルユーザーズマニュアル

→ MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

→ MELSEC iQ-F FX5ユーザーズマニュアル(応用編)

GX LogViewerの使用方法は、下記を参照してください。

→ GX LogViewer Version 1 オペレーティングマニュアル

### CPUユニットロギング設定ツールの起動方法

GX Works3からCPUユニットロギング設定ツールを起動する方法を示します。

選択したデバイス/ラベルのデータ名およびデータ型をCPUユニットロギング設定ツールの"データ"設定画面に自動で設定する場合は手順1から、自動で設定しない場合は手順2から操作してください。

**操作手順**

1. GX Works3の下記いずれかの画面でデバイス/ラベルを選択します。

- ラダーエディタ

- STエディタ

- FBD/LDエディタ

- ウォッチウィンドウ2. GX Works3の下記いずれかのメニューを選択します。

- [ツール] ⇒ [ロギング設定ツール]

- 右クリック ⇒ ショートカットメニュー[ロギング対象に設定(ロギング設定ツール)]

CPUユニットロギング設定ツールの起動時、下記の情報がCPUユニットロギング設定ツールに反映されます。

- プロジェクトの機種情報

- プロジェクトの現在の接続先

- 表示言語

- デバイス/ラベルのデータ名とデータ型("データ"設定画面に自動で設定されます。)

注意事項

- R00CPU、RnPCPU、RnPSFCPU、LHCPU、およびFX5CPUのプロジェクトで使用しているラベルのデータ名とデータ型は自動で設定されません。

- 設定可能な点数を超えるデバイス/ラベルは設定されません。

### GX LogViewerの起動方法

GX Works3からGX LogViewerを起動する方法は、下記を参照してください。

→ リアルタイムモニタの使用手順

## データロギング用内部バッファ容量の設定

### MXコントローラ

データロギング機能用の内部バッファ容量は、"CPUパラメータ"の"メモリ/デバイス設定"の"内部バッファ容量設定"で設定します。設定後、CPUユニットにCPUパラメータを書き込んでください。

ただし、CPUユニットのデータロギング機能とリアルタイムモニタ機能を併用する場合は、下記の条件を満たす範囲内で容量を設定してください。

データロギングバッファ容量の合計+リアルタイムモニタバッファ容量≤24576KB

各バッファ容量は、CPUパラメータで確認できます。

### RCPUおよびLHCPU

データロギング機能用の内部バッファ容量は、"CPUパラメータ"の"メモリ/デバイス設定"の"内部バッファ容量設定"で設定します。設定後、CPUユニットにCPUパラメータを書き込んでください。

ただし、CPUユニットのデータロギング機能、メモリダンプ機能、リアルタイムモニタ機能を併用する場合は、下記の条件を満たす範囲内で容量を設定してください。

データロギングバッファ容量の合計+メモリダンプバッファ容量+リアルタイムモニタバッファ容量≤3072KB

各バッファ容量は、下記で確認できます。

- データロギングバッファ容量、メモリダンプバッファ容量: CPUパラメータ

- リアルタイムモニタバッファ容量: GX LogViewerのリアルタイムモニタ設定

### FX5CPU

データロギング機能を使用するかどうかを、"CPUパラメータ"の"メモリ/デバイス設定"の"内部バッファ使用機能"で設定します。

データロギング機能用の内部バッファ容量は、"CPUパラメータ"の"メモリ/デバイス設定"の"内部バッファ容量設定"で設定します。

設定後、CPUユニットにCPUパラメータを書き込んでください。

ただし、下記の条件を満たす範囲内で容量を設定してください。

データロギングバッファ容量の合計≤320KB

FX5SCPUおよびFX5UJCPUの場合、内部バッファ容量はデフォルトから変更できません。

注意事項

データロギング機能とメモリダンプ機能の併用はできません。
