# 6.1 プログラミング機能について

### プログラム言語の種類と特長

GX Works3でサポートするプログラム言語を示します。

| プログラム言語 | 名称 | 内容 | プログラムの作成方法 | 言語仕様 |
|---|---|---|---|---|
| ラダー*1 | 回路図 | 接点やコイルからなる回路で表すグラフィック言語です。 ラダーエディタ内でSTプログラムを編集するインラインST機能が使用できます。 | → ラダープログラムの作成 | → MELSEC MXコントローラMX-Rモデルプログラミングマニュアル → MELSEC MXコントローラMX-Fモデルプログラミングマニュアル → MELSEC iQ-Rプログラミングマニュアル (プログラム設計編) → MELSEC iQ-F FX5プログラミングマニュアル(プログラム設計編) |
| ST | ストラクチャードテキスト | C言語などの高級言語と同様に条件文による選択分岐や、 反復文による繰り返しなどの構文による制御が可能です。 これにより、プログラムが簡潔に記述できます。 | → STプログラムの作成 | → MELSEC MXコントローラMX-Rモデルプログラミングマニュアル → MELSEC MXコントローラMX-Fモデルプログラミングマニュアル → MELSEC iQ-Rプログラミングマニュアル (プログラム設計編) → MELSEC iQ-F FX5プログラミングマニュアル(プログラム設計編) |
| FBD/LD | ファンクションブロックダイアグラム/ラダー | 部品を配置して結線する簡単な操作で、制御プログラムを作成するグラフィック言語です。 FBD/LDエディタ内でSTプログラムを編集するインラインST機能が使用できます。 | → FBD/LDプログラムの作成 | → MELSEC MXコントローラMX-Rモデルプログラミングマニュアル → MELSEC MXコントローラMX-Fモデルプログラミングマニュアル → MELSEC iQ-Rプログラミングマニュアル (プログラム設計編) → MELSEC iQ-F FX5プログラミングマニュアル(プログラム設計編) |

### グローバルデバイス/ローカルデバイスの入力

グローバルデバイスとローカルデバイスを使い分ける場合は、下記のように入力します。

- 一般グローバルデバイス: デバイス名をそのまま入力(例: D10)

安全グローバルデバイス: デバイス名の前に"SA¥"を付加して入力(例: SA¥D10)

- 一般ローカルデバイス: デバイス名の前に"#"を付加して入力(例: #D10)

安全ローカルデバイス: デバイス名の前に"SA¥#"を付加して入力(例: SA¥#D10)

**グローバルデバイス/ローカルデバイスの関連項目**

→ MELSEC MXコントローラMX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラMX-Fモデルユーザーズマニュアル

→ MELSEC iQ-RシーケンサCPUユニットユーザーズマニュアル

→ MELSEC iQ-RプロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R SIL2プロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

FX5CPUは、ローカルデバイスには対応していません。

### グローバルラベル/ローカルラベルの入力

グローバルラベルとローカルラベルに同名のラベルがある場合、プログラム内のラベルは下記のように扱います。

- グローバルラベルとローカルラベルを登録後、プログラム内にラベル名を入力した場合: ローカルラベルとして扱います。

- グローバルラベルを登録後、プログラム内にラベル名を入力し、その後同名のローカルラベルを登録した場合: ローカルラベルとして扱います。

- ローカルラベルを登録後、プログラム内にラベル名を入力し、その後同名のグローバルラベルを登録した場合: ローカルラベルとして扱います。

グローバルラベルとローカルラベルは、デフォルトで異なる色が設定されています。

色を変更したい場合は、"色およびフォント"画面から変更できます。

1. プログラムブロックのデータを作成します。(→ 新規作成)

3. プログラムを作成します。（→ ラダープログラムの作成／→ STプログラムの作成／→ FBD/LDプログラムの作成／→ SFCプログラムの作成）

4. 作成したプログラムをチェックします。(→ プログラムのチェック)

5. 変換します。(→ プログラムの変換)
