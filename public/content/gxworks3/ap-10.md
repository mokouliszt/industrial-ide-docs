# 付10 FX5CPUから機種変更する場合の注意事項

FX5CPUからMX-Fモデル/RCPUへ機種変更する場合の注意事項を説明します。

## MXコントローラへ機種変更する場合

### 機種変更前の注意事項

- プロジェクトに登録されたセキュリティの確認 • プロジェクトにセキュリティが設定されている場合、MX-Fモデルへの機種変更はできません。機種変更できないセキュリティの条件および処置方法を説明します。

| 条件 | 処置方法 | 参照 |
|---|---|---|
| | 設定を削除してから機種変更を行ってください。 | |
| | セキュリティバージョンを"1"に変更してから機種変更 を行ってください。 | |

- プログラムの確認 • SFCプログラムが存在する場合、MX-Fモデルへの機種変更はできません。SFCプログラムを削除してから機種変更を行ってください。

- マクロ型ファンクションブロックが存在する場合、MX-Fモデルへの機種変更はできません。マクロ型ファンクションブロックを削除、またはサブルーチン型ファンクションブロックに変更してから、機種変更を行ってください。

- 機種変更後に入力(X)/出力(Y)として認識される表記の確認"X08"や"Y0F"など、FX5CPUではデバイスとして認識されない要素は、機種変更時のデバイス変換の対象外です。

変更後のMX-Fモデルではデバイスとして認識されるため、機種変更前に修正してください。

機種変更手順1.機種変更前にプロジェクトデータを確認します。(→ 機種変更前の注意事項) 2. FX5CPUのプロジェクトをMX-Fモデルに機種/動作モード変更を実行します。3.機種/動作モード変更の実行後のプロジェクトで、ユニット構成を設定します。4.パラメータを設定します。5.プログラムの修正をします。(→ 機種変更後の修正)

機種/動作モード変更の実行後のシステム構成に合わせて、プログラムを見直します。6.シーケンサ上で、動作を確認します。

### 機種変更後の修正

- ユニット指定番号の置換ユニット指定番号は、デバイス/ラベル置換を使用して置き換えます。

(例: "U1"→"U4")

#### ・命令の置換え • 先頭デバイスと点数を指定する命令は、先頭デバイスが置き換えられます。デバイスの連続性が変わるため、点数もあわ

せて変更が必要な場合があります。

- ラダープログラムまたはSTプログラムの場合、MX-Fモデルで使用できない命令を使用可能な命令に置換します。使用できない命令は、変換時にエラーが発生することにより、確認できます。

#### ・NOP MX-FモデルではNOP命令を使用できません。FX5CPUのプロジェクトでNOP命令を使用している場合は、MX-Fモデルに機

種変更後、引き継がれるNOP命令を削除してください。(→ NOPの挿入/削除)

- FBD/LDプログラムの修正FX5CPUのFBD/LD言語とMX-FモデルのFBD/LD言語では、入出力引数の個数または順番が異なるファンクション部品があります。

それらのファンクション部品は、未定義のFUNとして表示されます。[編集] ⇒ [FB/FUNの更新]を選択して定義情報を更新し、プログラムを修正してください。

## RCPUへ機種変更する場合

### 機種変更前の注意事項

- プロジェクトに設定されたセキュリティの確認プロジェクトにセキュリティが設定されている場合、RCPUへの機種変更はできません。

機種変更できないセキュリティの条件および処置方法を説明します。

| 条件 | 処置方法 | 参照 |
|---|---|---|
| ブロックパスワードが設定されている部品が存在する | 設定を削除してから機種変更を行ってください。 | → 削除 |
| セキュリティキーが設定されている部品が存在する | | → 削除 |
| プロジェクトのセキュリティバージョンを"2"に設定している | セキュリティバージョンを"1"に変更してから機種変更 を行ってください。 | → セキュリティバー ジョンを"2"から"1"に変更す る場合 |

変更後のRCPUではデバイスとして認識されるため、機種変更前に修正してください。

### 機種変更後の修正

- デバイスの置換FX5CPUでは入力(X)/出力(Y)は先頭から連続して使用します。

RCPUでは入力(X)/出力(Y)は先頭XYに設定した番号から順番に使用され、入力(X)と出力(Y)で番号が重複することはありません。

そのため、入力(X)/出力(Y)のデバイス番号が重複しないように、デバイス置換を行う必要があります。
