下記の実行プログラムをプログラムエディタ上で確認するには、モニタ機能を使用します。

- ラダー

- ST

- FBD/LD

- SFC

あらかじめ、モニタするプログラムエディタを開いておいてください。

下記のオプション設定により、表示形式や各機能の詳細な動作設定が行えます。

[ツール][オプション]"モニタ"

### ファイルレジスタをモニタする場合の注意事項

"CPUパラメータ""ファイル設定""ファイルレジスタ設定""使用有無設定"で"プログラム別のファイルレジスタを使用する"設定時にファイルレジスタをモニタする場合、指定プログラムモニタ機能でモニタしたいプログラムを選択してからモニタしてください。

指定プログラムモニタ機能の操作方法は下記を参照してください。

→ 指定したプログラムのデバイス/ラベルをモニタする場合指定プログラムモニタ機能を使用しない場合指定プログラムモニタ機能を使用せずに"プログラム別のファイルレジスタを使用する"を設定してファイルレジスタをモニタする場合、"デバイス・ラベルアクセスサービス処理設定"の"指定方法"により動作が異なります。

| 指定方法 | 動作 |
|---|---|
| プログラム間とEND処理で行う | プログラム間でのモニタおよびEND処理でのモニタ実行となるため、下記いずれかの場合はFFFFH(-1)がモニタされま す。 • ファイルレジスタを"使用しない"に設定したプログラムのあとでモニタ要求を実行した。 • ファイルサイズが小さいファイルレジスタを使用しているプログラムのあとで、よりファイルサイズが大きいファイ ルレジスタの値をモニタした。 |
| 上記以外 | END処理直前に実行したプログラムで有効となっているファイルレジスタファイルがモニタされます。 例) プログラムの実行順序がABC(END処理)AB の場合、プログラムC実行時のファイルレジスタ値をモニタ そのため、下記いずれかの場合はFFFFH(-1)がモニタされます。 • END処理直前のプログラムでファイルレジスタを"使用しない"に設定した。 • END処理直前のプログラムでファイルサイズが小さいファイルレジスタを使用しているときに、よりファイルサイズ が大きいファイルレジスタの値をモニタした。 |

下記いずれかの場合はモニタするファイルレジスタをデバイスに転送するプログラムを追加し、転送したプログラムをモニタしてください。ただし、追加したプログラム分のステップ数およびスキャンタイムが増加するので注意してください。

- ファイルレジスタを"使用しない"に設定したプログラムが存在する。

- ファイルレジスタのサイズが異なる。

なお、"プログラム別のファイルレジスタを使用する"に設定したファイルレジスタをラッチ不要なデータとして使用している場合(必ず初回にゼロクリアをしているような場合)、ローカルデバイスに置き換えることで、プログラム追加によるステップ数の増加およびスキャンタイムの増加がなくなります。

### ラベルをモニタする場合の注意事項

命令引数のデータ型(ラベル)は総称データ型で表現されるものがあり、ANY16やANY32などの"符号付き/符号なし"双方の属性を持つ場合があります。

総称データ型のモニタ値は、指定したラベルのデータ型で表示されます。命令の仕様*1に合わせたデータ型を指定してください。

*1詳細は、下記を参照してください。MELSEC MXコントローラ MX-RモデルプログラミングマニュアルMELSEC MXコントローラ MX-FモデルプログラミングマニュアルMELSEC iQ-R プログラミングマニュアル(CPUユニット用命令/汎用FUN/汎用FB編)MELSEC iQ-F FX5プログラミングマニュアル(命令/汎用FUN/汎用FB編)

Zoomのモニタ[表示][Zoom/起動先ブロックを開く]を選択し、Zoomを開いてからモニタしてください。

Zoomのモニタの操作/表示方法は、各プログラムエディタと同じです。

モニタ可能なデバイスは、ステップリレー(S)を除き各プログラムエディタと同じです。ステップリレー(S)*1は、Zoom内のみモニタできます。

注意事項

移行条件が成立したスキャンでモニタが実行されると、実行していたステップ(移行元のステップ)が非活性、移行条件に接続された次のステップ(移行先のステップ)が活性の状態で表示されます。なお、連続移行なしの場合、移行先のステップは次のスキャンで実行されます。そのため、移行先のステップと動作出力を同時にモニタすると、移行先ステップは活性になりますが動作出力のモニタ値は変化しません。

### SFC自動スクロールモニタ

SFC図モニタ開始後、[オンライン][モニタ][SFC自動スクロールモニタ]()を選択します。

モニタ中、画面上見えない位置にあるステップが活性した場合、自動的にスクロールして再び画面上に活性ステップを表示します。

複数ステップが活性している場合、イニシャルステップの列に近いステップを優先して表示します。

SFC自動スクロールモニタ中は、クロスリファレンスの自動追従およびウォッチウィンドウへの自動登録はできません。

自動スクロールモニタ中の起動先ブロックの表示自動スクロールモニタ中に活性ステップがブロック起動ステップに移動したときに、自動で起動先ブロックのSFC図エディタを開き、モニタを開始できます。

下記のオプションにより、自動で起動先ブロックを開くかどうかを設定できます。

- [ツール][オプション]"モニタ""SFC図エディタ""自動スクロールモニタ設定""ブロック起動は新しいウィンドウを開いてモニタする"

自動スクロールモニタ中のZoomの表示自動スクロールモニタ中にZoomを開いていると、活性ステップの表示に合わせてZoomの表示内容も自動的に切り替わります。

注意事項

モニタ周期よりステップの活性状態の更新速度が速いと、モニタのタイミングによってステップの活性状態を取得できない場合があります。

活性状態を取得できなかったステップには自動スクロールしません。

### SFC活性済みステップモニタ

SFCプログラムのモニタ中、ブロックが起動してから終了するまでに活性化したステップを任意の色で表示できます。

これにより、プログラムが実行された順序をわかりやすくします。

[オンライン][モニタ][SFC活性済みステップモニタ]により、任意の色で活性済みステップを表示できます。

RnPCPUおよびRnPSFCPUでは対応していません。

活性済みステップの表示モニタ中の活性済みステップは、下記のように表示されます。

![図(元マニュアル p.767)](images/gxworks3/14-02-18.webp)

注意事項

活性済みステップを表示してモニタする場合、モニタの周期は遅くなります。

下記の場合、活性済みステップが活性する前の状態に戻ります。

- CPUユニットの電源OFF

- CPUユニットのリセット

- SM321によるSFCプログラムの停止

- POFF命令によるSFCプログラムの停止

### 活性状態の変更

モニタ中は、下記の操作によりブロック/ステップの活性状態を変更できます。

特定のブロック/ステップのみを起動/終了させ、動作を確認する場合に使用します。

| 対象 | 操作画面 | 操作 |
|---|---|---|
| ブロック | 各プログラムエディタ(Zoomを含む) | SFCブロックデバイス(BL)の現在値(TRUE/FALSE)を変更します。*1 |
| | "デバイス/バッファメモリ一括モニタ"ウィンド ウ | |
| | ウォッチウィンドウ | |
| ステップ | SFC図エディタ | ステップを選択し、[デバッグ][SFCステップ制御][選択ステップを活性する]/[選択ス テップを非活性する]/[選択ステップのみ活性する]を選択します。 |
| | Zoom | ステップリレー(S)またはブロック指定付ステップリレー(BL¥S)の現在値(TRUE/ FALSE)を変更します。*1 |
| | 各プログラムエディタ(Zoomを除く) | ブロック指定付ステップリレー(BL¥S)の現在値(TRUE/FALSE)を変更します。*1 |
| | "デバイス/バッファメモリ一括モニタ"ウィンド ウ | |
| | ウォッチウィンドウ | |

*1ブロック指定付ステップリレー(BL¥S)およびSFCブロックデバイス(BL)を割り付けたラベルの現在値を変更して、ブロック/ステップを活性化することもできます。

注意事項

- ブロックまたはステップの活性状態の変更は、CPUユニットの制御に影響するため注意してください。

- SFCブロックのRUN中書込みを実行している場合、そのブロックは現在値を変更しても活性状態を変更できません。

### 現在値の変更(デバイス/ラベル)

モニタ中は、SFC図上で下記の現在値を変更できます。デバイス/ラベルを選択し、+を押してください。

- SFC図エディタの移行条件/動作出力のビットデバイス/ビット型のラベル

- 移行条件に接続されたFBD/LD要素のビットデバイス/ビット型のラベル

Zoom内のデバイス/ラベルの現在値の変更方法は、各プログラムエディタと同じです。

ステップリレー(S)、ブロック指定付ステップリレー(BL¥S)、およびSFCブロックデバイス(BL)の場合は、下記を参照してください。

### SFCブロックリストのモニタ

SFCブロックリスト上に現在のブロック情報を表示します。

[表示][SFCブロックリストを開く]を選択し、SFCブロックリストを開いてからモニタしてください。

![図(元マニュアル p.768)](images/gxworks3/14-02-19.webp)

モニタ中にブロック欄をダブルクリックすると、指定したブロックのSFC図が表示できます。

### SFC全ブロックの一括モニタ/活性ステップモニタ

全ブロック/ステップの活性状態を表示します。

CPUユニットをRUNからSTOPにしてもブロック/ステップの活性状態は変わりません。そのため、モニタ画面にはSTOPしたときの状態が表示されます。

注意事項

本モニタ機能は、CPUユニットに書き込まれているプログラムとプロジェクト内のプログラムに差異がある場合、正しくモニタできないことがあります。

SFCブロック一括モニタ全ブロックの現在の活性/非活性状態を一覧で表示します。

画面表示

[オンライン][モニタ][SFC全ブロック一括モニタ]()を選択します。

![図(元マニュアル p.769)](images/gxworks3/14-02-20.webp)

活性ステップモニタ指定したブロック内に存在するステップの現在の活性/非活状態を一覧で表示します。

モニタ画面は、5画面まで表示できます。

操作手順

"ブロック一括モニタ"画面でモニタするブロックにカーソルを移動し、[活性ステップモニタ]ボタンをクリックします。
