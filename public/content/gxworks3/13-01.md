# 13.1 シーケンサデータの読み書き

作成したデータのCPUユニットおよびSDメモリカードへの書込み/読出しは、"オンラインデータ操作"画面で行います。

## オンラインデータ操作画面の構成

**画面の開き方**

[オンライン] ⇒ [シーケンサへの書込み]/[シーケンサからの読出し]/[シーケンサのデータ削除]を選択します。

例R08CPUへの書込み時の画面例です。

![図(元マニュアル p.697)](images/gxworks3/13-01-01.webp)

- グレー表示されているデータ名は、未変換状態であることを示しています。

- シーケンサへの書込み時、書込み先の空き容量が実際のファイルサイズよりも小さく表示される場合があります。

- [サイズ計算]ボタンをクリックすることでメモリ容量のグラフ表示を更新できます。

- "オンラインデータ操作"画面では下記のメニューを使用できます。

## シーケンサへの書込み

CPUユニットへデータを書き込みます。

**書込み時の注意事項の関連項目**

→ オンラインデータ操作の注意事項

**操作手順**

1. "オンラインデータ操作"画面の[書込み]タブを選択します。

2. 書き込むデータと書込み先を選択します。

3. [詳細]ボタンをクリックし、書込み範囲などの詳細設定を行います。

4. [実行]ボタンをクリックします。

安全プロジェクトの場合、RnPSFCPUまたはRnSFCPUへ安全プログラムやパラメータを書込み後、安全にご使用いただくうえで使用者に確認いただきたい内容を明示するためにマニュアルが表示されます。

複数のプログラムを書き込んだ場合、"実行タイプ指定なし"のプログラムはCPUユニット上で実行されません。CPUユニットのメモリ削減のため、書込み時に対象からはずすことを推奨します。

- プログラム復元情報「シーケンサへの書込み」を実行する時に、プログラム復元情報を書き込むかどうかを選択できます。

プログラム復元情報とは、GX Works3プロジェクトをシーケンサから読み出し、復元する際に使用する情報です。プログラムの描画情報や、ローカルラベルの定義情報を含みます。

下記のオプション設定で、プログラム復元情報を書き込むかどうかの選択を"オンラインデータ操作"画面に表示できます。

- [ツール] ⇒ [オプション] ⇒ "オンライン" ⇒ "プログラム復元情報" ⇒ "動作設定" ⇒ "プログラム復元情報の書込み有無を設定可能にする"

"オンラインデータ操作"画面の"プログラム復元情報"で"プログラム復元情報を書き込む(書込み対象データ: プログラム復元情報)"を選択すると、プログラム復元情報のみをCPUユニットへ書き込むことができます。

安全プログラム、安全FB、および安全FUNを書き込む場合、"プログラム復元情報"で"プログラム復元情報を書き込まない(

書込み対象データ: プログラム)"を選択しても、プログラム復元情報はRnSFCPUへ書き込まれます。

プログラム復元情報を書き込まずにプログラムを書き込んだ場合、「シーケンサからの読出し」でデータを読み出すことができません。また、「シーケンサとの照合」で、詳細照合結果画面を表示できません。

なお、本機能に未対応のCPUユニットでは、オプションを設定してもプログラム復元情報の書込み有無を選択できません。

- 変更のないファイルの書込みを省略データをCPUユニットへ書き込むときに、前回の書込み後、内容が変更されていないデータの書込みを省略できます。これにより、書込みに必要な時間を短縮できます。

**操作手順**

1. "オンラインデータ操作"画面で[実行]ボタンをクリックします。

2. メッセージを確認し、"変更のないファイルの書込みを省略する。"にチェックを入れます。

**注意事項**

書込みを省略できないデータを示します。

- デバイスメモリ

- ファイルレジスタ特定の条件に該当する場合、"変更のないファイルの書込みを省略する。"にチェックが入っているかどうかにかかわらず、下記のデータは上書きされます。

| データ | 条件 |
|---|---|
| プログラムファイル | プログラム復元情報が書き込まれていない状態のCPUユニットにプログラム復元情報を書き込む |
| FBファイル(SlibFbFile含む) | プログラム復元情報が書き込まれていない状態のCPUユニットにプログラム復元情報を書き込む |
| FUNファイル | プログラム復元情報が書き込まれていない状態のCPUユニットにプログラム復元情報を書き込む |
| グローバルラベル割付情報 | グローバルラベル設定の書込みが省略されない |
| グローバルラベル初期値 | グローバルラベル設定の書込みが省略されない |
| ローカルラベル初期値 | プログラムファイルの書込みが省略されない |

下記の場合、内容が変更されていなくても、データが上書きされます。

- プログラムの復元情報がRnPCPU(二重化モード)の制御系または待機系のいずれかにしか存在しない

- 書込み先にインテリジェント機能ユニットを選択した

- プログラム部品の重複チェックシーケンサへの書込み時に、書込み対象のプログラム部品とCPUユニット内のプログラム部品で、名前が重複していないかをチェックします。

二重化システム構成のCPUユニット使用時、両系に対してシーケンサへの書込みを行う場合は、両系に対してプログラム部品の名前が重複していないかをチェックします。

下記のオプション設定により、チェックするかどうかを設定できます。

- [ツール] ⇒ [オプション] ⇒ "オンライン" ⇒ "シーケンサへの書込み" ⇒ "動作設定" ⇒ "プログラム部品の重複チェックを実施する"

- モーション設定チェック"モーション設定"または"グローバルラベル初期値"にチェックが入っている場合、シーケンサへの書込み時にモーション設定の内容にエラーがないかをチェックします。

下記のオプション設定により、チェックするかどうかを設定できます。

- [ツール] ⇒ [オプション] ⇒ "オンライン" ⇒ "シーケンサへの書込み" ⇒ "動作設定" ⇒ "書込み時にモーション設定チェックを実施する"

- 二重化システム構成のCPUユニット使用時の動作RnPCPU(二重化モード)は、運転モードにより動作が異なります。

RnPSFCPUの運転モードはバックアップモードで動作します。

![図(元マニュアル p.702)](images/gxworks3/13-01-02.webp)

R04CPU、R08CPU、R16CPU、R32CPU、R120CPUのファームウェアバージョン30以前から31以降にアップデートを実行すると、データ配置が変更されます。

**動作の関連項目**

→ MELSEC iQ-RシーケンサCPUユニットユーザーズマニュアル

→ MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

そのため、バックアップしたデータをCPUユニットに書き込むと、プログラムメモリの空き容量が足りなくなる場合があります。空き容量が足りなくなった場合は、下記のいずれかの操作後に再度書き込んでください。

- "RUN中書込用確保ステップ"を削減する。(デフォルト: 500ステップ)

- プログラムステップ数を削減する。

- パラメータの整合性について一部のパラメータを追加で書き込む場合、およびパラメータを変更して書き込む場合は、パラメータ間の整合性を保った運用が必要です。整合性とは、システムパラメータと各パラメータ間でI/O割付設定などの情報(先頭I/O No.、スロットNo.など)を揃えることです。

整合性を保つには、2通りの方法があります。

- システムパラメータと各パラメータを読み出し、変更します。その後、システムパラメータと各パラメータを合わせて

CPUユニットへ書き込みます。システムパラメータを変更するため、CPUユニットのリセットが必要になります。

- 変更する各パラメータを読み出し、プロジェクト内のシステムパラメータと整合性が合うように変更します。その後、変更したパラメータのみCPUユニットへ書き込む。CPUユニットのリセットは、基本的には必要ありません。ただし、一部のユニットのパラメータを変更した場合はリセットが必要ですので、使用するユニットのマニュアルを参照してください。

- 先頭I/O No.が未設定のユニットの場合未設定のユニットパラメータは、シーケンサへの書込みが行えません。

- CC-Link IEフィールドネットワークのリモートI/Oユニット(デバイス局)のパラメータデバイス局へのパラメータの書込みは、"CC IE Field構成"ウィンドウで行います。詳細は、各リモートI/Oユニットのマニュアルを参照してください。

- CC-Link IE TSNのリモートI/Oユニット(デバイス局)のパラメータ下記の操作によって、デバイス局のパラメータをCPUユニットに書き込むとき、そのパラメータをデバイス局に設定できます。

**操作手順**

1. "CC-Link IE TSN構成"ウィンドウで、デバイス局の"パラメータ自動設定"にチェックを入れ、"<詳細設定>"を設定します。

2. "オンラインデータ操作"画面で、"CC-Link IE TSN構成" ⇒ "デバイス局設定"/"デバイス局設定(安全)"にチェックを入れます。

3. [詳細]ボタンをクリックします。

4. "CC-Link IE TSN構成-デバイス局設定"画面で、パラメータを書き込むデバイス局にチェックを入れます。

**"パラメータ自動設定"の関連項目**

→ MELSEC MXコントローラMX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラMX-Fモデルユーザーズマニュアル

→ MELSEC iQ-R CC-Link IE TSNユーザーズマニュアル(応用編)

→ MELSEC iQ-R CC-Link IE TSN Plusマスタ・ローカルユニットユーザーズマニュアル

→ MELSEC iQ-F FX5 CC-Link IE TSNマスタ・ローカルユニットユーザーズマニュアル

→ MELSEC iQ-Rモーションユニットユーザーズマニュアル(ネットワーク編)

→ MELSEC iQ-F FX5モーションユニットユーザーズマニュアル (CC-Link IE TSN編)

- 二重化システム構成におけるユニット拡張パラメータの書込み二重化システム構成において、インテリジェント機能ユニットにユニット拡張パラメータを書き込む場合、下記の制限があります。

- トラッキングケーブルを経由する場合、インテリジェント機能ユニットにユニット拡張パラメータを書き込めません。

- 接続系が制御系以外のCPUユニットの場合、二重化システム用増設ベースユニットに装着したインテリジェント機能ユニットにユニット拡張パラメータを書き込めません。

- CPUユニット、GX Works3両方のCPUパラメータに設定がない場合: "実行タイプ指定なし"で復元する

- 読出し範囲の設定"オンラインデータ操作"画面の[詳細]ボタンをクリックすると表示される"プログラム詳細設定"画面で、読み出すプログラム

(プログラムファイル)の範囲を設定します。

また、CPUユニットからRUN中書込用確保ステップを取得できます。

FX5CPUでは対応していません。

### グローバルラベル

- 初期値、外部機器からのアクセスシーケンサへの書込みとシーケンサからの読出しで動作が異なります。

- シーケンサへの書込み時: "グローバルラベル初期値"および"グローバルラベル割付情報"として書き込む

- シーケンサからの読出し時: プログラムを読み出すことでグローバルラベルの"初期値"および"外部機器からのアクセス"の情報も読み出される

FX5CPUは、ラベルの初期値と外部機器からのアクセスの設定には対応していません。

グローバルラベルが使用されているプログラムをプログラム単体で読み出すと、プロジェクト内にプログラムで使用しているラベルの定義が存在しない状態になることがあります。ラベル定義が存在しない場合、回路上のラベルの文字列がなくなり空欄になります。

その場合は、グローバルラベルをプログラムと同時に読み出してください。

## CPUユニット内のデータ削除

CPUユニット内のプログラムやパラメータなどを削除します。

安全プロジェクトの場合は、CPUユニットの安全動作モードがテストモード時に安全データの削除が可能です。

**操作手順**

1. "オンラインデータ操作"画面の[削除]タブを選択します。

2. 削除するファイルを選択し、[実行]ボタンをクリックします。

- 二重化システム構成のCPUユニット使用時の動作RnPCPU(二重化モード)は、運転モードにより動作が異なります。

RnPSFCPUの運転モードはバックアップモードで動作します。

| 運転モード | 動作 |
|---|---|
| セパレートモード | 接続系のCPUユニットに対して実行します。 |
| バックアップモード | 両系(制御系→待機系の順番)に実行します。 制御系のデータを削除中にエラーが発生した場合、待機系のデータは削除されません。待機系のデータを削除中にエラーが発生した場合、制御系のデータは削除前のデータに戻りません。 |

CPUユニットへ書き込んだあとに構造体の定義を編集した場合、構造体を使用しているデータを読み出したときに構造体の定義が編集前の状態に戻るため、注意してください。

- 一般プログラム

- 安全プログラム

- 一般FB

- 安全FB

- 一般FUN

- 安全FUN

- 一般グローバルラベル

- 一般/安全グローバルラベル

- 安全グローバルラベル
