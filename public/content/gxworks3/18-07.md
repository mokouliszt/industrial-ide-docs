# 18.7 ラベルメモリの読出し/書込み

CPUユニットに対してラベルメモリ(ラベルの現在値)を読み出す方法、および読み出したラベルメモリを書き込む方法を説明します。

FX5CPUでは対応していません。

### 読出し/書込み可能な条件

下記のすべての条件を満たしている場合にのみ操作を行えます。

| 操作 | 条件 |
|---|---|
| ラベルメモリの読出し | プロジェクトにラベルメモリを読み出せるラベルが存在する。(→ ラベルメモリの読出し可否) |
| | プロジェクト内のすべてのプログラムが変換されている。 |
| | 下記のデータ間でシステムパラメータおよびCPUパラメータが一致している。 • GX Works3で開いているプロジェクトデータ • CPUユニットのデータ RnSFCPUおよびRnPSFCPUの場合、安全CPUパラメータも一致している必要があります。 |
| | ユーザー認証が有効な場合、下記いずれかのアクセスレベルのユーザでプロジェクトおよびCPUユニットにログオン している。 • Administrators • ユーザ定義グループ(MXコントローラ) • Users(MXコントローラ) • Developers(RnSFCPUおよびRnPSFCPU) • Assistant Developers(RnSFCPUおよびRnPSFCPU) |
| ラベルメモリの書込み | プロジェクト内のすべてのプログラムが変換されている。 |
| | CPUユニットがSTOP状態である。 |
| | ラベルメモリの読出し時と書込み時のCPUユニットの機種/動作モードが一致している。 |
| | CPUユニットがテストモードである。(RnSFCPUおよびRnPSFCPUの場合のみ) |
| | 下記のデータ間でシステムパラメータおよびCPUパラメータが一致している。 • ラベルメモリを読み出したときのデータ • CPUユニットのデータ RnSFCPUおよびRnPSFCPUの場合、安全CPUパラメータも一致している必要があります。 |
| | ユーザー認証が有効な場合、下記いずれかのアクセスレベルのユーザでプロジェクトおよびCPUユニットにログオン している。 • Administrators • ユーザ定義グループ(MXコントローラ) • Users(MXコントローラ) • Developers(RnSFCPUおよびRnPSFCPU) • Assistant Developers(RnSFCPUおよびRnPSFCPU) |

### ラベルメモリの読出し

CPUユニットからラベルメモリを読み出します。

ラベルメモリを読み出す前に、CPUユニットに対して現在値変更などの機能を実行していないか確認してください。

読出し中に現在値が変更された場合、変更後の値が読み出されます。

操作手順1. [オンライン] ⇒ [CPUメモリ操作]を選択します。2."CPUメモリ操作"画面で"CPU内蔵メモリ"の"デバイス/ラベルメモリ"を選択します。3. [ラベルメモリの読出し/書込み]ボタンをクリックします。4."ラベルメモリの読出し/書込み"画面の"ファイル格納先"で、ラベルメモリの格納先を指定します。5."ラベルメモリからの読出し"を選択し、[実行]ボタンをクリックします。

手順4で指定した格納先に"BackupData"フォルダーが生成され、"BackupData"フォルダーにラベルメモリがファイルとして格納されます。

注意事項

- ラベルメモリの読出し中にCPUユニットをリセットしたり、CPUユニットの電源を切ったりしないでください。ラベルメモリの読出しが中断されます。(→ ラベルメモリの読出しが中断された場合)

- ラベルメモリの格納先にすでに"BackupData"フォルダーが存在する場合、そのフォルダーおよびフォルダー内のすべてのデータが削除された後、新たに"BackupData"フォルダーと各データが生成されます。

- RnPCPU(二重化モード)からラベルメモリを読み出す場合、接続系から読み出されます。

- ラベルメモリの読出し時に生成された"BackupData"フォルダー内の"LabelChg_before.csv"が300MBを超える場合、GX

Works3が使用するパソコンのメモリが不足し、読出しに失敗することがあります。

失敗した場合は、Windowsを再起動し再度実行してください。

それでも失敗する場合はプロジェクト上のラベルを減らす、またはラベル名を短くし、"LabelChg_before.csv"が300MB

以内となるように見直してください。

- ラベルメモリの読出し可否ラベルのクラスごとに、ラベルメモリを読み出せるかどうかを示します。

○: 読出し可、×: 読出し不可

| ラベル | クラス | 読出し可否 |
|---|---|---|
| グローバルラベル | VAR GLOBAL _ | ○ |
| | VAR GLOBAL CONSTANT _ _ | × |
| | VAR GLOBAL RETAIN _ _ | ○ |
| ローカルラベル | VAR INPUT _ | ○ |
| | VAR OUTPUT _ | ○ |
| | VAR OUTPUT RETAIN _ _ | ○ |
| | VAR | ○ |
| | VAR CONSTANT _ | × |
| | VAR RETAIN _ | ○ |
| | VAR IN OUT _ _ | ○ |
| | VAR PUBLIC _ | ○ |
| | VAR PUBLIC RETAIN _ _ | ○ |

下記のラベルに対しては、ラベルメモリを読み出せません。

- ポインタ型のラベル

- デバイスが割り付けられたラベル

- エイリアス

- ファンクション

- マクロ型ファンクションブロックのFBインスタンス

- 汎用ファンクションブロックのFBインスタンス

- ユニットFBのFBインスタンス

- "実行タイプ指定なし"のプログラム

- 未登録プログラムのローカルラベル

- ラベルメモリの読出しが中断された場合下記のいずれかの場合、ラベルメモリの読出しが中断されます。

- 読出し中にCPUユニットをリセットした。

- 読出し中にCPUユニットの電源を切った。

- 読出しをキャンセルした。

- ラベルメモリを読み出せなかった。(読出し先の空き容量が不足している場合や読出し先へのアクセス権がない場合など)

- 読出し中にGX Works3が使用できるパソコンのメモリが不足した。

読出しが中断された場合、"BackupData"フォルダーおよび"BackupData"フォルダー内のすべてのファイルが削除されます。

### ラベルメモリの書込み

CPUユニットにラベルメモリを書き込みます。

操作手順1. [オンライン] ⇒ [CPUメモリ操作]を選択します。2."CPUメモリ操作"画面で"CPU内蔵メモリ"の"デバイス/ラベルメモリ"を選択します。3. [ラベルメモリの読出し/書込み]ボタンをクリックします。4."ラベルメモリの読出し/書込み"画面の"ファイル格納先"で、ラベルメモリの読出し時に"BackupData"フォルダーが生成されたフォルダーを指定します。5."ラベルメモリへの書込み"を選択し、[実行]ボタンをクリックします。

下記のCPUパラメータを"保存する"に設定すると、ラベルメモリの書込み操作がデバイス書込み(n点)やラベル書込み(n点)のイベントとして"イベント履歴"画面に表示されます。

- "CPUパラメータ" ⇒ "RAS設定" ⇒ "イベント履歴設定" ⇒ "デバイス/ラベル操作の保存設定"

"パラメータをチェックする"にチェックを入れると、ラベルメモリへの書込み時に、CPUユニット内のデータと書き込むデータに含まれるパラメータ(システムパラメータおよびCPUパラメータ)を比較できます。

注意事項

- ラベルメモリの書込み中にCPUユニットをリセットしたり、CPUユニットの電源を切ったりしないでください。ラベルメモリの書込みが中断されます。

- "BackupData"フォルダー内のファイルを編集しないでください。ラベルメモリが正常に書き込まれない可能性があります。

- ラベル名の大文字と小文字は区別されます。

- デバイスが割り付けられたラベルのラベルメモリは書き込まれません。

- ラベルメモリを読み出してから再度書き込むまでにラベルの下記内容を編集した場合、そのラベルのラベルメモリは書き込まれません。

- ラベル名

- データ型

- 配列の次元数また、プログラム名を編集した場合、そのプログラムで使用しているローカルラベルのラベルメモリは書き込まれません。(大文字と小文字は区別されます。)

- ラベルメモリを読み出してから再度書き込むまでに配列の要素数を変更した場合は、ラベルメモリを書き込む前にデータをシーケンサへ書き込む必要があります。

また、ラベルメモリを書き込む時、変更前の要素数分のラベルメモリのみが書き込まれます。

- プログラムのモニタ中にラベルメモリを書き込むと、モニタが一時停止し、書込みの完了後にモニタが再開されます。

- GX Works3以外のエンジニアリングツールでプログラムをモニタしている場合、ラベルメモリの書込み完了時にモニタ値が変更されることがあります。

- RnPCPU(二重化モード)にラベルメモリを書き込む場合、接続系に書き込まれます。

- ラベルメモリへの書込み時に指定した"BackupData"フォルダー内の"LabelChg_before.csv"または"LabelChg_after.csv"が

300MBを超える場合は、GX Works3が使用するパソコンのメモリが不足し、書込みに失敗することがあります。

失敗した場合は、Windowsを再起動し再度実行してください。

それでも失敗する場合はプロジェクト上のラベルを減らす、またはラベル名を短くし、"LabelChg_before.csv"または

"LabelChg_after.csv"が300MB以内となるように見直してください。
