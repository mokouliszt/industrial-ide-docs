# 10.0 プログラムの部品化(機能概要)

## 10 プログラムの部品化

プログラム内で繰り返し使用する処理プログラムを部品化してシーケンスプログラムの中で再利用できます。

これにより、プログラム開発を効率化するとともにプログラムミスを削減し、プログラム品質が向上します。

部品化したプログラムファイルをプログラム部品(→ データ構成)と呼びます。

また、複数のプログラム部品をプロジェクトとは別のファイルにまとめることができます。これをライブラリと呼びます。

### プログラム部品

再利用可能なプログラム部品には、ファンクションブロックとファンクションがあります。

リモートヘッドユニットでは対応していません。

- ファンクションブロック内部メモリを持ち、その値と入力に従って演算結果を出力するプログラムを部品化したものです。

ファンクションブロックには、下記の種類があります。

| 種類 | 内容 | 参照 |
|---|---|---|
| ファンクションブロック | ユーザが作成するファンクションブロックです。なお、RnPCPU(プロセスモード/二重化モード)およびRnPSFCPUのプロジェクトでは、メーカ定義タグFBのプロセス制御処理をカスタマイズしたファンクションブロック(ユーザ定義タグFB)を作成できます。 | → ファンクションブロックの作成→ ユーザ定義タグFBの作成 |
| 汎用ファンクションブロック | 汎用的に使用できるファンクションブロックです。GX Works3内にあらかじめ用意されています。 | → 部品選択ウィンドウ |
| プロセス制御ファンクションブロック | プロセス制御用のファンクションブロックです。GX Works3内にあらかじめ用意されています。なお、プロセス制御ファンクションブロックは、下記のプログラム内で使用できます。• プロセス制御用FBD/LDプログラム• FBD/LDプログラム内に配置されるファンクションブロックのFBプログラム(FBD/LDプログラム)• FBD/LDプログラム内に配置されるユーザ定義タグFBのFBプログラム(FBD/LDプログラム) | → 部品選択ウィンドウ |
| モーション制御ファンクションブロック | モーション制御用のファンクションブロックです。GX Works3内およびアドオン内にあらかじめ用意されています。 | → 部品選択ウィンドウ |
| ユニットFB | ユニット用の処理を部品化したファンクションブロックです。GX Works3内にあらかじめ用意されています。 | → ユニットFBの活用 |

- ファンクション内部メモリを持たず、同じ入力に対して、常に同じ演算結果を出力するプログラムを部品化したものです。

ファンクションには、下記の種類があります。

| 種類 | 内容 | 参照 |
|---|---|---|
| ファンクション | ユーザが作成するファンクションです。 | → ファンクションの作成 |
| 汎用ファンクション | 汎用的に使用できるファンクションです。GX Works3内にあらかじめ用意されています。 | → 部品選択ウィンドウ |
| モーション制御ファンクション | モーション制御用のファンクションです。GX Works3内およびアドオン内にあらかじめ用意されています。 | → 部品選択ウィンドウ |

### ライブラリ

複数のプログラム部品、構造体を1つにまとめたものです。ライブラリ内の部品を複数のプロジェクトで使用できます。

- ユーザライブラリ作成済みのプログラム部品、構造体から作成する部品集です。

詳細は、下記を参照してください。

→ ユーザライブラリの活用

- アプリケーションライブラリ/MELSOFT Library(サンプルライブラリ)メーカによってあらかじめ用意されている部品集です。

三菱電機FAサイトからダウンロードし、プロジェクトまたはGX Works3に取り込むことで使用できます。

詳細は、下記を参照してください。

→ アプリケーションライブラリ/MELSOFT Libraryの活用リモートヘッドユニットでは対応していません。
