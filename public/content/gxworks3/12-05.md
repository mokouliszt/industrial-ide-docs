# 12.5 シリアルコミュニケーションユニット経由

シリアルコミュニケーションユニットを経由して自局や他局のCPUユニットにアクセスするときの設定方法を説明します。

MX-Fモデル、LHCPU、およびFX5CPUでは対応していません。

## 1 :1接続

パソコンとシリアルコミュニケーションユニットを接続してCPUユニットにアクセスする場合の設定例を示します。

![図(元マニュアル p.687)](images/gxworks3/12-05-01.webp)

| No. | 項目 | 選択項目 | 内部設定 | 入力値 |
|---|---|---|---|---|
| (1) | パソコン側I/F | シリアル/USB | COMポート | COM1 |
| | | | 伝送速度 | 115.2Kbps |
| (2) | シーケンサ側I/F | C24 | 形名 | RJ71C24 |
| | | | 局番*1 | ― |
| | | | パリティ | ― |
| | | | サムチェック | ― |
| | 他局指定 | 他局指定無し | 交信タイムチェック | ― |
| | | | リトライ回数 | ― |

*1RJ71C24と、局番を同じ値に設定します。局番は、ユニットパラメータで設定してください。

## 1 :n接続

CPUユニットが複数局あるシステムで、パソコンから他局のCPUユニットにアクセスする方法を説明します。

### シリアルコミュニケーションユニット経由時

シリアルコミュニケーションユニットを経由して他局のCPUユニットにアクセスする場合の設定例を示します。

シリアルコミュニケーションユニットを経由する場合、MELSOFT接続するためのユニットパラメータの設定が必要です。

詳細は、下記を参照してください。

→ MELSEC iQ-R シリアルコミュニケーションユニットユーザーズマニュアル(応用編)

![図(元マニュアル p.688)](images/gxworks3/12-05-02.webp)

| No. | 項目 | 選択項目 | 内部設定 | 入力値 |
|---|---|---|---|---|
| (1) | パソコン側I/F | シリアル/USB | COMポート | COM1 |
| (1) | パソコン側I/F | シリアル/USB | 伝送速度 | 115.2Kbps |
| (2) | シーケンサ側I/F | C24 | 形名 | RJ71C24 |
| (2) | シーケンサ側I/F | C24 | 局番 | 0 |
| (2) | シーケンサ側I/F | C24 | パリティ | 奇数 |
| (2) | シーケンサ側I/F | C24 | サムチェック | ― |
| (2) | 他局指定 | 他局(単一ネットワーク) | 交信タイムチェック | 30秒 |
| (2) | 他局指定 | 他局(単一ネットワーク) | リトライ回数 | 0回 |
| (3) | ネットワーク通信経路 | C24 | 先頭I/O | 20 |
| (3) | ネットワーク通信経路 | C24 | 局番 | 1 |

### CPUユニット直結時

パソコンとCPUユニットを直結し、シリアルコミュニケーションユニットを経由して他局のCPUユニットにアクセスする場合の設定例を示します。

シリアルコミュニケーションユニットのCH2側でのみ他局のシーケンサCPUにアクセスできます。

LHCPUにはアクセスできません。

![図(元マニュアル p.688)](images/gxworks3/12-05-03.webp)

| No. | 項目 | 選択項目 | 内部設定 | 入力値 |
|---|---|---|---|---|
| (1) | パソコン側I/F | シリアル/USB | USB | ― |
| (2) | シーケンサ側I/F | CPUユニット | CPUモード | RCPU/MXR |
| (2) | 他局指定 | 他局(単一ネットワーク) | 交信タイムチェック | 30秒 |
| (2) | 他局指定 | 他局(単一ネットワーク) | リトライ回数 | 0回 |
| (3) | ネットワーク通信経路 | C24 | 先頭I/O | 20 |
| (3) | ネットワーク通信経路 | C24 | 局番 | 1 |
