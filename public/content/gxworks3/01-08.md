# 1.8 プロジェクト作成からCPUユニット稼働までの手順

プログラムを作成してCPUユニットを稼働するまでのGX Works3での操作手順を示します。

CPUユニットの操作方法は、下記を参照してください。

MELSEC MXコントローラ MX-Rモデルユーザーズマニュアル

MELSEC MXコントローラ MX-Fモデルユーザーズマニュアル

MELSEC MXコントローラ MX-Fモデル(EtherCAT対応)ユーザーズマニュアル

MELSEC iQ-R シーケンサCPUユニットユーザーズマニュアル

MELSEC iQ-R プロセスCPUユニットユーザーズマニュアル

MELSEC iQ-R SIL2プロセスCPUユニットユーザーズマニュアル

MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

MELSEC iQ-F FX5ユーザーズマニュアル(応用編)

MELSEC iQ-R CC-Link IEフィールドネットワークリモートヘッドユニットユーザーズマニュアル(応用編)

### 一般プロジェクトの場合

対象機種: MXコントローラ、RnCPU、RnENCPU、RnPCPU(プロセス制御拡張を無効)

開　　始

1. プロジェクトを新規作成する

GX Works3 を起動します。

プロジェクトを新規作成します。

→ 起動→ 新規作成する

2. パラメータを設定する

→ パラメータの設定

3. プログラムを作成するプログラム部品(POU) を作成します。

実行順序/ 実行タイプを設定します。

グローバルラベル/ ローカルラベルを設定します。

各プログラム部品のプログラムを編集します。

変換します。

→ 新規作成→ プログラムの実行順序/実行タイプの設定→ ラベルの登録→ ラダープログラムの作成、→ STプログラムの作成、→ FBD/LDプログラムの作成、→ SFCプログラムの作成、→ ファンクションブロックの作成、→ ファンクションの作成→ プログラムの変換

4. シミュレータでデバッグする

→ プログラムのシミュレーション

5. パソコンとCPU ユニットをケーブルで接続し、接続先を設定する

→ CPUユニットへの経路設定

6. CPU ユニットにパラメータ/ プログラムを書き込む

→ シーケンサへの書込み

7. 動作を確認するプログラムの実行状態、デバイスの内容をモニタして動作を確認します。

CPU ユニットのエラー発生状況を確認します。

→ 実行プログラムをプログラムエディタ上で確認する→ ユニットの診断

8. 稼働する

1. プロジェクトを新規作成する

GX Works3 を起動します。

プロジェクトを新規作成します。

→ 起動→ 新規作成する

→ パラメータの設定→ ネットワーク構成、相手機器の設定→ ネットワークラベル設定

3. モーション設定を設定するモーションシステム設定を設定します。

軸データ、軸パラメータを設定します。

軸グループを設定します。

モーション入出力データを設定します。

演算プロファイルを設定します。

→ モーションシステム設定→ 軸データの作成、→ 軸パラメータの設定→ 軸グループの作成→ モーション入出力データの作成→ 演算プロファイルの作成

4. プログラムを作成するプログラム部品(POU) を作成します。

実行順序/ 実行タイプを設定します。

グローバルラベル/ ローカルラベルを設定します。

各プログラム部品のプログラムを編集します。

変換します。

→ 新規作成→ プログラムの実行順序/実行タイプの設定→ ラベルの登録→ ラダープログラムの作成、→ STプログラムの作成、→ FBD/LDプログラムの作成、→ ファンクションブロックの作成、→ ファンクションの作成→ プログラムの変換

5. パソコンとCPU ユニットをケーブルで接続し、接続先を設定する

→ CPUユニットへの経路設定

6. CPU ユニットにパラメータ/ プログラムを書き込む

→ シーケンサへの書込み

7. 動作を確認するプログラムの実行状態、デバイスの内容をモニタして動作を確認します。

CPU ユニットのエラー発生状況を確認します。

軸モニタ、位置データ履歴で動作を確認します。

→ 実行プログラムをプログラムエディタ上で確認する→ ユニットの診断→ MXコントローラのモーション動作を確認する

8. 稼働する

### プロセス制御用プロジェクトの場合

対象機種: RnPCPU(プロセス制御拡張を有効)

プログラムの作成については、本項に記載の内容のほかに、下記も参照してください。

MELSEC iQ-R プログラミングマニュアル(プロセス制御FB/命令編)

開　　始

1. プロジェクトを新規作成する

GX Works3 を起動します。

プロジェクトを新規作成します。

→ 起動→ 新規作成する

2. パラメータを設定する

→ パラメータの設定

3. 計装機能を使用するための設定を行うプログラムファイルに対してプロセス制御拡張を有効にします。

プロセス制御拡張のオプションを設定します。

変換操作のオプションを設定します。

CPU パラメータを設定します。

→ 計装機能を使用するための設定

4. プログラムを作成するタグFB を登録します。

プロセス制御用FBD/LD プログラムを作成します。

FB プロパティの初期値を設定します。

シーケンス制御用プログラムを作成します。

変換します。

→ タグFBの登録→ 新規作成→ FBプロパティの表示/設定、→ FBプロパティ管理(オフライン)→ FBD/LDプログラムの作成、→ ファンクションブロックの作成、→ タグFBの流用、→ ユーザ定義タグFBの作成、→ ファンクションの作成→ プログラムの変換

5. パソコンとCPU ユニットをケーブルで接続し、接続先を設定する

→ CPUユニットへの経路設定

6. CPU ユニットにパラメータ/ プログラムを書き込む

→ シーケンサへの書込み

7. 動作を確認するプログラムの実行状態、デバイスの内容をモニタして動作を確認します。

CPU ユニットのエラー発生状況を確認します。

フェースプレートでタグFB の制御状態を確認します。

FB プロパティの現在値をウォッチウィンドウで変更します。

FB プロパティの現在値をFB プロパティの初期値に反映します。

→ 実行プログラムをプログラムエディタ上で確認する、→ ファンクションブロックを一時停止/再開する→ ユニットの診断→ デバイス/ラベルを登録して現在値を確認する→ タグデータを確認する→ FBプロパティ管理(オンライン)画面

8. 稼働する

各プログラム部品のプログラムを編集します。

変換します。

4. パソコンとCPU ユニットをケーブルで接続し、接続先を設定する

5. CPU ユニットにパラメータ/ プログラムを書き込む

→ シーケンサへの書込み

6. 動作を確認する( テストモード)

プログラムの実行状態、デバイスの内容をモニタして動作を確認します。

CPU ユニットのエラー発生状況を確認します。

→ 実行プログラムをプログラムエディタ上で確認する→ ユニットの診断

7. 安全動作モードを切り替える書き込まれたプロジェクトが正しいか確認します。

セーフティモードに切り替えます。

→ 安全データ同一性チェック→ 安全動作モード切替え

8. 稼働する
