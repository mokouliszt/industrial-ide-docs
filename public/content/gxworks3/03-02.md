# 3.2 プロジェクトファイルの作成

プロジェクトを新規作成する、開く、保存するなどのプロジェクトを操作する方法について説明します。

## 新規作成する

プロジェクトを新規作成する方法を説明します。

安全プロジェクトの新規作成時には、ユーザを登録する必要があります。(→ 新しいユーザの追加)

画面表示

[プロジェクト] ⇒ [新規作成]を選択します。

![図(元マニュアル p.116)](images/gxworks3/03-02-01.webp)

**操作手順**

各項目を選択し、[OK]ボタンをクリックします。

注意事項

安全プロジェクトを作成する場合は、Windowsのスクリーンセーバー機能により、一定の時間パソコンを操作しない状態が続いたときに自動的にロックされるようにしておいてください。

詳細は、Windowsのヘルプとサポートを参照してください。

- MXコントローラのプロジェクトを作成する場合プロジェクトの新規作成時に、ユーザ認証機能を有効化できます。

詳細は下記を参照してください。

→ MXコントローラのプロジェクトへの不正なアクセスの防止(ユーザ認証)

- RnENCPUのプロジェクトを作成する場合RnENCPUは、CPU部とネットワーク部からなる2スロットを占有するCPUユニットです。

新規作成後はCPU部のみが単体で配置されています。

ネットワーク部を、下記のいずれかの方法で配置してください。

- "ユニット構成図"ウィンドウで、CPU拡張ユニット(_RJ71EN71)を配置

- "システムパラメータ"の"I/O割付設定"で、CPU拡張ユニット(_RJ71EN71)を設定

- LHCPUのプロジェクトを作成する場合LHCPUのプロジェクトを新規作成すると、"システムパラメータ"の"I/O割付設定"にCPU拡張ユニット(_IO)が自動で設定されます。

- MX-FモデルおよびFX5CPUの詳細な形名を指定する場合MX-FモデルおよびFX5CPUの詳細な形名は、"新規作成"画面では指定できません。下記のいずれかの方法で指定してください。

- ユニット構成図でCPUユニットを右クリック ⇒ ショートカットメニュー[コントローラ形名変更]/[CPU形名変更]を選択

- "システムパラメータ"画面の[I/O割付設定]タブ内の"I/O割付設定"で、CPUユニットの形名を選択

- QCPU(Qモード)、LCPU、またはFXCPUを選択した場合QCPU(Qモード)、LCPU、またはFXCPUを選択した場合は、Q/L/FXシリーズ互換モードでGX Works3が起動します。

Version 1.566Q以降のGX Works2がインストールされている場合のみ、自動的に起動します。

インストールされていない場合は、GX Works3のインストールDVD(Disk2)内の"setup.exe"を実行してください。

### 新しいユーザの追加

RnPSFCPUまたはRnSFCPUで安全プロジェクトを新規作成する場合、"新しいユーザの追加"画面でユーザを登録します。

画面表示1."新規作成"画面の"機種"でRnPSFCPUまたはRnSFCPUを選択します。

2. [OK]ボタンをクリックします。

![図(元マニュアル p.117)](images/gxworks3/03-02-02.webp)

**操作手順**

1. 各項目を設定し、[OK]ボタンをクリックします。

GUESTユーザを登録する場合は、"GUESTユーザを追加する"にチェックを入れてください。(→ GUESTユーザ)

2. "名前を付けて保存"画面で各項目を設定し、[保存]ボタンをクリックします。

### シーケンサからの新規読出し

プロジェクトを新規に作成していない状態でシーケンサからの読出しを実行すると、CPUユニットおよびインテリジェント機能ユニットから読み出したデータでプロジェクトを新規に作成できます。

QCPU(Qモード)、LCPU、またはFXCPUを選択した場合は、Q/L/FXシリーズ互換モードでGX Works3が起動し、プロジェクトを新規作成できます。

また、プロジェクトにユーザ情報が登録されている場合は、ユーザ認証が必要です。詳細は下記を参照してください。

(MXコントローラ: → MXコントローラへのログオン、RnSFCPUおよびRnPSFCPU: → CPUユニットへのログオン)

**操作手順**

1. GX Works3を起動後、[オンライン] ⇒ [シーケンサからの読出し]を選択します。

2. "シリーズ選択"画面で読み出す対象のシリーズを選択し、[OK]ボタンをクリックします。

3. "接続先指定"画面でCPUユニットとアクセスするための通信経路を設定し、[OK]ボタンをクリックします。

4. "オンラインデータ操作"画面で読出しを実行します。

"オンラインデータ操作"画面での読出し方法は、下記を参照してください。

→ シーケンサからの読出し注意事項シーケンサからの新規読出しでパラメータを読み出していない場合は、新規作成時のデフォルトのパラメータが設定されます。パラメータの設定を確認してください。

## 開く

パソコンのハードディスクなどに保存されているプロジェクトを読み出します。

また、プロジェクトにユーザ情報が登録されている場合は、ユーザ認証が必要です。(→ MXコントローラのプロジェクトへのログオン、→ プロジェクトへのログオン)

画面表示

[プロジェクト] ⇒ [開く]を選択します。

![図(元マニュアル p.119)](images/gxworks3/03-02-03.webp)

**操作手順**

各項目を設定し、[開く]ボタンをクリックします。

注意事項

- ワークスペース名、フォルダー構成などの変更ワークスペース、プロジェクトのフォルダーおよびファイルの格納場所の変更、ファイル名の変更などをWindowsのエクスプローラーなどで行わないでください。

- ほかのユーザによって編集中のプロジェクトを開く場合読み取り専用で開けます。ただし、下記の機能が使用できなくなります。

- プロジェクトの上書き保存

- 機種/動作モード変更

- ネットワークドライブやリムーバブルメディアなどに保存されたプロジェクト直接開かず、パソコンのハードディスクに保存してから開いてください。

- 安全プロジェクトを開く場合Windowsのスクリーンセーバー機能により、一定の時間パソコンを操作しない状態が続いたときに自動的にロックされるようにしておいてください。

詳細は、Windowsのヘルプとサポートを参照してください。

- QCPU(Qモード)、LCPU、またはFXCPUのプロジェクトを開く場合下記の形式のプロジェクトを選択した場合、プロジェクトはQ/L/FXシリーズ互換モードで起動したGX Works3で開きます。

- GX Works2の1ファイル形式のプロジェクト

- GX Works2のワークスペース形式のプロジェクト

- GX Works3のQ/L/FXシリーズ互換モードで作成した1ファイル形式のプロジェクトなお、GX Works2で圧縮保存したプロジェクトを開く場合、事前にGX Works2で解凍しておいてください。

- ユニットFBを使用しているプロジェクトを開く場合ユニットFBを使用しているプロジェクトを開くとき、下記のメッセージが表示される場合があります。

- メッセージ: プロジェクトで使用しているユニットFBは、ご使用のGX Works3のものとバージョンが異なります。インストールされていない部品が有る場合は、GX Works3、またはご使用のライブラリをバージョンアップしてください。

このメッセージに表示されたユニットFBはGX Works3にインストールされていないため、部品選択ウィンドウの[ユニット]

タブに表示されません。

ユニットFBを更新する場合は、下記の手順に従ってください。

**操作手順**

1. 部品選択ウィンドウで、バージョンアップ後のユニットFBを選択します。

ユニットFBのバージョンは、ファンクションブロック名の最後3文字で確認できます。

(例: 「M+RCPU_Msynchronization_Delay1OUT_00A」の場合、「00A」がバージョンです。)

2. 右クリック ⇒ ショートカットメニュー[プロジェクトに追加]を選択します。

3. ユニットFBのFBインスタンスが登録されているラベルエディタを開きます。

4. FBインスタンスのデータ型を手順2で追加したユニットFBに変更します。

5. 使用しなくなったユニットFBの定義をナビゲーションウィンドウの"M_FBLIB"から削除します。

6. プログラムを変換します。

- ユニットFBを更新しない場合、そのままプロジェクトを使用できます。

- FBインスタンスが登録されているラベルエディタは、文字列検索機能でユニットFBの名称を検索すると確認できます。(→ 文字列の検索/置換)

- モーション制御ファンクションブロックを使用しているプロジェクトを開くMXコントローラのモーション制御ファンクションブロックを使用しているプロジェクトを開くとき、またはシーケンサから読み出すとき、下記のメッセージが表示される場合があります。

- メッセージ: 以下の表に示すプロジェクトで使用しているモーション制御部品のバージョンよりも新しいバージョンのモーション制御部品が利用可能です。プロジェクトで使用しているモーション制御部品をバージョンアップしますか？

モーション制御ファンクションブロックを更新せずにメッセージを閉じた場合でも、後から下記のメニューを使用してモーション制御ファンクションブロックを更新できます。

- [プロジェクト] ⇒ [ライブラリ操作] ⇒ [モーション制御部品の更新]

詳細は、下記を参照してください。

→ モーション制御部品を更新する

## 他形式プロジェクトを開く

ほかのエンジニアリングツールで作成したプロジェクトをGX Works3で開き、使用できます。

本マニュアルでは、ほかのエンジニアリングツールで作成したプロジェクトを「他形式プロジェクト」と呼び、そのプロジェクトデータを置き換えるGX Works3プロジェクトを「GX Works3形式プロジェクト」と呼びます。

他形式プロジェクトをGX Works3で開くと、CPUユニットの機種がGX Works3で使用できる機種に変更されます。

これにより、CPUユニットのシリーズが異なっていてもプロジェクトデータを引き継げます。

各他形式プロジェクトを開く方法とデータの置換えの詳細は、下記を参照してください。

| 他形式プロジェクト | 参照 | |
|---|---|---|
| GX Works2 | → GX Works2形式プロジェクトを開く | → GX Works2形式プロジェクトからの置換え |
| PX Developer | → PX Developer形式プロジェクトを開く | → PX Developer形式プロジェクトからの置換え |
| GX Developer | → GX Developer形式プロジェクトを開く | → GX Developer形式プロジェクトからの置換え |

## GX Works2形式プロジェクトを開く

GX Works2で作成したプロジェクト(GX Works2形式プロジェクト)をGX Works3で開き、使用できます。

本機能は、下記ユニットのプロジェクトにのみ対応しています。

- ユニバーサルモデルQCPU

- ユニバーサルモデル高速タイプQCPU

- ユニバーサルモデルプロセスCPU

- 二重化CPU

- LCPU

- FXCPU(FX3S、FX3U、FX3UC、FX3G、FX3GC)

変更される機種を示します。

| 変更前 | 変更後*1 |
|---|---|
| ユニバーサルモデルQCPU | R120CPU |
| ユニバーサルモデル高速タイプQCPU | R120CPU |
| ユニバーサルモデルプロセスCPU | R120PCPU(プロセスモード) |
| 二重化CPU | R120PCPU(二重化モード) |
| LCPU | • Version 1.087R以前のGX Works3で開く場合: L16HCPU • Version 1.090U以降のGX Works3で開く場合: L32HCPU |
| FXCPU(FX3S、FX3G、FX3GC、FX3U、FX3UC) | FX5SCPU*2、FX5UCPU、FX5UCCPU*3、FX5UJCPU |

*1FXCPUのプロジェクトを開く場合、"機種/変換方式の選択"画面で変更後の機種を設定できます。*2FX3UおよびFX3UCからFX5SCPUには変更できません。*3FX5UCCPUに変更したい場合は、"機種/変換方式の選択"画面で"機種"を"FX5U"に設定してください。プロジェクトが開いた後、FX5UCCPUのユニット形名を設定することで、FX5UCCPUのプロジェクトとして使用できます。(→ MX-FモデルおよびFX5CPUの詳細な形名を指定する場合) GX Works2形式プロジェクトに設定したCPUユニットの機種によって、対応するGX Works3のバージョンが異なります。

詳細は、下記を参照してください。

→ 以前のバージョンからの追加/変更点

GX Works2形式プロジェクトをGX Works3形式プロジェクトとして使用するために必要なデータの置換えの詳細は、下記を参照してください。

→ GX Works2形式プロジェクトからの置換えなお、本機能を使用するには、最新のGX Works2がインストールされている必要があります。

**操作手順**

1. [プロジェクト] ⇒ [他形式ファイルを開く] ⇒ [GX Works2形式] ⇒ [プロジェクトを開く]を選択します。

2. プロジェクトを選択し、[開く]ボタンをクリックします。

3. 表示されるメッセージを確認し、[OK]ボタンをクリックします。*1

4. 表示されるメッセージを確認し、CPUユニットのサンプルコメントを読み出すかどうかを選択します。

プロジェクトデータの変更点が"出力"ウィンドウに表示されます。

*1Version 1.040S以降のGX Works3でFX5CPUのプロジェクトを開く場合は、操作手順が異なります。詳細は、下記を参照してください。→ MELSEC FX3G/FX3U/FX3UCシリーズからMELSEC iQ-Fシリーズへの置換えの手引き注意事項下記のいずれかの条件に該当するプロジェクトはGX Works3で開けません。事前にGX Works2で確認および処置してください。

- コンパイル時にエラーが発生するプログラムが存在する: コンパイルできることを確認してください。

- ユーザ管理およびアクセス権限が設定されている: 設定を解除してください。

- 圧縮されている(*.gwz): 解凍してください。

また、GX Works3の変換時のチェック機能は、GX Works2と比較して強化されています。そのため、GX Works2では変換エラーが発生しないプログラムをGX Works3で変換すると、エラーが発生する場合があります。その場合は、エラーの内容を確認し、修正してください。

## 保存

パソコンのハードディスクなどへプロジェクトを保存します。

"プロジェクト保存時のMELSOFT iQ AppPortal情報ファイル出力"の設定により、プロジェクトの保存時にiQ AppPortal用の情報ファイルを出力できます。

詳細は、下記を参照してください。

→ MELSOFT iQ AppPortal用設定

### プロジェクトに名前を付けて保存

画面表示

[プロジェクト] ⇒ [名前を付けて保存]を選択します。

![図(元マニュアル p.132)](images/gxworks3/03-02-08.webp)

ワークスペース形式でプロジェクトを保存する場合は、[ワークスペース形式プロジェクトを保存する]ボタンで画面を切り替えてください。

**操作手順**

各項目を設定し、[保存]ボタンをクリックします。

Q/L/FXシリーズ互換モードで起動したGX Works3でプロジェクトを保存する場合、"プロジェクトの名前を付けて保存"画面の"ファイルの種類"で下記のいずれかを選択できます。

- GX Works3プロジェクト(*.gx3)

- GX Works2プロジェクト(*.gxw)

## 削除

パソコンのハードディスクなどに保存されているプロジェクトを削除します。

画面表示

[プロジェクト] ⇒ [削除]を選択します。

![図(元マニュアル p.133)](images/gxworks3/03-02-09.webp)

**操作手順**

削除するプロジェクト(：)を選択し、[削除]ボタンをクリックします。

注意事項

ワークスペース形式のプロジェクトを削除後にプロジェクトファイルが存在しなくなった場合、ワークスペース自体を削除するかを確認します。削除を実行した場合、ワークスペースフォルダーは削除されますが、"workspacelist.xml"は削除されません。

また、削除を実行した場合、指定したプロジェクトに対応するiQ AppPortal用の情報ファイルも削除されます。
