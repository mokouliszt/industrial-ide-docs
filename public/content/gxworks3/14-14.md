# 14.14 タグデータを確認する

## タグデータを計器ウィンドウ(フェースプレート)で確認する

フェースプレートとは、タグFBのタグデータの内容を調節計などの形式を模して表示する画面です。

タグデータにアクセスすることで、該当するプロセスの状態監視や条件設定を行います。

フェースプレートでは、表示だけではなく、各タグデータ項目の値を操作することも可能です。

フェースプレートは、同時に4つまで表示できます。

タグFBの詳細は、下記を参照してください。

→ MELSEC iQ-R プログラミングマニュアル(プロセス制御FB/命令編)

画面表示1.プロセス制御用FBD/LDプログラムをモニタ開始します。

2. FBD/LDエディタ上でタグFBを選択します。

3. [オンライン] ⇒ [モニタ] ⇒ [フェースプレート]を選択します。

![図(元マニュアル p.806)](images/gxworks3/14-14-01.webp)

フェースプレートは呼び出し元プログラムがモニタ中の場合のみ表示できます。そのため、通信異常などによりモニタが停止した場合は、フェースプレートが閉じられます。

| (2)タグ名称表示エリア | タグ名称が表示されます。 |
| (3)タグコメント表示エリア | タグFB設定エディタで定義されたコメントが表示されます。 |
| (4)アラーム表示エリア | タグアラームが表示されます。 |

| 項目 | 内容 |
|---|---|
| (5)PV値バー表示エリア | 現在のPV値がバー形式で表示されます。PV工学値上限からPV工学値下限の範囲で、グラフが表示されます。PV上上限警報値、PV上限警報値、PV下限警報値、PV下下限警報値が設定されている場合、警報バーが表示されます。 |
| (6)SV値バー表示エリア | SV上限値からSV下限値までが緑色のバーで表示され、現在のSV値が黄色の指針で表示されます。 |

| (13)モニタ状態表示エリア | 現在のモニタ状態が表示されます。 |

(1)

注意事項

Version 1.101F以前のGX Works3でフェースプレートを使用する場合、最新のPX Developerがインストールされている必要があります。

Version 1.105K以降のGX Works3でフェースプレートを使用する場合、PX Developerのインストールは不要です。

フェースプレートを使用する場合、タグFB設定のタグ名およびタグコメントの入力文字は、半角英数字と日本語文字のみ使用してください。

### フェースプレートで表示される部品の詳細

フェースプレート上には、表示したいタグデータの内容に合わせて、表示灯や押しボタンを模した部品が表示されます。

表示される部品はPX Developerのフェースプレートに表示される部品と同じです。詳細は、下記を参照してください。

→ PX Developer Version 1 オペレーティングマニュアル(モニタツール編)

上記マニュアルは下記の方法で閲覧できます。

- PX Developerモニタツールを起動後、ヘルプメニュー ⇒ [オペレーティングマニュアル] ⇒ [モニタツール編]を選択

### タグFBのモニタ値が不正な場合

タグFBのモニタ値が不正な場合、フェースプレートの表示エリアが黒で塗りつぶされ、「FUNC Mismatch」の文字列が表示されます。

一度フェースプレートを閉じ、下記の処置を行った後、再度フェースプレートを表示してください。

| モニタ値が不正となるケース | 処置 |
|---|---|
| プロジェクトをシーケンサに書き込んでいない場合 | 全変換(再割付)を行い、シーケンサへ書き込みます。 |
| ファイルレジスタ設定を行っていない場合 | "CPUパラメータ"の"ファイルレジスタ設定"で下記の設定を行い、シーケンサへ書き込みます。• 使用有無設定: 全プログラムで共通のファイルレジスタを使用する• 容量: 任意(必要なサイズを指定)• ファイル名: 任意(使用するデバイスメモリ名を指定する) |
| ファイルレジスタがラッチ(2)に設定されていない場合 | "CPUパラメータ"の"デバイス/ラベルメモリエリア詳細設定"で、ラッチ(2)にZR(R)を設定し、シーケンサへ書き込みます。(→ CPUパラメータの設定) |
| CPUパラメータとグローバルラベル設定(ファイルレジスタ)*1を、同時にシーケンサへ書き込んだ場合 | CPUパラメータを書き込み、CPUユニットの電源をOFF→ONまたはリセットします。その後、グローバルラベル設定(ファイルレジスタ)*1をシーケンサへ書き込みます。 |

*1：グローバルラベル設定をCPUユニットへ書き込む場合、下記のオプションで設定した範囲のファイルレジスタのデータが自動的に書き込まれます。[ツール] ⇒ [オプション] ⇒ "変換" ⇒ "プロセス制御拡張設定" ⇒ "システムリソース" ⇒ "ファイルレジスタ: ZR"

## PX Developerモニタツール連携

GX Works3で作成したプロセス制御用FBD/LDプログラムを、PX Developerモニタツールでモニタ(調整、監視、および制御)

できます。

また、PX DeveloperモニタツールのGOT画面生成機能を使用して、GOT画面プロジェクトを生成できます。

本機能を使用する場合、下記の操作手順に従ってください。

| No. | 操作手順 | 参照 |
|---|---|---|
| 1 | GX Works3で割付情報データベースファイルをエクスポートします。 | → 割付情報データベースファイルをエクスポートする |
| 2 | PX Developerモニタツールで割付情報データベースファイルを設定します。 | → PX Developer Version 1 オペレーティングマニュア |
| 3 | PX Developerモニタツールでプロセス制御用FBD/LDプログラムをモニタします。 | ル(モニタツール編) |
| | PX DeveloperモニタツールでGOT画面プロジェクトを生成します。 | → PX Developer Version 1 オペレーティングマニュアル(GOT画面生成機能編) |

### 割付情報データベースファイルをエクスポートする

GX Works3プロジェクトの割付情報データベースファイルをエクスポートする操作手順を示します。

**操作手順**

1.プロセス制御用FBD/LDプログラムを使用しているGX Works3プロジェクトを開きます。

2.ツールバーの：(割付情報データベースファイルをエクスポート)をクリックします。

3."割付情報データベースファイルをエクスポート"画面で[保存]ボタンをクリックします。

4.表示されたメッセージを確認し、PX DeveloperモニタツールでGOT画面生成機能を使用するかどうかを選択します。

割付情報データベースファイル(*.FADB)がエクスポートされます。

下記のオプションを"はい"に設定すると、割付情報データベースファイルをエクスポートするときに、タグ名の末尾の"_"(アンダースコア)が削除されます。

- [ツール] ⇒ [オプション] ⇒ "変換" ⇒ "プロセス制御拡張設定" ⇒ "タグFB" ⇒ "PX Developerモニタツール連携" ⇒ "

タグ名の末尾のアンダースコアを取り除く"

注意事項

- 割付情報データベースファイルをエクスポートする場合、タグFB設定のタグ名およびタグコメントの入力文字には、半角英数字と日本語文字のみ使用してください。

- PX DeveloperモニタツールのGOT画面生成機能を使用する場合、Version 1.045X以降のGX Works3で割付情報データベースファイルをエクスポートしてください。

- Version 1.050C以降のGX Works3でエクスポートした割付情報データベースファイルをVersion 1.50C以前のPX Developer

モニタツールで使用する場合、割付情報データベースファイルをエクスポートする前に、下記のオプションの値を480以下に設定してください。その後、タグFB設定エディタの[適用]ボタンをクリックしてください。

- [ツール] ⇒ [オプション] ⇒ "変換" ⇒ "プロセス制御拡張設定" ⇒ "タグFB" ⇒ "最大タグ数"
