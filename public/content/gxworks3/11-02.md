# 11.2 シミュレーションの実行

シミュレーションを実行します。

## CPUユニットのシミュレーション

1つのCPUユニットに対してシミュレーションを行う場合に使用します。

### シミュレーションの開始

画面表示

[デバッグ] ⇒ [シミュレーション] ⇒ [シミュレーション開始]を選択します。

シミュレーションの開始時、GX Simulator3が起動します。

シミュレーション中は接続先CPUがGX Simulator3になり、ステータスバーに"シミュレーション((システム番号).(号機番号))"

が表示されます。

![図(元マニュアル p.643)](images/gxworks3/11-02-01.webp)

**操作手順**

表示される"オンラインデータ操作"画面で書き込むデータにチェックを入れ、[実行]ボタンをクリックします。

下記のオプションを"はい"に設定すると、シミュレーション開始時にパラメータおよびプログラムがGX

Simulator3に自動で書き込まれます。このとき、"オンラインデータ操作"画面は表示されません。

- [ツール] ⇒ [オプション] ⇒ "シミュレーション" ⇒ "シミュレーション開始" ⇒ "動作設定" ⇒ "シミュレーション開始時にプログラムとパラメータを自動で書き込む"

また、"お気に入り設定"画面で任意のデータの"お気に入り選択ボタン"列にチェックを入れている場合、そのデータも自動で書き込まれます。(→ お気に入り設定画面)

- MXコントローラのシミュレーション • GX Simulator3を1台起動するごとにシステムドライブに5.0GB以上の空き容量が必要です。

- 下記の状態で動作します。

| モデル | 動作 |
|---|---|
| MX-Rモデル | GX Simulator3の起動時、プログラムが自動で変換されます。*1 |
| MX-Rモデル | シミュレーション中、下記の機能を使用できません。 • プロジェクトの照合 • パフォーマンスモニタ • アドオン管理 |
| MX-Fモデル | GX Simulator3の起動時、プログラムが自動で変換されます。 |
| MX-Fモデル | シミュレーション中、下記の機能を使用できません。 • プロジェクトの照合 • パフォーマンスモニタ • アドオン管理 |
| MX-Fモデル | "ユニット診断"画面などの画面には、システムパラメータに設定されたユニット形名が表示されます。 |
| MX-Fモデル | 実際のMX-FモデルのLED表示と異なるGX Simulator3のLED表示を示します。下記以外のLED表示は同じです。 • PWR LED: 表示されません。 • RDY LED: "READY"と表示されます。 |

*1未変換状態のデータが含まれる場合は自動で変換されません。

- RnPCPU(二重化モード)のシミュレーション下記の状態で動作します。

- 運転モード: セパレートモード

- 制御系/待機系: 制御系

- A/B系設定: A系

- トラッキング転送: 実行されません。

- 二重化システム用増設ベースユニットの増設ケーブル: 一重化構成また、シミュレーション開始時、下記のメッセージが表示されます。

- メッセージ: 接続しているシーケンサは、プロセスモードで動作しており、プロジェクトの動作モードと異なります。書込み完了後、シーケンサをリセットすることで二重化モードに変更します。実行してもよろしいですか?

- RnPSFCPUのシミュレーション下記の状態で動作します。

- 運転モード: バックアップモード

- 制御系/待機系: 制御系

- A/B系設定: A系

- トラッキング転送: 実行されません。

注意事項

RnPCPU(二重化モード)またはRnPSFCPUをシミュレーションする場合の注意事項を示します。

- シミュレーション実行中は、下記の機能を使用できません。

[オンライン] ⇒ [二重化シーケンサ操作]

- システムパラメータのベース設定でベースユニットの形名とスロット数を設定した状態でシミュレーションしてください。ベースユニットの形名が未設定のままシミュレーションすると、二重化システム構成異常エラーが発生します。

### シミュレーションの終了

**操作手順**

[デバッグ] ⇒ [シミュレーション] ⇒ [シミュレーション停止]を選択します。
