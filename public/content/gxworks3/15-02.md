# 15.2 プログラムの不正な閲覧の防止(セキュリティキー)

プログラム(プログラムファイル単位)の不正な閲覧をキーで防止するには、セキュリティキー認証機能を使用します。

MXコントローラでは対応していません。

## プロジェクトのセキュリティキー認証機能について

セキュリティキーを使用することで操作を制限できます。

セキュリティキー設定対象

| 対象 | 用途 |
|---|---|
| プログラムファイル(プログラムファイル、FBファイル、FUNファイル) → データ構成) セキュリティキーはプログラムファイルごとに登録できますが、プロジェク ト内のプログラムファイルはすべて同じセキュリティキーです。 | • プロジェクト内のプログラムファイルの閲覧を制限する。 • CPUユニットにてプログラムの実行を制限する。 |
| パソコン つのパソコンに対して複数のセキュリティキーを登録できます。 | セキュリティキーによって保護されているプログラムファイルの閲覧を可能 にする。 |

制限できる操作: 下記の操作を制限できます。

プログラム(プログラムファイル、FBファイル、FUNファイル)の閲覧プログラムファイルとパソコンのセキュリティキーの一致/不一致により操作を制御します。

![図(元マニュアル p.826)](images/gxworks3/15-02-01.webp)

## セキュリティキーの複製

: パソコンに登録されているセキュリティキーは、エクスポート/インポートすることでほかのパソコンでも同じセキュリティキーを登録できます。セキュリティキーを複製する場合、有効期限を設定できます。

![図(元マニュアル p.826)](images/gxworks3/15-02-02.webp)

設定方法: → セキュリティキーの複製注意事項

GX Works3のアンインストールを行っても、パソコンに設定されているセキュリティキーは削除されません。削除する場合は、"セキュリティキー管理"画面から行ってください。

## プロジェクトのセキュリティキー認証機能の使用手順

セキュリティを有効にする手順1.セキュリティキーを作成します。(→ セキュリティキーの作成/削除)

2. プログラムファイルへセキュリティキーを登録します。(→ プログラムファイルへのセキュリティキーの登録)

3. プロジェクトを保存します。(→ 保存)

プロジェクトに登録したセキュリティキーと同じキーを持たないパソコンからは閲覧できなくなります。

セキュリティでロックされているプログラムファイルを閲覧可能にする手順1.保護されているプログラムファイルと同じセキュリティキーを複製します。(→ エクスポート)

2. プログラムファイルを閲覧するパソコンにセキュリティキーを登録します。(→ インポート)

3. プログラムファイルを開きます。(→ 開く)

プログラムファイルとパソコンのセキュリティキーが一致している場合、閲覧できます。

プログラムファイルに対してセキュリティキーの登録を行ったパソコンでは、上記の操作は不要です。ただし、パソコンのセキュリティキーを削除した場合は、上記の操作をしてください。

## セキュリティキーの作成/削除

は、"セキュリティキー管理"画面で行います。

画面表示

[プロジェクト] ⇒ [セキュリティ] ⇒ [セキュリティキー管理]を選択します。

![図(元マニュアル p.828)](images/gxworks3/15-02-03.webp)

表示内容

| 項目 | 内容 |
|---|---|
| 生成日 | セキュリティキーを登録した日時が表示されます。 |
| 有効期限 | エクスポート時に設定したセキュリティキーの有効期限が表示されます。 |
| エクスポート | エクスポート時に設定した、再エクスポートの許可/禁止が表示されます。 |
| 保護対象の選択 | エクスポート時に設定した、保護するプロジェクトデータの選択の許可/禁止が表示されます。 |
| CPUへのキー登録 | エクスポート時に設定した、CPUユニットへの書込みの許可/禁止が表示されます。 |

### セキュアモード

セキュリティキーのセキュアモードを切り替えます。

セキュアモードを有効化すると、脆弱性に対するリスクを低減できます。

**操作手順**

1. セキュリティキーがパソコンに登録されていないかを事前に確認してください。

セキュリティキーがパソコンに登録されている場合は、セキュリティキーを削除してください。(→ 削除)

2. "セキュリティキー管理"画面の[セキュアモード切替え]ボタンをクリックします。

3. 有効/無効を切り替えます。

4. [OK]ボタンをクリックします。

### 新規作成

セキュリティキーを新規作成し、パソコンに登録します。

セキュリティキーの登録数: パソコンのログオンユーザごとに作成されます。最大登録数は、新規作成したセキュリティキーとインポートしたセキュリティキーを合わせて、パソコンのログオンユーザごとに128 個です。

**操作手順**

1. [新規作成]ボタンをクリックします。

2. "セキュリティキーの新規作成"画面でセキュリティキーの名称を入力し、[OK]ボタンをクリックします。

注意事項

CPUユニットおよびプロジェクトをロックしているセキュリティキーを誤って削除または紛失した場合、すべての情報へのアクセスができなくなります。

十分な有効期限を設定したセキュリティキーファイルをエクスポートして厳重に管理してください。

### 削除

パソコンに登録されているセキュリティキーを削除します。

プロジェクトをロックしたセキュリティキーを削除すると、プロジェクト内のロックしたデータが閲覧/編集できなくなります。

セキュリティキーを削除する前に、セキュリティキーを解除することを推奨します。

"セキュリティキー設定"画面でセキュリティキーを削除することで、セキュリティキーを解除できます。

詳細は、下記を参照してください。

- プログラムファイルに登録したセキュリティキーを解除する場合: → 削除

- CPUユニットに書き込まれているセキュリティキーを解除する場合: → 削除また、削除後に同名のセキュリティキーを新規作成しても、同じセキュリティキーにはなりません。セキュリティキーの削除は、十分注意して実行してください。

**操作手順**

1. "セキュリティキー管理"画面で削除対象のセキュリティキーを選択します。

2. [削除]ボタンをクリックします。

## セキュリティキーの複製

(インポート/エクスポート)は"セキュリティキー管理"画面で行います。

### エクスポート

パソコンに登録したセキュリティキーをインポート可能なファイル(*.ity、*.ity2)にエクスポートします。

エクスポートするセキュリティキーには、有効期限および操作の制限を付加できます。

**操作手順**

1. "セキュリティキー管理"画面でエクスポート対象のセキュリティキーを選択します。

2. [エクスポート]ボタンをクリックします。

3. "エクスポート"画面で、各項目を設定します。

| 項目 | 内容 |
|---|---|
| 制限 | エクスポートするセキュリティキーファイルに制限を付加するかどうかを設定します。 |
| 有効期限 | エクスポートするセキュリティキーの有効期限を設定します。 |
| エクスポート | エクスポートしたセキュリティキーをインポート後、さらにエクスポートすることを許可するかどうかを設定します。 |
| 保護対象の選択 | エクスポートしたセキュリティキーでプログラムファイルに対してセキュリティキーの登録を許可するかどうかを設定します。 |
| CPUへのキー登録 | エクスポートしたセキュリティキーでCPUユニットに対してセキュリティキーの書込みを許可するかどうかを設定します。 |

4. インポート時に要求するパスワードを設定し、[エクスポート]ボタンをクリックします。

注意事項

- エクスポートしたセキュリティキーファイルは、厳重に管理してください。

- GX Works3で作成したセキュリティキーファイルとGX Works2で作成したセキュリティキーファイルは、互換性がありません。

エクスポートされるセキュリティキーファイルの拡張子は、セキュアモードの有効/無効により、下記のように異なります。

- セキュアモードが有効の場合: *.ity2

- セキュアモードが無効の場合: *.ity

### インポート

エクスポートしたセキュリティキーファイルをパソコンにインポートして、セキュリティキーを登録します。

**操作手順**

1. "セキュリティキー管理"画面の[インポート]ボタンをクリックします。

2. "セキュリティキーのインポート"画面でファイル(*.ity、*.ity2)を選択し、[開く]ボタンをクリックします。

3. "パスワード認証"画面でエクスポート時に設定したパスワードを入力し、[OK]ボタンをクリックします。

- 有効期限 • パソコンに登録されているセキュリティキーの有効期限が切れた場合、プログラムの閲覧、セキュリティキーの再エクスポート、プログラムファイルへのセキュリティキーの登録、CPUユニットへのセキュリティキーの書込みができなくなります。

- 有効期限が切れたあと引き続き同じセキュリティキーを使用する場合は、はじめにエクスポートしたパソコンから再度セキュリティキーをエクスポートし、パソコンにインポートしてください。

注意事項

拡張子が"*.ity2"のセキュリティキーファイルは、セキュアモードが有効な状態でのみインポートできます。

## プログラムファイルへのセキュリティキーの登録

セキュリティキーは、あらかじめ"セキュリティキー管理"画面で作成しておいてください。

セキュリティキーはプログラムファイルごとに登録できますが、プロジェクト内のプログラムファイルはすべて同じセキュリティキーです。

画面表示

[プロジェクト] ⇒ [セキュリティ] ⇒ [セキュリティキー設定]を選択します。

![図(元マニュアル p.831)](images/gxworks3/15-02-04.webp)

### 登録

**操作手順**

1. "名称"からプログラムファイルに登録するセキュリティキーを選択します。

2. [プロジェクト]タブから閲覧を禁止するプログラムファイルにチェックを入れ、[登録]ボタンをクリックします。

プログラムファイルに登録したセキュリティキーは、プロジェクトを保存すると有効になります。

セキュリティキーが登録されている場合、ナビゲーションウィンドウでは下記のように表示されます。

パソコンのセキュリティキーとプロジェクトのセキュリティキーが一致する場合パソコンのセキュリティキーとプロジェクトのセキュリティキーが一致しない場合

- FBファイルのセキュリティキーロックセキュリティキーによってロックされたFBファイルのファンクションブロックを使用しているプログラムを、該当するセキュリティキーが登録されていないパソコンで閲覧した場合、FBプログラムの閲覧はできません。

ただし、ロックされたFBのインスタンスの生成は、セキュリティキーが登録されていないパソコンでも可能です。

### 削除

**操作手順**

[プロジェクト]タブから、[削除]ボタンをクリックします。

### 注意事項

プロジェクトのセキュリティバージョンが"2"の場合、[プロジェクト]タブで[登録]/[削除]ボタンをクリックすると、プロジェクトは未変換状態になります。
