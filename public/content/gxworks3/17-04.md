# 17.4 収集したデータをプログラムエディタ上で確認する

シーケンサから収集したデータをGX Works3上で表示するには、オフラインモニタ機能を使用します。

現場から離れた場所でもトラブル発生時の制御データをモニタ画面で確認でき、異常が発生した場合などの原因を調査できます。

データを収集する方法は、下記を参照してください。

→ メモリダンプ機能

→ データロギング機能

→ レコーディング機能

R00CPU、RnPSFCPU、およびリモートヘッドユニットでは対応していません。

## メモリダンプ結果を確認する

メモリダンプ結果ファイルに保存されているスナップショットのデータをGX Works3のモニタ画面に反映できます。

モニタ対象のメモリダンプ結果ファイルを切り替えると、連動してGX Works3のモニタ値が変わります。

MXコントローラ、LHCPUでは対応していません。

### モニタ対象の選択

画面表示

[レコーディング] ⇒ [オフラインモニタ開始] ⇒ [メモリダンプ]を選択します。

![図(元マニュアル p.915)](images/gxworks3/17-04-01.webp)

**操作手順**

1.モニタするプログラムエディタを開きます。

2."オフラインモニタ(メモリダンプ)"画面の[追加]ボタンをクリックします。

3."ファイルを開く"画面でメモリダンプ結果ファイルを選択し、[開く]ボタンをクリックします。(複数選択可)

### オフラインモニタの開始

**操作手順**

1."オフラインモニタ(メモリダンプ)"画面の一覧からモニタするファイルを選択し、[モニタ開始]を選択します。

2.オフラインモニタ実行中に[前へ]/[次へ]を選択してファイルを切り替えます。

選択したファイルのデバイス値がGX Works3のモニタに反映されます。

- オフラインモニタ中のオンライン操作オフラインモニタ中は、下記のオンライン機能が使用できます。

- プログラムエディタ上でのモニタ

- デバイス/バッファメモリ一括モニタ

- ウォッチ

### オフラインモニタの停止

**操作手順**

"オフラインモニタ(メモリダンプ)"画面の[モニタ停止]を選択します。

すべてのオフラインモニタが停止されます。

### モニタする場合の注意事項

- モニタ値を正常に表示できない場合下記の条件に該当する場合、モニタ値は正常に表示されません。

| 条件 | 表示される値 |
|---|---|
| 選択したファイル内にモニタするデバイス値が存在しない | • ビットデバイス、ワードデバイスのビット指定: FALSE(0) • ワードデバイス、ダブルワードデバイス、倍精度実数型: -1 |
| 間接指定したデバイスをモニタした | • ビットデバイス、ワードデバイスのビット指定: FALSE(0) • ワードデバイス、ダブルワードデバイス、倍精度実数型: -1 |
| インデックス修飾を使用したデバイスをモニタした | インデックス修飾部分を除くデバイス値が表示されます。 (例: "D0Z0"の場合、"D0"が表示されます。) |

- SFCプログラムのブロック/ステップの活性状態SFC図エディタ上には、活性状態は表示されません。

また、SFC全ブロックの一括モニタには、すべて"未作成ブロック"として表示されます。

## ロギングデータを確認する

データロギング機能で収集したデータの値をGX Works3上でモニタできます。

本機能はCPUユニットから取得したロギングファイル(*.bin)にのみ対応しています。

また、下記の画面でのみ対応しています。

- プログラムエディタ(ラダー(Zoomを含む)、ST、FBD/LD)

- デバイス/バッファメモリ一括モニタ

- ウォッチウィンドウ

### モニタできるデバイス

オフラインモニタでモニタできるデバイスは下記を参照してください。

→ モニタできるデバイス

### オフラインモニタの開始

**操作手順**

1.モニタするプロジェクトを開きます。

2. [レコーディング] ⇒ [オフラインモニタ開始] ⇒ [ロギングファイル]を選択します。

3.ロギングファイル(*.bin)を選択し、[開く]ボタンをクリックします。

4.表示されたメッセージに従ってボタンをクリックします。

"シークバー"ウィンドウが開き、オフラインモニタが開始されます。(→ シークバーを使用する)

GX LogViewerで波形表示する場合は下記を参照してください。

→ 波形データを表示する下記のウィンドウを表示しているときにオフラインモニタを開始すると、複数のエラーメッセージが表示される場合があります。

- デバイス/バッファメモリ一括モニタ

- インテリジェント機能ユニットモニタ

- プログラム一覧モニタ

- 割込みプログラム一覧モニタ該当するエラーはオフラインモニタに影響しません。

[OK]ボタンをクリックし、エラーメッセージを閉じてください。

- 収集対象外のデバイス/ラベルの表示データロギング機能で収集対象に指定していないデバイス/ラベルのモニタ値は、下図のように表示されます。

![図(元マニュアル p.917)](images/gxworks3/17-04-02.webp)

*1モニタ値の色は、"色およびフォント"画面の"非モニタ情報"に設定した色で表示されます。(デフォルト: 灰色)

### オフラインモニタの停止

**操作手順**

[レコーディング] ⇒ [オフラインモニタ停止]を選択します。

GX Works3ですべての画面のモニタが停止され、"シークバー"ウィンドウが閉じられます。(→ シークバーを使用する)

### 注意事項

- ロギングファイルのデータ型と出力形式データロギング機能設定時、データ型と出力形式は一致させてください。

モニタするロギングファイル(*.bin)のデータ型と出力形式が異なる場合、正常にモニタできない場合があります。

データロギング機能の設定方法は下記を参照してください。

→ MELSEC MXコントローラ MX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラ MX-Fモデルユーザーズマニュアル

→ MELSEC iQ-R シーケンサCPUユニットユーザーズマニュアル

→ MELSEC iQ-R プロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

- マクロ型ファンクションブロックをモニタする場合マクロ型ファンクションブロックの入出力ラベルはモニタできません。

- 重複するデバイス1つのロギングファイル(*.bin)内に同じデバイスが複数存在する場合、CPUユニットロギング設定ツールの"データ"設定画面でより小さい"No."のデバイスの値が、GX Works3のモニタ画面に反映されます。

- モニタ値を正常に表示できない場合下記の条件に該当する場合、モニタ値が正常に表示されない場合があります。

設定を確認してください。

| 条件 | 表示されるモニタ値 |
|---|---|
| 記録されていないデバイス/ラベルをモニタした。 | 不定値が表示されます。 |
| データが出力されたときにCPUユニットに書き込まれていたプロジェ クトと異なるプロジェクトでモニタした。 | 下記の可能性があります。 • デバイス/ラベルのモニタ値がモニタ中のパラメータまたはプログラム内でのデ バイス/ラベルの動作状態と一致しない。 • デバイス/ラベルのモニタ値のデータ型が異なる。 |

- セキュリティで保護されているデータセキュリティキーまたはブロックパスワードによってロックされているプログラム部品は、表示されません。

- ネットワークドライブに保存したファイルをモニタする場合オフラインモニタ中、ネットワークの切断などによりファイルにアクセスできなくなった場合、オフラインモニタが停止することがあります。

- ファイルをリムーバブルメディアに保存した場合オフラインモニタを開始するときは、ファイルをパソコンのハードディスクに保存してから開いてください。

## レコーディングファイルを確認する

レコーディング機能で収集したデータの値をGX Works3上でモニタできます。

本機能はMX-Rモデル、RnCPU(R04CPU、R08CPU、R16CPU、R32CPU、R120CPU)、RnENCPU、およびRnSFCPUで対応しています。

また、下記の画面でのみ対応しています。

- プログラムエディタ

- デバイス/バッファメモリ一括モニタ

- ウォッチウィンドウ

- SFCブロックリストモニタ*1

- SFC全ブロック一括モニタ*1

*1RnCPU(R04CPU、R08CPU、R16CPU、R32CPU、R120CPU)およびRnENCPUで対応しています。

### モニタできるデバイス

オフラインモニタでモニタ可能なデバイスを示します。

デバイス

X、Y、M、L、B、F、SB、V、T、ST、LT、LST、C、LC、D、W、SW、SM、SD、J□¥X、J□¥Y、J□¥B、J□¥SB、J□¥W、J□¥SW、U□¥G、U3E□¥G、U3E□¥HG、Z、LZ、R、ZR、RD、BL*1、BL□¥S、#M、#V、#T、#ST、#LT、#LST、#C、#LC、#D、SA¥X、SA¥Y、SA¥M、SA¥B、SA¥T、SA¥ST、SA¥C、SA¥D、SA¥W、SA¥SM、SA¥SD

*1レコーディング機能で収集したデータの場合のみ、モニタできます。

注意事項

下記のデバイスはモニタ対象外です。

- ローカルインデックスレジスタ

- インデックス修飾および間接指定されたデバイスまた、レコーディング設定をVersion 1.065Tおよび1.066UのGX Works3で書き込み、かつプロジェクトデータを同時に読み出していないレコーディングファイルの場合、インデックスレジスタ(Z)、ロングインデックスレジスタ(LZ)はモニタできません。

### オフラインモニタの開始

**操作手順**

1. [レコーディング] ⇒ [オフラインモニタ開始] ⇒ [レコーディングファイル]を選択します。

2.レコーディングファイル(*.melrc)を選択し、[開く]ボタンをクリックします。

3.表示されたメッセージに従ってボタンをクリックします。

"シークバー"ウィンドウが開き、オフラインモニタが開始されます。(→ シークバーを使用する)

GX LogViewerで波形表示する場合は下記を参照してください。

→ 波形データを表示する下記のウィンドウを表示しているときにオフラインモニタを開始すると、複数のエラーメッセージが表示される場合があります。

- デバイス/バッファメモリ一括モニタ

- インテリジェント機能ユニットモニタ

- プログラム一覧モニタ

- 割込みプログラム一覧モニタ該当するエラーはオフラインモニタに影響しません。

[OK]ボタンをクリックし、エラーメッセージを閉じてください。

レコーディングファイルに含まれる動画ファイルはGX VideoViewerで再生できます。

GX VideoViewerの詳細は、下記を参照してください。

→ GX VideoViewer Version 1 オペレーティングマニュアル

- 複数のレコーディングファイルをモニタする場合演算周期が異なる複数のレコーディングファイルを使用してオフラインモニタを実行する手順を示します。

演算周期の詳細は下記を参照してください。

→ MELSEC MXコントローラ MX-Rモデルユーザーズマニュアル

**操作手順**

1.任意のプロジェクトを開きます。

2. [レコーディング] ⇒ [オフラインモニタ開始] ⇒ [レコーディングファイル]を選択します。

3.レコーディングファイル(*.melrc)を選択し、[開く]ボタンをクリックします。

4.表示されたメッセージに従ってボタンをクリックします。

モニタするファイル数分、手順を繰り返します。

- 収集対象外のデバイス/ラベルの表示レコーディング機能で収集対象に指定していないデバイス/ラベルのモニタ値は、下図のように表示されます。

![図(元マニュアル p.920)](images/gxworks3/17-04-03.webp)

*1モニタ値の色は、"色およびフォント"画面の"非モニタ情報"に設定した色で表示されます。(デフォルト: 灰色)

注意事項

- レコーディングファイルと同時に読み出したプロジェクトデータを使用することを推奨します。

レコーディングファイルと同時にプロジェクトデータを読み出す方法は下記を参照してください。

→ レコーディングファイルの読出し/削除

- 総レコード数が5,232,645を超えるレコーディングファイルをモニタする場合、5,232,645を超えた古い時刻のレコードは表示されません。

- MELSOFT NavigatorからGX Works3を起動した場合、レコーディングファイルに保存されたプロジェクトを開けません。

GX Works3で開いているプロジェクトを使用してオフラインモニタを開始してください。

レコーディングファイルに保存されたプロジェクトを使用する場合は、MELSOFT Navigatorから起動せずに単独で起動したGX Works3でオフラインモニタを開始してください。

- 下記のレコーダユニットまたはカメラレコーダユニットでレコーディングファイルをファイルサーバに保存した場合は、レコーディング設定をCPUユニットに書き込んだときのプロジェクトを開いてから、オフラインモニタを開始してください。

- ファームウェアバージョンが03または04のレコーダユニット

- ファームウェアバージョンが01のカメラレコーダユニット

- レコーディング機能が動作中、かつレコーディングファイルの保存先がネットワークドライブのとき、ネットワークドライブに保存されたレコーディングファイルを使用してオフラインモニタを開始すると、失敗する場合があります。オフラインモニタを開始するときは、ファイルをパソコンのハードディスクに保存してから開いてください。

### 注意事項

## レコーディングファイルを確認する

場合の注意事項を示します。

ロギングデータと共通する注意事項は下記を参照してください。

→ 注意事項

- 不定値が表示された場合記録されていないデバイス/ラベルをモニタした場合、不定値が表示されることがあります。

記録されたデバイス/ラベルをモニタしているかどうかは、"レコーディング設定"画面で確認できます。

"レコーディング設定"画面の詳細は、下記を参照してください。

→ MELSEC iQ-R システムレコーダユーザーズマニュアル(応用編)

- MXコントローラのプロジェクトでモニタする場合配列の添字として定数以外を指定したラベルはモニタできません。

- 安全プロジェクトを含むレコーディングファイルをモニタする場合レコーディングファイルに含まれる安全プロジェクトは、ユーザ認証が省略されGUESTユーザとしてログオンした状態で開きます。(→ GUESTユーザ)

下記に注意してください。

- プロジェクトにGUESTユーザを追加していない場合でも、ユーザ認証は省略されGUESTユーザとして開きます。

- レコーディングファイルに含まれる安全プロジェクトをユーザ認証し直すことはできません。

- レコーディングファイルに含まれる安全プロジェクトを保存した場合、保存したプロジェクトにはユーザ認証可能なパスワードが設定されません。そのため、保存したプロジェクトにログオンできるのはGUESTユーザとしてのみです。

- SFC図エディタでモニタする場合下記モニタは使用できません。

- SFC自動スクロールモニタ

- SFC活性済みステップモニタモニタ中の注意事項は下記を参照してください。

→ SFC図モニタ

- ブロック指定付ステップリレーをモニタする場合下記の条件すべてに該当するブロック指定付ステップリレー(BL□¥S)をモニタすると、モニタ値がTRUE、ステップの活性状態が活性で表示される場合があります。

- ブロック指定付ステップリレー(BL□¥S)をSFCプログラムで使用していない。

- ブロック指定付ステップリレー(BL□¥S)を、レコーディング設定の"デバイス/ラベル一覧から指定"または"デバイス範囲を指定"で収集した。

## シークバーを使用する

オフラインモニタを開始すると、"シークバー"ウィンドウが表示されます。

シークバーとは、モニタするデータのインデックス(1スキャンごとにレコーディングファイルに記録される通し番号)を指定する機能です。

シークバーは指定したインデックス、またはそのインデックスでのモニタ値を、連携するモニタ画面、"イベント履歴(オフラインモニタ)"ウィンドウ、およびGX LogViewerに反映できます。

### ログマーカー情報ファイルの読込み

データに追加されたログマーカーは、ログマーカー情報ファイルとしてレコーディングファイルまたはロギングファイルが格納されたフォルダーに保存されます。

ログマーカー情報ファイルの拡張子は、ログマーカーを追加したツールによって異なります。

| 拡張子 | ツール | ツールのバージョン |
|---|---|---|
| *.lms | GX Works3 | Version 1.065Tおよび1.066U |
| *.lms | GX LogViewer | Version 1.106K |
| *.lmcs | GX Works3 | Version 1.070Y以降 |
| *.lmcs | GX LogViewer | Version 1.112R以降 |
| *.vms | GX VideoViewer | Version 1.006G以降 |

ログマーカー情報ファイル(*.lms、*.lmcs)が含まれるデータを選択しオフラインモニタを開始すると、ログマーカーが自動的に読み込まれ、シークバー上に反映されます。

また、下記の手順でログマーカーを読み込むこともできます。

**操作手順**

- 指定したログマーカー情報ファイルを読み込む場合1."シークバー"ウィンドウのツールバーの[ログマーカー] ⇒ [読込み]をクリックします。

2.ログマーカー情報ファイル(*.lms、*.lmcs、*.vms)を選択し、[開く]ボタンをクリックします。

- 動画ファイルのログマーカーを読み込む場合"シークバー"ウィンドウのツールバーの[ログマーカー] ⇒ [GX VideoViewerからの読込み]をクリックします。

GX VideoViewerで開いている動画ファイルのログマーカー、またはオフラインモニタ中のレコーディングファイルに含まれるログマーカー情報ファイル(*.vms)が読み込まれます。

注意事項

- Version 1.070Y以降のGX Works3およびVersion 1.112R以降のGX LogViewerで作成されたログマーカー情報ファイル

(*.lmcs)を、Version 1.065Tおよび1.066UのGX Works3で開くことはできません。

- ログマーカー情報ファイルを読み込んでも、すでに追加されているログマーカーのコメント/色は上書きされません。

### GX LogViewerとの連携

GX Works3とGX LogViewerが同時かつ同じデータをモニタしている場合、どちらか一方で追加/削除/編集されたログマーカーの状態は、もう一方にも反映されます。

GX LogViewerにおけるログマーカー機能の詳細は、下記を参照してください。

→ GX LogViewer Version 1 オペレーティングマニュアル

GX Works3およびGX LogViewerのバージョンごとに、ログマーカーを同期させる方法を示します。

| GX Works3 | GX LogViewer | 同期させる方法 |
|---|---|---|
| Version 1.065Tおよび1.066U | Version 1.106K | 反映先のツールにて、ログマーカー情報ファイル(*.lms)の読込みを実行してください。 |
| Version 1.070Y以降 | Version 1.112R以降 | 自動で同期します。(操作は必要ありません。) |

### GX VideoViewerとの連携

GX Works3でGX VideoViewerのログマーカー情報ファイル(*.vms)を読み込むことで、動画ファイルに追加したログマーカーをレコーディングファイル、またはロギングファイルに反映できます。(→ 動画ファイルのログマーカーを読み込む場合)

また、GX Works3で追加したログマーカーをGX VideoViewerに反映することもできます。

GX VideoViewerにおけるログマーカー機能の詳細は、下記を参照してください。

→ GX VideoViewer Version 1 オペレーティングマニュアル

時刻情報が含まれていないロギングファイルの場合、ログマーカー機能は使用できません。

### GOTオフラインモニタの起動

**操作手順**

1.レコーディングファイルを選択し、オフラインモニタを開始します。(→ オフラインモニタの開始) 2. [レコーディング] ⇒ [GOTオフラインモニタ] ⇒ [起動] ⇒ [GOT2000]/[GOT3000]を選択します。

3. GT Designer3のプロジェクトファイルを選択し、[開く]ボタンをクリックします。

### GOTオフラインモニタの停止

**操作手順**

[レコーディング] ⇒ [GOTオフラインモニタ] ⇒ [終了]を選択します。

## 映像を再生する

カメラレコーダユニットまたはカメラ録画パッケージを使用して生成した動画ファイルを、GX VideoViewerで再生します。

本機能を使用するには、GX VideoViewerがインストールされている必要があります。

GX VideoViewerの使用方法は、下記を参照してください。

→ GX VideoViewer Version 1 オペレーティングマニュアル

![図(元マニュアル p.936)](images/gxworks3/17-04-15.webp)

### GX VideoViewerの起動

**操作手順**

1.オフラインモニタを開始します。

(ロギングデータ: → オフラインモニタの開始、レコーディングファイル: → オフラインモニタの開始) 2. [ツール] ⇒ [GX VideoViewer]を選択します。

3.動画ファイルを選択し、[開く]ボタンをクリックします。
