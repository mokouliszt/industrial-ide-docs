# 10.1 ファンクションブロックの作成

ファンクションブロックを使用して、シーケンスプログラムを作成する方法を説明します。

## ファンクションブロックについて

**ファンクションブロックの関連項目**

→ MELSEC MXコントローラMX-Rモデルプログラミングマニュアル

→ MELSEC MXコントローラMX-Fモデルプログラミングマニュアル

→ MELSEC iQ-Rプログラミングマニュアル(プログラム設計編)

→ MELSEC iQ-F FX5プログラミングマニュアル(プログラム設計編)

### 使用手順

- 新規作成する場合1.ファンクションブロックのデータを新規作成します。(→ データの作成)

2. ファンクションブロックのプログラム本体で使用するローカルラベルを登録します。ローカルラベルはファンクションブロック内に最大5120個登録できます。(→ ラベルの設定)

3. ラベルを使用して、プログラム本体を作成します。(→ プログラムの作成)

4. ファンクションブロックをシーケンスプログラム上に貼り付けます。

(ラダープログラム: → ファンクションブロックの挿入、STプログラム: → ファンクションブロックの挿入)

- 既存の回路から作成する場合プログラムエディタ上で回路ブロックを選択し、ファンクションブロックを作成します。(→ 既存の回路から作成)

作成したファンクションブロックは自動でシーケンスプログラム上に挿入されます。

ラダープログラムの回路からのみ作成できます。

## 新規作成

ファンクションブロックを新規作成する方法を説明します。

**作成できるファンクションブロックの個数の関連項目**

→ 作成できるデータの最大個数(MXコントローラ)

→ 作成できるデータの最大個数(RCPU/LHCPU/FX5CPU)

ただし、FBファイルとFUNファイルの構成により、作成可能な上限に達しない場合があります。

### ラベルの設定

プログラム本体で使用するラベルを、ラベルエディタで定義します。

**画面上の操作方法は、ほかのラベルエディタと同じです。詳細の関連項目**

→ ラベルの登録

**操作手順**

ナビゲーションウィンドウ ⇒ "FB/FUN" ⇒ "(ファイル名)" ⇒ "(FB)" ⇒ "ローカルラベル"を選択します。

**注意事項**

ファンクションブロックのインスタンスをエイリアスに割り付けないでください。(→ エイリアス)

### プログラムの作成

ラベルを使用して、ファンクションブロックのプログラム本体を作成します。

ファンクションブロックのローカルラベルとグローバルラベルが使用できます。

プログラムの入力方法は、各プログラムと同じです。

**操作手順**

1. ナビゲーションウィンドウ ⇒ "FB/FUN" ⇒ "(ファイル名)" ⇒ "(FB)" ⇒ "プログラム本体"を選択します。

2. プログラムを入力します。

### ファンクションブロックの種類の変更

ファンクションブロックの種類(マクロ型/サブルーチン型)を変更します。

MXコントローラの場合、"サブルーチン型"から変更できません。

**操作手順**

1. ナビゲーションウィンドウ ⇒ "FB/FUN" ⇒ "(ファイル名)"を選択します。

2. [プロジェクト] ⇒ [データ操作] ⇒ [プロパティ]を選択します。

3. "プロパティ"画面の"FBの種類"で、変更するファンクションブロックの種類を設定し、[OK]ボタンをクリックします。

### 予約領域容量の設定

FBインスタンスの予約領域容量をファンクションブロックごとに設定します。

設定した予約領域容量をプログラムに反映するには、全変換(再割付)が必要です。

**予約領域容量の関連項目**

→ MELSEC MXコントローラMX-Rモデルプログラミングマニュアル

→ MELSEC MXコントローラMX-Fモデルプログラミングマニュアル

→ MELSEC iQ-Rプログラミングマニュアル(プログラム設計編)

→ MELSEC iQ-F FX5プログラミングマニュアル(プログラム設計編)

**操作手順**

1. 下記のオプションを"はい"に設定します。

[ツール] ⇒ [オプション] ⇒ "変換" ⇒ "基本設定" ⇒ "動作設定" ⇒ "ファンクションブロック" ⇒ "予約領域の設定を可能にする"2.ナビゲーションウィンドウ ⇒ "FB/FUN" ⇒ "(ファイル名)" ⇒ "(FB)"を選択します。

3. [プロジェクト] ⇒ [データ操作] ⇒ [プロパティ]を選択します。

4. "プロパティ"画面の"ラベル予約領域"、"ラッチラベル予約領域"、または"シグナルフロー予約領域"に、予約領域容量を設定し、[OK]ボタンをクリックします。

5. プログラムを全変換(再割付)します。

**注意事項**

- FX5CPUのプロジェクトでは、ファンクションブロックの予約領域容量を変更できません。

- サブルーチン型のファンクションブロックのみ、"シグナルフロー予約領域"に予約領域容量を設定できます。

### ファンクションブロックの変換について

- FBインスタンスを作成していない、またはFBインスタンスが未登録プログラム内でのみ作成されている場合、変換してもファンクションブロック(プログラム部品)は変換済みになりません。

- ファンクションブロックを編集した場合、FBファイルおよびファンクションブロックを使用しているすべてのプログラムが未変換状態になります。

- サブルーチン型ファンクションブロックのプログラム本体のみ編集する場合下記のオプションの設定により、未変換状態になる対象を、編集したFBプログラムおよびFBファイルのみにできます。

- [ツール] ⇒ [オプション] ⇒ "変換" ⇒ "基本設定" ⇒ "動作設定" ⇒ "変換、変換+RUN中書込み対象設定"

オプションを"高速"から"低速"に変更したあと、未変換状態のファンクションブロックを変換すると、ファンクションブロックを使用しているプログラム部品も変換されます。

MXコントローラの場合、"高速"から変更できません。

**注意事項**

オプションを"高速"に設定し、かつサブルーチン型ファンクションブロックで下記の命令を使用していると、変換エラーが発生する場合があります。再度"変換"または"変換+RUN中書込み"を実行すると、正常に完了できます。

- 立上り命令

- 立下り命令

- SCJ命令

- STMR命令

- 入力変数と入力変数に接続している配列型ラベルの要素数が異なる場合入力変数(ファンクションブロックの入力引数)と入力変数に接続している配列型ラベルの要素数が異なる場合、変換エラーが発生します。

下記のオプションを"いいえ"に設定すると、配列型ラベルの要素数が入力変数の要素数より少ない場合でもプログラムを変換できます。

- [ツール] ⇒ [オプション] ⇒ "変換" ⇒ "基本設定" ⇒ "動作設定" ⇒ "ファンクションブロック" ⇒ "入力引数に接続した配列ラベルの要素数をチェックする"

プログラムの実行時、入力変数の要素のうち不足している要素の値には0(FALSE)が設定されます。

MXコントローラ、RCPU、およびLHCPUでのみ対応しています。

例入力変数「parameter[0..15]」に配列型ラベル「argument[0..2]」を接続している場合

![図(元マニュアル p.606)](images/gxworks3/10-01-01.webp)

|---|---|

| 入力変数 | 配列の要素数が1024以下である。 |
| 入力変数に接続している配列型ラベル | 1次元配列のラベルである。 |
| 入力変数に接続している配列型ラベル | データ型が入力変数と一致している。 |
| 入力変数に接続している配列型ラベル | 入力変数の要素数よりも、配列型ラベルの要素数の方が少ない。 |

下記の場合は、オプションを"いいえ"に設定してもプログラムを変換できません。

- MOVE命令で入力変数に値を代入している。

- FBインスタンスのメンバに値を代入している。

- ファンクションとファンクションブロックを接続している。(FBD/LDプログラム)

- ファンクションブロック同士を接続している。(FBD/LDプログラム)

### 作成時の注意事項

- デバイスの使用FBプログラムの作成には、ラベルを使用することを推奨します。

デバイス(X10、Y10など)を使用したFBプログラムを複数箇所で使用すると、正常に動作しない場合があります。

また、OUT命令にデバイスを使用したFBプログラムを複数箇所で使用すると、2重コイルとなります。2重コイルは、SET/

RST命令で回避できます。

- マスタコントロール命令の使用FBプログラム内でマスタコントロール命令を使用する場合、MC命令とMCR命令をセットで使用してください。

- 1スキャンで複数回実行するシーケンスプログラムにファンクションブロックを使用する場合実行完了までに複数スキャンを要する命令、立上り命令/立下り命令を含むファンクションブロックを、1スキャンで複数回実行するシーケンスプログラムで使用すると、正常に動作しない可能性があります。

## ユーザ定義タグFBの作成

ユーザ定義タグFBを作成することで、メーカ定義タグFBのプロセス制御処理をカスタマイズできます。

ユーザ定義タグFBのFBプログラム内では、プロセス制御ファンクションブロックのタグアクセスFBとタグデータ参照用構造体ラベルを使用して、タグデータを参照できます。

ユーザ定義タグFBは、RnPCPU(プロセスモード/二重化モード)およびRnPSFCPUのプロジェクトのプロセス制御用FBD/LDプログラムでのみ使用できます。

使用手順1.ファンクションブロックのデータを新規作成します。(→ データの作成)

2. ユーザ定義タグFBのプログラム本体を作成します。(→ プログラムの作成)

3. タグFB設定エディタで、ユーザ定義タグFB型を指定したタグFBを登録します。(→ タグFBの登録)

4. 登録したユーザ定義タグFBのタグFBインスタンスを使用して、プロセス制御用FBD/LDプログラムを作成します。

5. FBプロパティの初期値を設定します。(→ FBプロパティの表示/設定)

6. シーケンサへプログラムを書き込みます。(→ シーケンサへの書込み)

7. モニタし、フェースプレートを表示してデバッグします。(→ タグデータを確認する)

8. FBプロパティの初期値を反映します。(→ FBプロパティ管理(オンライン)画面)

**注意事項**

ユーザ定義タグFBの定義をナビゲーションウィンドウで削除すると、プロジェクトが不正になる場合があります。

ユーザ定義タグFBを削除する場合は、文字列検索で削除するFB定義名を検索し、プロジェクト内で使用していないことを確認してから、削除してください。

## ユニットFBの活用

GX Works3には、各ユニットの処理を部品化(ファンクションブロック)した「ユニットFB」があらかじめ用意されています。

ユニットFBを使用することで、ユニット独自の処理内容をプログラミングする必要がなくなり、簡単に設定や動作が行えます。

ただし、安全プログラムでは使用できません。

### ユニットFBをプロジェクトに取り込む

ユニットFBは、下記の操作をした場合に自動的に部品選択ウィンドウの[ユニット]タブに表示されます。

- ユニット構成図でユニットを配置し、パラメータを確定する

- ナビゲーションウィンドウでユニットを新規追加する部品選択ウィンドウからユニットFBのマニュアルを表示できます。

ユニットFBを選択し、右クリック ⇒ ショートカットメニュー[ヘルプ]を選択してください。

### ユニットFBをプログラムで使用する

**操作手順**

1. 部品選択ウィンドウの[ユニット]タブからプログラム内にドラッグ&ドロップします。

2. "FBインスタンス名入力"画面のプルダウンリストから、ローカルラベルまたはグローバルラベルのどちらに登録するか選択し、FBインスタンス名を入力します。

FBインスタンスがプログラムに挿入されます。

また、ユニットFBの定義がナビゲーションウィンドウの"M_FBLIB"に追加されます。

**注意事項**

- ユニットFBを使用する場合は、事前(プロジェクト新規作成時)にユニットラベルを登録しておくことをお薦めします。

登録していない場合、ユニットFBを使用するときに時間がかかることがあります。

- ユニットFBのインスタンスをエイリアスに割り付けないでください。(→ エイリアス)

### ユニットFBの動作パラメータを設定する

**操作手順**

1. ユニットFBのFBインスタンスが登録されているラベルエディタ(ユニットFBを使用しているプログラムのローカルラベルまたはグローバルラベル)を開きます。

2. FBインスタンスを選択し、拡張表示エリア内のラベルの初期値に値を入力します。

ただし、動作パラメータが配列型の場合、ラベルエディタでは初期値を設定できません。プログラムにて設定してください。

ラベルの初期値で設定したユニットFBの動作パラメータをプログラムにて値を変更した場合、その値でユニットFBが動作します。ユニットFBのラベルの値を変更する場合は、変更前にクロスリファレンスを活用して影響範囲を確認してください。

### ユニットFBを編集する

ユニットFBのローカルラベルおよびプログラム本体は編集できません。

ただし、ユニットFBはナビゲーションウィンドウ上でコピーでき、コピーされたユニットFBは編集できます。

コピーされたユニットFBは、FB名の"+"が"_"に変更されます。

### ユニットFBの定義を追加する

部品選択ウィンドウの[ユニット]タブからナビゲーションウィンドウの"FB/FUN"にユニットFBをドラッグ&ドロップすることで、ナビゲーションウィンドウにユニットFBの定義を追加できます。
