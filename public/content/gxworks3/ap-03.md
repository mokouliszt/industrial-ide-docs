# 付3 過去バージョンの機能を使用する場合

## Version 1.063R以前品でロギングデータを確認する

GX LogViewerに表示されているロギングデータの値をGX Works3のモニタ画面に反映できます。

GX LogViewerの赤カーソルを移動させると、連動してGX Works3のモニタ値が変わります。

![図(元マニュアル p.1087)](images/gxworks3/ap-03-01.webp)

### 操作手順

ロギングデータを確認する操作手順を示します。

操作手順1. GX LogViewerで、ロギングファイルを開きます。2. GX Works3で、モニタするロギングファイルを設定します。(→ モニタ対象の選択) 3. GX Works3で、オフラインモニタを開始します。(→ オフラインモニタの開始) 4. GX Works3で、オフラインモニタを停止します。(→ オフラインモニタの停止)

### モニタ対象の選択

モニタするロギングファイルは、"オフラインモニタ(ロギング)接続先設定"画面で設定します。

RCPUおよびFX5CPUで出力したロギングファイル(*.bin)のみを設定できます。

画面表示

[デバッグ][オフラインモニタ][オフラインモニタ(ロギング)][接続先設定]を選択します。

![図(元マニュアル p.1088)](images/gxworks3/ap-03-02.webp)

操作手順

ロギングファイルを選択し、[OK]ボタンをクリックします。

ステータスバーに"オフラインモニタ"が表示されます。

注意事項

- GX LogViewerでロギングファイルを開く前にGX Works3で"オフラインモニタ(ロギング)接続先設定"画面を開くと、ロギングファイルが一覧に表示されません。GX LogViewerでロギングファイルを開いた後、"オフラインモニタ(ロギング)接続先設定"画面の[更新]ボタンをクリックしてください。

- GX LogViewerで下記の機能を実行している場合、ロギングファイルが一覧に表示されません。

[グラフ表示][グラフ描画対象のデータを変更]

[グラフ操作][前の(古い)グラフを見る]または[次の(新しい)グラフを見る]

### オフラインモニタの開始

操作手順1. GX Works3でモニタを開始します。(→ 各種モニタの開始/停止について) 2. GX LogViewerの"ヒストリカルトレンド"ウィンドウ上で赤カーソルを移動します。

更新されたグラフ凡例エリアのデバイス値がGX Works3のモニタ画面に反映されます。

オフラインモニタ中のオンライン操作オフラインモニタ中は、下記のオンライン機能が使用できます。

- プログラムエディタ上でのモニタ

- デバイス/バッファメモリ一括モニタ

- ウォッチ

### オフラインモニタの停止

操作手順

[デバッグ][オフラインモニタ][オフラインモニタ(ロギング)][接続先解除]を選択します。

GX Works3ですべての画面のモニタが停止されます。

オフラインモニタの一時停止と再開GX Works3でモニタを停止すると、停止した画面のみオフラインモニタが一時停止します。

GX Works3でモニタを再開すると、オフラインモニタも再開されます。

### 注意事項

重複するデバイス1つのロギングファイル内に複数の同じデバイスが存在する場合、GX LogViewerのグラフ凡例エリアの最も下に表示されているデバイスの値がGX Works3のモニタ画面に反映されます。

SFCプログラムのブロック/ステップの活性状態SFC図エディタ上には、活性状態は表示されません。

また、SFC全ブロックの一括モニタには、すべて"未作成ブロック"として表示されます。

モニタ値を正常に表示できない条件下記のいずれかの条件に該当する場合、モニタ値が正常に表示されません。

| 条件 | 値 |
|---|---|
| "ヒストリカルトレンド"ウィンドウで開いているファイル内に、モニタす るデバイス値が存在しない | 下記の値が表示されます。 • ビットデバイス、ワードデバイスのビット指定: FALSE(0) • ワードデバイス、ダブルワードデバイス、倍精度実数型: -1 |
| 下位ビットまたは上位ビットのどちらか一方のみ存在するデバイスを、 ダブルワード形式で指定し、モニタした | |
| 間接指定したデバイスをモニタした | |
| インデックス修飾を使用したデバイスをモニタした | インデックス修飾部分を除くデバイス値が表示されます。 (例: "D0Z0"の場合、"D0"が表示されます。) |
| ロギングファイルが出力されたときにCPUユニットに書き込まれていた プロジェクトとモニタするプロジェクトが一致しない | ロギングファイル内とモニタするプロジェクト内のラベルの割付状態が一致し ない場合、GX LogViewerの"ヒストリカルトレンド"ウィンドウに表示されてい るラベルの値と異なる値が表示されることがあります。 |

ラベルをモニタできる条件下記のすべての条件を満たす場合にのみ、ロギングファイル内のラベルの値をモニタできます。

- Version 1.045X以降のGX Works3を使用している

- Version 1.82L以降のGX LogViewerを使用している

- ロギングファイルが出力されたときにCPUユニットに書き込まれていたプロジェクトのモニタを実行している

- ロギングファイル内とモニタするプロジェクト内のラベルの割付状態が一致している
