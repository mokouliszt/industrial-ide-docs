# 10.2 ファンクションの作成

ファンクションを使用して、シーケンスプログラムを作成する方法を説明します。

## ファンクションについて

**ファンクションの関連項目**

→ MELSEC MXコントローラMX-Rモデルプログラミングマニュアル

→ MELSEC MXコントローラMX-Fモデルプログラミングマニュアル

→ MELSEC iQ-Rプログラミングマニュアル(プログラム設計編)

→ MELSEC iQ-F FX5プログラミングマニュアル(プログラム設計編)

使用手順1.ファンクションのデータを新規作成します。(→ データの作成) 2. FUNプログラムで使用するローカルラベルを登録します。ローカルラベルはファンクション内に最大5120個登録できます。(→ ラベルの設定)

3. ラベルを使用して、FUNプログラムを作成します。(→ プログラムの作成)

4. ファンクションをシーケンスプログラム上に貼り付けます。

(ラダープログラム: → ファンクションの挿入、STプログラム: → ファンクションの挿入)

## 作成

ファンクションの作成方法を説明します。

**作成できるファンクションの個数の関連項目**

→ 作成できるデータの最大個数(MXコントローラ)

→ 作成できるデータの最大個数(RCPU/LHCPU/FX5CPU)

ただし、FBファイルとFUNファイルの構成により、作成可能な上限に達しない場合があります。

### データの作成

プロジェクト内に、"データ新規作成"画面からファンクションのデータを作成します。

| 項目 | | | 内容 |
|---|---|---|---|
| 詳細設定 | EN/ENOを使用する | はい | EN/ENOを持つファンクションとなります。 EN/ENOラベルは、ローカルラベルに登録しなくてもプログラムで使用可能となります。 EN/ENOの詳細は、下記を参照してください。 → MELSEC MXコントローラMX-Rモデルプログラミングマニュアル → MELSEC MXコントローラMX-Fモデルプログラミングマニュアル → MELSEC iQ-Rプログラミングマニュアル(プログラム設計編) → MELSEC iQ-F FX5プログラミングマニュアル(プログラム設計編) |
| 詳細設定 | EN/ENOを使用する | いいえ | EN/ENOを持たないファンクションとなります。 |
| 詳細設定 | 追加先のFUNファイル | いいえ | 作成するファンクションの格納先ファイルを選択します。 ファイル名を直接入力して、新規作成も可能です。 |

### ラベルの設定

プログラム本体で使用するラベルを、ラベルエディタで定義します。

**画面上の操作方法は、ほかのラベルエディタと同じです。詳細の関連項目**

→ ラベルの登録

**操作手順**

ナビゲーションウィンドウ ⇒ "FB/FUN" ⇒ "(ファイル名)" ⇒ "(ファンクション)" ⇒ "ローカルラベル"を選択します。

### プログラムの作成

ラベルを使用して、ファンクションのプログラム本体を作成します。

ファンクションのローカルラベルが使用できます。

プログラムの入力方法は、各プログラムと同じです。

**操作手順**

1. ナビゲーションウィンドウ ⇒ "FB/FUN" ⇒ "(ファイル名)" ⇒ "(ファンクション)" ⇒ "プログラム本体"を選択します。

2. プログラムを入力します。

### ファンクションの変換について

- ファンクションがプログラム内で使用されていない、またはファンクションが未登録プログラム内でのみ使用されている場合、変換してもファンクション(プログラム部品)は変換済みになりません。

- ファンクションを編集した場合、FUNファイルおよびファンクションを使用しているすべてのプログラムが未変換状態になります。

- ファンクションのプログラム本体のみ編集する場合下記のオプションの設定により、未変換状態になる対象を、編集したFUNプログラムおよびFUNファイルのみにできます。

- [ツール] ⇒ [オプション] ⇒ "変換" ⇒ "基本設定" ⇒ "動作設定" ⇒ "変換、変換+RUN中書込み対象設定"

オプションを"高速"から"低速"に変更したあと、未変換状態のファンクションを変換すると、ファンクションを使用しているプログラム部品も変換されます。

MXコントローラの場合、"高速"から変更できません。
