# 18.6 メモリの初期化/ゼロクリア

CPUユニットに対してメモリ操作(初期化およびゼロクリア)を行う方法を説明します。

メモリ操作によって初期化される内容は、下記を参照してください。

→ MELSEC MXコントローラ MX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラ MX-Fモデルユーザーズマニュアル

→ MELSEC iQ-R シーケンサCPUユニットユーザーズマニュアル

→ MELSEC iQ-R プロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R SIL2プロセスCPUユニットユーザーズマニュアル

→ MELSEC iQ-R CPUユニットユーザーズマニュアル(応用編)

→ MELSEC iQ-F FX5ユーザーズマニュアル(応用編)

### 初期化

下記のデータを削除します。

- CPUユニットのデータメモリ/デバイスメモリ/ラベルメモリのファイル

- MXコントローラのプログラムメモリ/ファンクションメモリ/モーションデータメモリのファイル

- SDメモリカードに格納されたフォルダー/ファイルただし、ユーザ認証の情報は残ります。情報を削除する場合は、全情報の初期化を行ってください。(→ ユーザ認証機能のユーザ名/パスワードを紛失した場合)

初期化後は、再度CPUユニットに対してシーケンサデータの書込みを行ってください。

初期化後に、電源OFF→ONまたはリセットを行うと、デバイス/ラベルメモリは初期状態(各エリアはデフォルト容量)の配置となります。

操作手順1. [オンライン] ⇒ [CPUメモリ操作]を選択します。(→ メモリ使用状況の確認) 2."CPUメモリ操作"画面で初期化するメモリを選択し、[初期化]ボタンをクリックします。

SDメモリカードを初期化する場合は、画面左側に表示されている"SDメモリカード"を選択してください。

注意事項

CPUユニットに装着して使用するSDメモリカードは、本機能を使用して初期化してください。

Windowsのフォーマット機能などにより初期化した場合は、CPUユニットに装着して使用できなくなることがあります。

- 二重化システム構成のCPUユニット使用時の動作RnPCPU(二重化モード)は、運転モードにより動作が異なります。

RnPSFCPUの運転モードはバックアップモードで動作します。

| 運転モード | 動作 |
|---|---|
| セパレートモード | 接続系のCPUユニットに対して実行します。 |
| バックアップモード | 両系(制御系→待機系の順番)に実行します。 |

### 値のクリア

デバイス、ラベル、ファイルレジスタの値のゼロクリア、およびラッチクリアを行います。

ファンクションメモリ、モーションデータメモリの値はクリアできません。

操作手順1. [オンライン] ⇒ [CPUメモリ操作]を選択します。(→ メモリ使用状況の確認) 2."CPUメモリ操作"画面で"デバイス/ラベルメモリ"を選択し、[値のクリア]ボタンをクリックします。3."値のクリア"画面で各項目を設定し、[実行]ボタンをクリックします。

CPUユニットにデバイス初期値ファイル/ラベル初期値ファイルが書き込まれている場合は、CPUユニットをリセットすることでデバイス/ラベルの値の初期化が行われます。

- 二重化システム構成のCPUユニット使用時の動作RnPCPU(二重化モード)は、運転モードにより動作が異なります。

RnPSFCPUの運転モードはバックアップモードで動作します。

運転モード動作

セパレートモード接続系のCPUユニットに対して実行します。

バックアップモード両系(制御系→待機系の順番)に実行します。
