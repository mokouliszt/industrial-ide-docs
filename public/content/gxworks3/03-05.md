# 3.5 プロジェクトの機種/動作モード変更

編集中のプロジェクトをほかの機種/動作モードに変更します。

プロジェクト変更履歴に履歴が登録されているとき、同一シリーズの場合は機種/動作モード変更の実行後も履歴は引き継がれます。

プロジェクトのセキュリティバージョンが"2"の場合は、機種/動作モード変更の実行後もプロジェクトパスワードと管理者パスワードは引き継がれます。

MX-Rモデルは、軸数のみの変更が可能です。(→ 軸数変更が可能な組合せ)

MX-Fモデルは、軸数のみ、または内蔵ネットワーク種別のみの変更が可能です。

(軸数変更: → 軸数変更が可能な組合せ、内蔵ネットワーク種別変更: → 内蔵ネットワーク種別変更が可能な組合せ)

RnPCPUは、動作モードのみの変更が可能です。(→ 動作モード変更が可能な組合せ)

リモートヘッドユニットは、機種/動作モード変更には対応していません。

画面表示

[プロジェクト] ⇒ [機種/動作モード変更]を選択します。

![図(元マニュアル p.152)](images/gxworks3/03-05-01.webp)

操作手順1.各項目を選択し、[OK]ボタンをクリックします。2.変更後のプロジェクトと変更前のプロジェクトをプロジェクト照合により比較し、変更点を確認します。3.変更後の機種/動作モードに合わせて各データを編集します。

注意事項

- 機種/動作モード変更の実行後は、元のデータには戻せません。あらかじめ、プロジェクトデータを保存してから実行してください。

また、変更後はプロジェクトが未保存の状態になります。

- CPUユニットのユニットラベルを使用している場合、機種変更前のユニットラベルは削除され、機種変更後のユニットラベルが追加されます。そのため、機種変更後にプログラムの修正が必要な場合があります。

### シリーズの組合せによる注意事項

- 異なるシリーズに機種変更する場合 • 異なるシリーズに機種/動作モード変更を実行すると、プロジェクト変更履歴が削除されます。あらかじめ、プロジェクトデータを保存してから実行してください。

- 機種/動作モード変更の実行は、機種変更後のプロジェクトで装着できないユニットを削除してから行ってください。機種変更後は、システム構成に合わせてプログラムを見直してください。

機種変更後のプロジェクトで装着可能なユニットは、下記を参照してください。

→ MELSEC MXコントローラ MX-Rモデルユーザーズマニュアル

→ MELSEC MXコントローラ MX-Fモデルユーザーズマニュアル

→ MELSEC iQ-R ユニット構成マニュアル

→ MELSEC iQ-F FX5S/FX5UJ/FX5U/FX5UCユーザーズマニュアル(ハードウェア編)

- RCPUからMXコントローラ MX-Rモデルに機種変更する場合 • 下記の条件に該当する場合、MX-Rモデルへの機種変更はできません。機種変更できない条件およびその処置方法を説明します。

- CPUユニット以外のユニットがスロットに配置されている。

例R08CPUをR08ENCPUに変更する操作手順を示します。

RJ71EN71を配置します。

2. .機種を変更します。(→ プロジェクトの機種/動作モード変更)

スロットNo.0に配置されていたRJ71EN71が削除され、スロットNo.0には_RJ71EN71が配置されます。

また、ナビゲーションウィンドウに表示されているRJ71EN71の先頭I/O No.は"未設定"に変更されます。

- システムヘッダーおよびシステムフッター
