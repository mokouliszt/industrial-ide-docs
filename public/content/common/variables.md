# 変数・デバイス
デバイス、ラベル、変数、データ型など「データの入れ物」に関するカテゴリです。

3社で設計思想の違いが最も表れやすい領域です。

## 思想の違い

- **GX Works3**: 伝統的な**デバイス**(X/Y/M/D等)と、IEC風の**ラベル**を併用。ラベルはデバイス割付あり/なしを選択可能。
- **KV STUDIO**: **デバイス**(R/MR/DM等)中心。変数/ラベル機能で名前によるプログラミングも可能。
- **Sysmac Studio**: **変数**のみでプログラミング(IEC 61131-3準拠)。ハードへのアクセスはI/Oマップで変数に割り付けた「デバイス変数」経由。

## 用語対応

| 概念 | GX Works3 | KV STUDIO | Sysmac Studio |
|---|---|---|---|
| ビットデバイス | X, Y, M, L, B ... | R, MR, LR, CR ... | (変数のBOOL型) |
| ワードデバイス | D, W, R/ZR ... | DM, EM, FM, W ... | (変数のINT/WORD型等) |
| グローバル変数 | グローバルラベル | 変数(グローバル) | グローバル変数 |
| ローカル変数 | ローカルラベル | 変数(ローカル) | (POUの)ローカル変数/内部変数 |
| 構造体 | 構造体(ラベル) | 構造体 | 構造体(派生データ型) |
| 初期値 | デバイス初期値 / ラベル初期値 | デバイス初期値データ | 変数の初期値属性 |
| 保持指定 | ラッチ設定 | 電源OFF保持 | Retain属性 |

## 主な参照先

- **GX Works3**: 5章(ラベルの登録)・7章(デバイスメモリの設定)・8章(デバイス初期値)。編集可能デバイス一覧は付5、ラベルメモリは付6。
- **KV STUDIO**: デバイス操作は各章に分散(5章編集、9-9 デバイス値一括変更/読出 等)。
- **Sysmac Studio**: 4-1(変数の登録)・4-3(データ型の登録)・4-4(名前空間)。
