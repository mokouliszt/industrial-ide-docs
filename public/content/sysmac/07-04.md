# 7-4 オンラインデバッグの準備

- オンラインデバッグの準備とは

Sysmac StudioおよびNJ/NX/NYシリーズコントローラでは、プログラムで使用する変数と外部機器との入出力情報を分離した特長を持っています。そのため、変数と入出力情報の結合を定義する作業が必要になります。

## 7-4-1 変数と実I/Oの割り付け

- 変数と実I/Oの割り付けとは

“論理と物理の結合の定義”のことで、プログラム内で使用する変数と実際の機器からの入出力情報の結合の定義のことです。具体的にはグローバル変数とI/Oポートを割り付けることを言います。

- 変数と実I/Oの割り付け機能の起動手順

1. マルチビューエクスプローラで［構成・設定］–［I/Oマップ］をダブルクリックします。

![図(元マニュアル p.629)](images/sysmac/07-04-01.webp)

**使用上の注意**

本操作を行うには、あらかじめ、［構成・設定］–［CPU・増設ラック］の編集で、実際の機器に合わせてユニットを登録しておく必要があります。

- 変数と実I/Oの割り付け操作手順

［ポート］列に表示されているI/Oデバイス名に対して、［変数］列で変数名を割り付けます。

1. I/Oデバイス名に対応する［変数］列をクリック、またはカーソルを移動させ［F2］キーを押します。編集モードになり、グローバル変数がリスト形式で一覧表示されます。

![図(元マニュアル p.630)](images/sysmac/07-04-02.webp)

2. リストからグローバル変数を選択します（グローバル変数をクリック、またはカーソルを移動させ［Enter］キー押します）。次の画面に示すように、選択した変数が表示されます。また変数名を直接入力することで、入力した文字で絞り込んで変数を一覧表示し、リストから選択することも可能です。

![図(元マニュアル p.630)](images/sysmac/07-04-03.webp)

- I/Oポートを表示するには、あらかじめマルチビューエクスプローラの［構成・設定］–［CPU・増設ラック］でユニットを登録する必要があります。ユニットを登録することで、そのユニットの持つI/Oポートは自動的に登録されます。

- ［ポート］列のアイコン（：、）をクリックすることにより、CPUラック（または増設ラッ

- ク）、ユニット名の単位で、I/Oポートの構成を展開・省略表示することができます。

- ドロップダウンリストに変数を表示するためには、あらかじめグローバル変数テーブルで登録する必要があります。

- I/Oマップから変数名を直接入力し確定させることで、グローバル変数を新規作成することも可能です。

I/Oマップの各列の内容

I/Oマップの各列の内容は、以下のとおりです。

| 列 | 内容 | 編集の可否 |
|---|---|---|
| 位置 | ユニット位置を示す情報を表示します（ラック／ユニット行のみ）。 | 編集不可 |
| ポート | I/Oポートが属している、ネットワーク、ユニットを階層的に表示します。 同一ポートに複数の変数が登録されている場合、注意アイコンとツールチップが表示されます。 | 編集不可 |
| 説明 | I/Oポートの説明が表示されます。 | 編集不可 |
| R/W | I/OポートのRead/Write属性が以下で表示されます。 読出専用：R、書込専用：W、読書可能：RW | 編集不可 |
| データ型 | I/Oポートのデータ型が表示されます。 基本データ型と、基本データ型の配列型が表示されます。 | 編集不可 |
| 変数 | / 変数セルに変数名を直接入力して、ポートに変数を割り付けることができます。 / 直接入力した名称の変数がすでに別のポートに割り付けられている、または異なるデータ型ですでに定義されている場合、エラーアイコンとツールチップが表示されます。 / I/Oポートに割り付けられた変数名が表示されます。I/Oポートに変数が割り付けられていない場合、空白が表示されます。 / ポップアップメニューの［割付変数一覧］を選択することで、対象ポートに複数個割り付けられた変数を確認することができます。 | 編集可能 |
| 変数コメント | / 割り付けられた変数のコメントを表示、編集します。 / 変数セルが入力済みのときに編集可能です。 | 編集可能 |
| 変数種別 | / デバイス変数が属する変数テーブルを表示、編集します。 コンボボックスから以下の変数テーブルを選択できます。 グローバル変数、登録済みのプログラム、ファンクション、ファンクションブロック / I/Oポートに変数が割り付けられている場合、その変数が属している変数テーブルを表示します。I/Oポートに変数が割り付けられていない場合、初期表示は空白です。空白の場合、グローバル変数テーブルが選択されます。 | 編集可能 |

## 7-4-2 軸変数と入出力デバイスの割付け

- 軸変数と入出力デバイスの割り付けとはプログラム内で使用する軸変数と実際の機器で動作させる入出力デバイスを定義することです。軸変数と、EtherCATスレーブのドライブやNXユニットの位置インターフェースデバイスを割り付けます。

- 軸変数と入出力デバイスの割り付けの操作手順マルチビューエクスプローラに登録されている軸変数に対してEtherCATスレーブを割り付ける場合を例に説明します。

1. マルチビューエクスプローラの［構成・設定］–［モーション制御設定］–［軸設定］から軸変数をダブルクリックします。または右クリックし、メニューから［編集］を選択します。

2. 表示される軸基本設定の設定項目の軸番号、軸使用、軸種別、フィードバック情報を入力します。またはリストから選択します。入力デバイスをリストから割付対象のEtherCATスレーブを選択します。次の画面に示すように、選択した変数が表示されます。

![図(元マニュアル p.631)](images/sysmac/07-04-04.webp)

- 軸変数は、マルチビューエクスプローラの［構成・設定］–［モーション制御設定］–［軸設定］で、使用する軸をあらかじめ登録しておく必要があります。軸を登録することで、その軸の変数が自動的に登録されます。

- 入力デバイスのリストに表示されるEtherCATスレーブは、マルチビューエクスプローラの［構成・設定］–［EtherCAT］でEtherCATスレーブの軸機能のユニットをあらかじめ登録しておく必要があります。

軸機能のユニットを登録することで、そのユニットの持つEtherCATスレーブが自動的に登録されます。
