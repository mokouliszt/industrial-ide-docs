# 8-3 セキュリティ設定

資産の保護および誤操作の防止のために、コントローラ内のユーザプログラムや各種データを保護したり、Sysmac Studio での操作を制限したりすることができます。

- セキュリティ設定の概要セキュリティ設定用途と機能概要は以下のとおりです。

| 機能名 | 用途 | 機能概要 |
|---|---|---|
| セキュア通信設定 | データの機密性／ 整合性の確保 | セキュア通信はSysmac Studio とコントローラ間 の通信データを第三者によるデータの盗聴や改ざん を防止することができます。このセキュア通信の保 護レベルを設定する機能です。 |
| 操作権限認証 | 誤操作の防止 | 操作者の権限により、操作できる機能を制限します。 |
| ユーザ認証 | 資産の盗用防止／ 誤操作の防止 | オンライン接続時にユーザ名とパスワードによるユ ーザ認証を行い、オンライン操作するユーザを特定 します。 |
| ユーザプログラム実行用ID | 資産の盗用防止 | ユーザプログラムを複製しても、別のCPU ユニット では動作させないようにします。 |

## 8-3-1 セキュア通信設定

セキュア通信はSysmac Studio とコントローラ間の通信データを暗号化し、またハッシュ値を付けて送受信を行うため、第三者によるデータの盗聴や改ざんを防止することができます。このセキュア通信の保護レベルをセキュア通信設定の［セキュア通信バージョン］によって設定することができます。

セキュア通信は、以下のCPU ユニットのユニットバージョンとSysmac Studio の組み合わせで使用可能です。セキュア通信バージョン2 を使用できるユニットバージョンでは、セキュア通信バージョン1 とセキュア通信バージョン2 のどちらを使用するかを設定できます。運用に応じてセキュア通信バージョンを選択してください。

|---|---|---|---|---|

| セキュア通信バージ ョン設定不可 （セキュア通信バー ジョン1 として動 作） | Ver.1.49～ Ver.1.68 | Ver.1.60～ Ver.1.68 | Ver.1.29～ Ver.1.35 | Ver.1.50～ Ver.1.62 |
| セキュア通信 | NJ シリーズ、形 NX102、形NX1P2 CPU ユニット | 形NX502 CPU ユ ニット | 形NX701 CPU ユ ニット | Sysmac Studio の バージョン |
| セキュア通信非対応 | Ver.1.48 以前 | Ver.1.59 以前 | Ver.1.28 以前 | Ver.1.49 以前 |

## 8-3-2 操作権限認証

コントローラのデータを変更する場合、操作ミスにより人的被害や物的被害を与える危険性があります。操作者の権限により操作できる機能を制限させることで操作ミスの防止を実現できます。

操作権限認証

- 操作権限認証とは操作ミスにより装置や人に危害を与える可能性があるため、操作権限に応じて、オンラインで操作できる機能を制限するための機能です。

オンライン接続時に操作権限のパスワードが要求され、パスワード認証に合格すると、操作権限に応じたオンラインでの機能操作が可能となります。

管理者が操作権限ごとにパスワードを設定し、スキルに応じて操作権限名とパスワードを使用者に連絡することで運用します。

なお、操作権限の認証設定はコントローラに転送して保存するため、別のパソコンを接続しても操作権限の認証が可能です。

使用上の注意

- 操作権限認証の設定はオンライン接続時のみ可能です。

- 操作権限認証の変更およびコントローラへの転送は管理者のみ可能です。

- 操作権限のパスワードを忘れるとオンライン接続ができません。忘れないよう、パスワードを管理してください。

- 操作権限の種類

- Sysmac Studio の操作権限の種類は、権限の大きい順に以下の5 種類です。

- 管理者

- 設計者

- 保全者

- 操作者

- 観察者（パスワードは必要ありません）

都度認証

操作ミスによる装置や人に危害を加える可能性のある操作については、一定権限以下のユーザは都度認証を行います。対象となる機能と権限は、「対象となるオンライン操作機能」の△を確認してください。

認証ダイアログボックスでパスワードを5 回間違えると、Sysmac Studio の操作が10 分間ロックされるので、ご注意ください。

![図(元マニュアル p.678)](images/sysmac/08-03-10.webp)

操作ロック

操作ロックは、［操作権限の設定］ダイアログボックスの設定時間の間、ユーザが操作を行わないと認証をロック状態にし、操作ができなくなる機能です。この機能により異なるユーザが間違って使用してしまうことを防止することができます。操作ロック状態になった場合、再度、認証確認を行う必要があります。操作ロック機能を使用するためには、操作権限の設定が設定されている必要があります。単独で使用することはできません。

- 操作ロックの設定手順

1. ［コントローラ］–［セキュリティ］–［操作権限の設定］を選択します。［操作権限の設定］ダイアログボックスが表示されます。

2. ［操作ロック］の［有効］にチェックをつけ、［設定時間］を入力して［転送［パソコン→コントローラ］］ボタンをクリックします。

ユーザ認証機能

- 概要

コントローラを操作するユーザをあらかじめコントローラに登録しておき、コントローラをオンライン操作するユーザを特定する機能です。コントローラにオンライン接続する際に、ユーザ名とパスワードの入力が要求されます。ユーザ名およびパスワードがあらかじめ設定されたものと一致しないと、オンライン接続することができません。さらに、各ユーザには、管理者、設計者、保全者、操作者、および観察者のいずれかの操作権限を割り当てます。これにより、各ユーザが自分に割り当てられた権限範囲でしかコントローラへのオンライン操作ができないようにします。ユーザ名、パスワード、およびそのユーザの操作権限の情報などのユーザ認証設定は、コントローラに保存されます。したがって、別のパソコンからコントローラに接続しても、ユーザ認証機能を使用することができます。ユーザ認証設定は、プロジェクトには保存されません。ユーザ認証機能を使用するコントローラごとに、ユーザ認証設定を行います。

使用上の注意

- ユーザ認証の設定はオンライン接続時のみ可能です。

- ユーザ認証の変更およびコントローラへの転送は管理者のみ可能です。

- ユーザ認証のパスワードを忘れるとオンライン接続ができません。忘れないよう、パスワードを管理してください。パスワードを忘れた場合の対処方法については、後述の「管理者のユーザ名、パスワードを忘れた場合の対処方法」を参照してください。
