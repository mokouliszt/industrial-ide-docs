# 8-6 メモリオールクリア

- メモリオールクリアとは

Sysmac Studioから、CPUユニットや装着されているユニットのデータを工場出荷状態に初期化する機能です。

## 8-6 -1 CPUユニットのメモリオールクリア

CPUユニット内のユーザプログラム、コントローラ構成・設定、変数、絶対値エンコーダ原点位置オフセット、XバスユニットおよびNXユニットのデータを工場出荷状態に初期化します。

- 操作によりクリアされる対象とクリア後の状態

| データ単位 | | メモリオールクリア後の状態 |
|---|---|---|
| ユーザプログラム | | 1つのプログラムが、プライマリ定周期タスク（タスク周期：4ms）に割り当てられた状態となります。プログラムは空状態となります。 |
| コントローラ構成・設定 | | 初期値となります。 セキュリティ情報（プロテクト有無・セキュア通信バージョン・権限認証・ユーザ認証など）も初期化されます。 |
| 変数 | システム定義変数 | 変数の種類によります |
| 変数 | デバイス変数 | 変数テーブルがクリアされることにより、デバイス変数は削除されます。 |
| 変数 | ユーザ定義変数 | 変数テーブルがクリアされることにより、ユーザ定義変数は削除されます。 |
| 絶対値エンコーダ原点位置オフセット | ユーザ定義変数 | クリアされます。 |
| Xバスユニット*1 | | CPUラック上のXバスユニットのデータがメモリオールクリアされます。 |
| NXユニット*2 | | CPUラック上のNXユニットのデータがメモリオールクリアされます。 |

*1.形NX502 CPUユニットにXバスユニットが装着されている場合だけ対象となります。*2.形NX502、形NX102、形NX1P2 CPUユニットにNXユニットが装着されている場合だけ対象となります。

- メモリオールクリアの操作手順

1. オンライン状態で、メニューから［コントローラ］–［メモリオールクリア］を選択します。［メモリオールクリアの実行確認］ダイアログボックスが表示されます。

![図(元マニュアル p.712)](images/sysmac/08-06-01.webp)

2. 初期化するエリアを確認します。必要に応じてオプションのチェックボックスにチェックを付けます。

3. ［OK］ボタンをクリックし、メモリオールクリアを実行します。メモリオールクリアが完了すると、次のダイアログボックスが表示されます。

![図(元マニュアル p.713)](images/sysmac/08-06-02.webp)

4. ［OK］ボタンをクリックします。運転モードではメモリオールクリアは実行できません。

- メモリオールクリアのオプション

| 項目 | 内容 |
|---|---|
| イベントログをクリアする。 | チェックを入れるとイベントログがクリアされます。 |
| OPC UAサーバ証明書、 セキュリティプロファイルをクリアする。 *1 | チェックを入れると、OPC UAサーバ証明書、セキュリティプロファイルがクリアされます。 |
| データの完全消去を行う。 *2 | チェックを入れると、メモリオールクリア機能で消去した領域に無意味なデータを上書きすることにより、データの復元を防止します。この場合、メモリオールクリアの完了までに時間がかかることがあります。 本チェックボックスを選択すると、［イベントログをクリアする。］、［OPC UAサーバ証明書、セキュリティプロファイルをクリアする。］チェックボックスも自動的に選択されます。 |

- *1.OPC UA機能をサポートするコントローラの場合だけ表示されます。詳細は、『NJ/NXシリーズCPUユニットユーザーズマニュアルOPC UA編（SBCD-374）』を参照してください。*2.以下のコントローラの場合だけ表示されます。

- NJシリーズ、形NX502、形NX102、形NX1P2 CPUユニット: Ver.1.60以降

- 形NX701 CPUユニット: Ver.1.32以降データの完全消去については、『NJ/NXシリーズCPUユニットユーザーズマニュアルソフトウェア編（SBCA-467）』を参照してください。

安全上の要点

メモリオールクリア後は、工場出荷状態のコントローラでシステム構成を作成した直後と同様の動作となります。

**使用上の注意**

- メモリオールクリアは、プログラムモード時のみ実行可能です。

- メモリオールクリアは、セキュリティ機能「CPUユニットへの書込プロテクト」が設定されていると、実行できません。CPUユニットへの書込プロテクトを解除してから実行してください。

- コントローラのユーザ認証機能が有効になっている場合、メモリオールクリア完了後はユーザアカウントが削除され通信異常が発生します。作業を継続するには、一度オフラインにし、再度オンライン接続した状態でユーザアカウントの新規設定を実行してください。ユーザアカウントの新規設定の詳細は「8-3-3ユーザ認証」を参照してください。

- メモリオールクリアを実行すると、絶対値エンコーダ原点位置オフセットの値はクリアされます。メモリオールクリア前の絶対値エンコーダ原点位置オフセットの値を残したい場合は、メモリオールクリアを実行する前にあらかじめコントローラバックアップを実行してください。メモリオールクリア後に絶対値エンコーダ原点位置オフセットを元の値に戻したい場合は、コントローラバックアップ機能でバックアップしたファイルのリストアを実行してください。コントローラバックアップ機能については「8-10-1コントローラバックアップ機能」を参照してください。

- メモリオールクリアの実行によって初期化されるNXユニットのエリアの詳細と初期化後の状態は、ユニットによって異なります。詳細については、NXユニットを接続するCPUユニットおよび通信カプラユニットのユーザーズマニュアルを参照してください。

## 8-6 -2 Xバスユニットのメモリオールクリア

CPUユニットに装着されたXバスユニットのデータを個々に初期化します。

バージョン関連情報本機能は、形NX502 CPUユニットVer.1.60以降とSysmac Studio Ver.1.54以降の組み合わせで使用可能です。

- メモリオールクリアの操作手順

1. オンライン状態で、対象のXバスユニットを右クリックして［Xバスユニットのメモリオールクリア］を選択します。メモリオールクリアダイアログボックスが表示されます。

![図(元マニュアル p.714)](images/sysmac/08-06-03.webp)

2. 初期化するエリアを確認し、［メモリオールクリア実行］ボタンをクリックします。実行確認ダイアログボックスが表示されます。

3. ［はい］ボタンをクリックします。メモリオールクリアが完了し、Xバスユニットのリスタートが行われます。

安全上の要点

Xバスユニットのメモリオールクリアは、設備に影響がないことを確認してから実行してください。

- メモリオールクリアのオプション

| 項目 | 内容 |
|---|---|
| イベントログをクリアする。 | チェックを入れるとイベントログがクリアされます。 |

## 8-6 -3 NXユニットのメモリオールクリア

CPUユニットに装着されたNXユニットや、スレーブターミナル上のNXユニットのデータを個々に初期化します。

- メモリオールクリアの操作手順

EtherCATカプラユニットを例に示します。

1. オンライン状態で、対象のNXユニットを右クリックして［メモリオールクリア］を選択します。メモリオールクリアダイアログボックスが表示されます。

![図(元マニュアル p.715)](images/sysmac/08-06-04.webp)

2. 初期化するエリアを確認し、［メモリオールクリア実行］ボタンをクリックします。実行確認ダイアログボックスが表示されます。

3. ［はい］ボタンをクリックします。メモリオールクリアが完了し、NXユニットのリスタートが行われます。

**使用上の注意**

形NX1P2CPUユニットに装着されたNXユニットは、個々にメモリオールクリアすることはできません。CPUユニットのメモリオールクリアを行ってください。

- メモリオールクリアのオプション

| 項目 | 内容 |
|---|---|
| イベントログをクリアする | チェックを入れるとイベントログがクリアされます。 |

## 8-6-4 スレーブターミナル上のユニットのメモリオールクリア

スレーブターミナル上の、個々のユニットまたはすべてのユニットのデータを初期化します。

1. オンライン状態で、スレーブターミナル構成編集画面上のユニットを右クリックして［メモリオールクリア］を選択します。すべてのユニットのメモリオールクリアを行う場合は、通信カプラユニットを右クリックして［メモリオールクリア］を選択します。メモリオールクリアダイアログボックスが表示されます。

![図(元マニュアル p.716)](images/sysmac/08-06-05.webp)

2. 初期化する対象エリアを確認し、［メモリオールクリア実行］ボタンをクリックします。イベントログをクリアする場合は、イベントログにチェックを付けます。すべてのユニットのメモリオールクリアを行う場合は、「通信カプラのエリア選択」で［通信カプラ＋NXユニット］を選択してから行います。実行確認ダイアログボックスが表示されます。

3. ［はい］ボタンをクリックします。メモリオールクリアが完了し、リスタートが行われます。

**使用上の注意**

- メモリオールクリアの実行によって初期化されるエリアの詳細と初期化後の状態については、通信カプラユニットのユーザーズマニュアルを参照してください。

- メモリクリア完了後はリスタートを行います。これに伴って通信マスタとのIO通信が停止します。リスタートの対象は、メモリオールクリアの実行対象、および通信カプラユニットのユニットバージョンによって異なります。詳細は、通信カプラユニットのユーザーズマニュアルを参照してください。
