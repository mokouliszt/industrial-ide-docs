# 8-10 バックアップ機能

バックアップ機能とは、CPU ユニットなどのハードの交換や装置のデータ復元を目的に、NJ/NX/NYシリーズコントローラのユーザプログラムやデータなどを、バックアップやリストア、照合する機能です。次の機能があります。

| 機能 | 説明 |
|---|---|
| コントローラバックアップ機能 *1 | コントローラ内のデータを一括でバックアップ・リストア・ 照合します。 |
| SD メモリカードバックアップ機能 *1 | NJ/NX シリーズコントローラに装着しているSD メモリカ ード、またはNY シリーズコントローラの仮想SD メモリカ ードに、コントローラ内のデータを一括でバックアップおよ び照合します。 |
| 変数/メモリのバックアップ機能 *2 | コントローラの保持メモリの内容をバックアップ・リストア します。 |
| バックアップファイルインポート/エクスポート | ｢コントローラバックアップ機能｣あるいは｢SD メモリカー ドバックアップ機能｣でバックアップされたファイルを、 Sysmac Studio のプロジェクトへインポートする、あるい はSysmac Studio のプロジェクトデータからバックアッ プファイルへエクスポートします。 |

*1.CPU ユニットVer.1.03 以降とSysmac Studio Ver.1.04 以降の組み合わせで使用可能です。*2.本機能でバックアップするデータは「コントローラバックアップ機能」でバックアップするデータに含まれます。

バックアップ機能の詳細や対象機種、対象データについては、『NJ/NX シリーズ CPU ユニット ユーザーズマニュアル ソフトウェア編（SBCA-467）』または『NY シリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル ソフトウェア編（SBCA-436）』を参照してください。

![図(元マニュアル p.722)](images/sysmac/08-10-01.webp)

安全上の要点

- コントローラは絶対値エンコーダの情報である「絶対値エンコーダ原点位置オフセット」を内蔵し、バッテリで保持しています。以下のいずれかの条件を満たす場合は、絶対値エンコーダ原点位置オフセットをリストア対象からはずしてリストアを実施してください。そして、原点確定をやり直してください。原点を確定しなかった場合、制御対象が予期しない動作をする恐れがあります。a) サーボモータとドライバの個体が、バックアップを作成したときの個体と異なるときb) バックアップを作成した以降に、絶対値エンコーダのセットアップを実行しているときc) 絶対値エンコーダの絶対値データが失われているとき

- バックアップしたデータの一部をリストア時に復元対象から外す場合は、外しても問題がないことを確認してから行ってください。装置が誤動作する恐れがあります。

- バックアップデータをユニットへリストアする際に、不適切なバックアップデータが設定されると、設備が予期しない動作をする恐れがあります。設備に影響がないことを確認してから、転送を実行してください。

- ハードウェアリビジョンが異なると性能が異なる場合があります。異なるハードウェアリビジョンのCPU ユニットにユーザプログラムおよび各種データ・設定値を転送する際には、十分な動作確認を行った後、本運転に移行してください。

## 8-10-1 コントローラバックアップ機能

コントローラのデータをファイルにバックアップし、バックアップしたファイルの内容を照合、リストアします。バックアップ機能の詳細や対象機種、対象データについては、『NJ/NX シリーズ CPU ユニット ユーザーズマニュアル ソフトウェア編（SBCA-467）』または『NY シリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル ソフトウェア編（SBCA-436）』を参照してください。

バックアップの実行

1. Sysmac Studio で対象のコントローラに接続し、［ツール］–［バックアップ］–［コントローラのバックアップ］を選択します。

![図(元マニュアル p.723)](images/sysmac/08-10-02.webp)

バックアップファイルの保存先フォルダを指定するダイアログボックスが表示されます。

![図(元マニュアル p.723)](images/sysmac/08-10-03.webp)

2. フォルダを選択し、［OK］ボタンをクリックします。バックアップダイアログボックスが表示されます。例：NJ シリーズコントローラの場合

![図(元マニュアル p.724)](images/sysmac/08-10-04.webp)

3. バックアップ対象データを選択し、［実行］ボタンをクリックします。バックアップが開始されます。バックアップが完了すると、続けて照合確認ダイアログボックスが表示されます。

![図(元マニュアル p.724)](images/sysmac/08-10-05.webp)

［はい］を選択すると、照合対象の確認ダイアログが表示されます。

照合の手順については、後述の「照合の実行」を参照してください。

参考

- バックアップファイル名は対象のコントローラに応じて自動的につけられます。ファイル名を変更することはできません。

- 異なるシリーズのコントローラの間で、バックアップファイルに互換性はありません。

使用上の注意

- リストアを実行した際に、コントローラのCPU ユニット形式またはCPU ユニットバージョンがバックアップファイルとコントローラとで異なる場合、確認ダイアログボックスが表示されます。対象のコントローラを確認してからリストアを実行してください。

- リストアによってEtherNet/IP ポートのIP アドレスが変更された場合、通信が切断される可能性があります。IP アドレスが変更されても問題がないことを確認してリストアを実行してください。接続先のコントローラのユニットバージョンがVer.1.14 以降の場合は、内蔵EtherNet/IP ポートのIP アドレスの設定をリストア対象から除外することができます。［対象データ］の項目［内蔵EtherNet/IP ポート設定のIP アドレス］のチェックを外して実行します。詳細については、『NJ/NX シリーズ CPU ユニット ユーザーズマニュアル ソフトウェア編（SBCA-467）』または『NY シリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル ソフトウェア編（SBCA-436）』を参照してください。• リストア実行中は、キャンセルすることができません。

## 8-10-2 SD メモリカードバックアップ機能

SD メモリカードバックアップ機能とは、SD メモリカードまたは仮想SD メモリカードを用いて、コントローラのデータのバックアップ、リストア、照合をする、コントローラがもつ機能です。Sysmac Studio からは、このうちバックアップ、照合を実行することができます。

バックアップ対象データの詳細およびリストアの手順については、『NJ/NX シリーズ CPU ユニットユーザーズマニュアル ソフトウェア編（SBCA-467）』または『NY シリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル ソフトウェア編（SBCA-436）』を参照してください。

バックアップの実行

1. SD メモリカードウィンドウでバックアップファイルを作成するフォルダを表示し、ツールバーからSD メモリカードバックアップボタン（：）をクリックします。

SD メモリカードバックアップダイアログボックスが表示されます。

![図(元マニュアル p.727)](images/sysmac/08-10-11.webp)

2. ［実行］ボタンをクリックします。バックアップが開始されます。バックアップが完了すると、完了ダイアログボックスが表示されます。

![図(元マニュアル p.727)](images/sysmac/08-10-12.webp)

使用上の注意

対象のコントローラでバックアップ、リストア、照合が実行されているあいだは、バックアップを実行することはできません。エラーメッセージを表示して処理を終了します。

参考

バックアップファイル名は対象のコントローラに応じて自動的につけられます。ファイル名を変更することはできません。

照合の実行

1. SD メモリカードウィンドウで照合対象のファイルを表示し、ツールバーからSD メモリカードバックアップファイル照合ボタン（：）をクリックします。

SD メモリカードバックアップファイル照合ダイアログボックスが表示されます。

![図(元マニュアル p.728)](images/sysmac/08-10-13.webp)

## 8-10-3 変数／メモリのバックアップ機能

- 変数／メモリのバックアップ機能とは

NJ/NX/NY シリーズコントローラの交換を目的に、交換前の実機の保持メモリの内容をファイルにバックアップし、交換後の実機へバックアップしたファイルの内容をリストアすることができる機能です。

使用上の注意

- 変数／メモリのバックアップファイルとファイルを作成したプロジェクトファイルはセットで保管、運用してください。

- 変数／メモリのバックアップファイルを作成したプロジェクトファイルをコピー、ファイル名を変更したファイルからでも、バックアップファイルのリストアは可能です。

- 作成したプロジェクトファイルから変数名の変更、変数の削除などバックアップした変数に編集があった場合、その変数／メモリの値はリストアされません。リストアができなかった変数／メモリの情報はエラーとして残ります。

参考

構造体変数のメンバを追加してCPU ユニットVer.1.10 以前のコントローラに転送する場合、構造体変数の既存メンバの現在値が初期化されます。このような場合は、メンバを追加してダウンロードを実行する前に保持属性の構造体変数の現在値をバックアップし、メンバを追加してコントローラへ転送したあとに、対象の構造体変数を選択して現在値をリストアすることで、既存メンバの現在値を書き戻すことができます。対象変数は、[保持属性の変数の現在値]のオプションから選択します。CPU ユニットVer.1.11 以降のコントローラでは、構造体変数の既存メンバの現在値は保持されます。現在値の保持動作の詳細は、『NJ/NX シリーズ CPU ユニット ユーザーズマニュアルソフトウェア編（SBCA-467）』または『NY シリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル ソフトウェア編（SBCA-436）』を参照してください。

- 変数値は、データ型ごとに定められた値の範囲、フォーマットで入力してください。不正な値を入力するとリストアできません。データ型ごとの値の範囲、フォーマットは『NJ/NX シリーズ CPU ユニット ユーザーズマニュアル ソフトウェア編（SBCA-467）』または『NYシリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル ソフトウェア編（SBCA-436）』を参照してください。

- TIME_OF_DAY 型の変数値をExcel で変更するときには次のことに注意してください。TIME_OF_DAY 型の変数値は、次のフォーマットでCSV ファイルに保存されます。「hh:mm:ss.tt」(hh 時、mm 分、ss 秒、tt ミリ秒)

このCSV ファイルをExcel で開くと、TIME_OF_DAY 型の変数値は次のフォーマットで表示されます。「mm:ss」(mm 分、ss 秒)

この表示のまま保存すると、「hh」および「tt」部分のデータが失われ、リストアを実行時にTIME_OF_DAY 型の変数がエラーとなります。これを回避するために、TIME_OF_DAY 型の変数値の表示フォーマットを「hh:mm:ss.tt」に変更してから保存してください。
