# 3-3 新規プロジェクトの作成

Sysmac Studio の起動と終了、新規プロジェクトの作成・保存などの基本操作について説明します。

## 3-3-1 Sysmac Studio の起動と終了

使用上の注意

- Sysmac Studio および周辺ツールを起動する前に、以下の点にご注意ください。

- アドミニストレータ権限のないユーザがSysmac Studio を起動すると、ファイルアクセスエラーが発生し使用できないことがあります。この場合、Windows の「ユーザーアカウント制御（UAC）」を有効にしてから再度Sysmac Studio を起動してください。

- 表示器のシミュレーション機能を使用する場合は、必ずアドミニストレータかアドミニストレータ権限のあるユーザでWindows にログインし、Sysmac Studio を起動してください。

- Sysmac Studio の使用に不要なアプリケーションは終了してください。ウィルスチェックソフトなどによりSysmac Studio の起動や動作に影響が出る場合は、Sysmac Studio をチェック対象から除外するなどの対策を実施してください。

- パソコンのハードディスクやプリンタをネットワークで他のパソコンに共有させている時は、共有を解除してください。

- ノートパソコンで、初期設定がUSB ポートをモデムや赤外線に割当てているものがあります。パソコンの説明書にしたがってUSB ポートを通常のシリアルポートとして使用できるように設定を変更してください。

- ノートパソコンで、初期設定が省電力のためにUSB およびEthernet ポートに規定の電力を供給しない設定になっているものがあります。省電力設定はWindows の設定のほか、パソコン独自のユーティリティやBIOS で設定されている場合があります。パソコンの説明書にしたがって、すべての省電力機能を停止させてください。

- ライセンスの登録にはWindows の管理者権限が必要です。

使用上の注意

- デバイスを変更した場合、変更後の機種でサポートしていない機能については設定値が失われることがあります。

- 画像センサでは「デバイスの変更」を行うことはできません。

- 異なるカテゴリへの変更はできません。

コントローラのバージョンについてデバイス選択でコントローラを選択した場合、「バージョン」を設定してください。「バージョン」には、使用するコントローラのユニットバージョン、またはそれ以前のユニットバージョンを設定します。これにより、設定したユニットバージョンのコントローラがサポートする範囲で、プログラミングや設定が可能となります。設定したユニットバージョンがサポートしていない機能については、使用できないか、または使用するとエラーとなります。

たとえば、NJ シリーズコントローラで「バージョン」を1.01 に設定すると、ユニットバージョン1.01 のコントローラがサポートする範囲で使用することができます。作成したユーザプログラムは、ユニットバージョン1.02 のコントローラに転送できますが、ユニットバージョン1.00 のコントローラには転送できません。

「バージョン」は、コントローラを追加したあとに変更することもできます。

コントローラのユニットバージョンとサポートする機能についての詳細は、『NJ/NX シリーズ CPUユニット ユーザーズマニュアル ソフトウェア編（SBCA-467）』および『NY シリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル ソフトウェア編（SBCA-436）』を参照してください。

使用上の注意

バージョンを変更した場合、変更後のユニットバージョンでサポートしていない機能については設定値が失われることがあります。

安全上の要点

EtherCAT ネットワーク構成に、形式変更後のコントローラで使用できないEtherCAT スレーブが含まれる場合、EtherCAT ネットワークの構成・設定はすべてクリアされます。EtherCATネットワーク構成の作成、設定を再度行ってください。
