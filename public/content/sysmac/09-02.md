# 9-2 装置・プログラム資産の再利用ユースケース

代表的な装置の改造事例と、設計資産の再利用のケースを、従来のアドレスによるプログラミングとSysmac Studio による変数プログラミングの作業内容を比較して示します。

## 9-2-1 I/O 割り付けの変更

- 事例）装置の保全性改善において、入出力機器から基本I/O ユニットへの長距離配線を、リモートI/O スレーブ機器を使用したネットワークによる省配線の装置構成へ、入出力機器からリモートI/Oスレーブ機器へ設計変更を実施するとき束線によるダクト内配線：通信ケーブルによるダクト内配線

入力機器：出力機器：入力機器：出力機器

| | 目的 | 従来のプログラミング | Sysmac Studio のプログラミング |
|---|---|---|---|
| 1 | 旧機器の割り付けア ドレスを確認します。 | 基本I/O ユニットの割り付けアドレスA と I/O コメント情報を確認します。 | － |
| 2 | 新機器の割り付けア ドレスを確認します。 | リモートI/O スレーブ機器が割り付くアドレ スB を算出します。 （注）算出したアドレスB がすでに内部リレ ーとして使用している場合、内部リレー を空きアドレスC に割り付けます。 | － |
| 3 | 旧機器のアドレスを 新機器のアドレスに 置き換えます。 | アドレスA のI/O コメント情報を算出したア ドレスB に振り直します。 （注）算出したアドレスB がすでに内部リレ ーとして使用している場合、先にアドレ スB のI/O 情報をアドレスC に振り直 します。 | 基本I/O ユニットをユニット構成から削除し ます（I/O デバイスと変数の割り付けは解除さ れます）。 |
| 3 | 旧機器のアドレスを 新機器のアドレスに 置き換えます。 | プログラムで使用のアドレスA を対応するア ドレスB に置き換えます。 （注）算出したアドレスB がすでに内部リレ ーとして使用している場合、先にアドレ スB をアドレスC に振り直します。 | リモートI/O スレーブをネットワーク構成に 登録します。 |
| 3 | 旧機器のアドレスを 新機器のアドレスに 置き換えます。 | リモートI/O スレーブをネットワーク構成に 登録します。 | 変数とリモートI/O 機器のI/O デバイスの割 り付けを行います。 |
| 4 | プロジェクトのユニ ット構成を変更しま す。 | 基本I/O ユニットをユニット構成から削除し ます。 | － |
| 4 | プロジェクトのユニ ット構成を変更しま す。 | アドレスA を変数テーブルから削除します。 | － |
| 5 | 装置構築および配線 を実施します。 | 装置構成図、配線図をもとに入出力機器をリ モートI/O 機器の端子に配線します。 | 装置構成図、配線図をもとに入出力機器をリ モートI/O 機器の端子に配線します。 |
| 6 | 配線チェック、アドレ スチェックを行いま す。 | 配線図とおりにカラーコードが接続されてい ることを目視で、またはテスタなどにより導 通チェックで確認します。 | 配線図とおりにカラーコードが接続されてい ることを目視で、またはテスタなどにより導 通チェックで確認します。 |
| 6 | 配線チェック、アドレ スチェックを行いま す。 | アドレスB への出力結果がリモートI/O スレ ーブ機器に出力、また入力機器からのダミー 入力が入力エリアに反映されることを確認し ます。 | 変数への出力結果がリモートI/O スレーブ機 器に出力、また入力機器からのダミー入力が 変数に反映されることを確認します。 |
| 7 | プログラムのデバッ グを行います。 | 機器変更によるアドレス振り替え修正ミス、 動作タイミングに影響がないことを確認しま す。 | 機器変更による動作タイミングに影響がない ことを確認します。 |
| 8 | 装置の試運転を行い ます。 | 変更後の機器構成、制御プログラムの試運転 を実施します。 | 変更後の機器構成、制御プログラムの試運転 を実施します。 |

## 9-2-2 装置オプションの削除

- 事例） 生産規模の縮小などを目的に、装置の機器を部分的に縮小、削除するとき

| | 目的 | 従来のプログラミング | Sysmac Studio のプログラミング |
|---|---|---|---|
| 1 | 削除する機器の割り 付けアドレスを確認 します。 | 削除する機器で使用しているアドレスを確認 します。 | － |
| 2 | 削除する機器の制御 プログラムを確認し ます。 | 削除する機器を制御しているプログラムを確 認します。 | 削除する機器を制御しているプログラム、プ ログラムタスクを確認します。 |
| 2 | 削除する機器の制御 プログラムを確認し ます。 | 削除する機器の監視プログラム、インターロ ック回路などを確認します。 | 削除する機器の監視プログラム、インターロ ックプログラム、プログラムタスクを確認し ます。 |
| 3 | プロジェクトのユニ ット構成を変更しま す。 | 削除する機器をユニット構成から削除しま す。 | 削除する機器をユニット構成から削除します （I/O デバイスと変数の割り付けは解除されま す）。 |
| 4 | 不要となる機器の制 御プログラムを削除 します。 | 削除する機器を制御しているプログラムを削 除します。 | 削除する機器を制御するプログラムをプログ ラムタスクから削除します。 |
| 4 | 不要となる機器の制 御プログラムを削除 します。 | 削除する機器の監視プログラム、インターロ ック回路などを修正します。 | 削除する機器の監視プログラム、インターロ ックプログラムを修正します。 |
| 4 | 不要となる機器の制 御プログラムを削除 します。 | 削除する機器で使用していたアドレスを変数 テーブルから削除します。 | － |
| 5 | 削除する機器で使用 しているユニットを 取り外します。 | 削除する機器で使用しているユニットから外 部機器への配線を取り外します。 | 削除する機器で使用しているユニットから外 部機器への配線を取り外します。 |
| 5 | 削除する機器で使用 しているユニットを 取り外します。 | 削除する機器で使用しているユニット、外部 機器を取り外します。 | 削除する機器で使用しているユニット、外部 機器を取り外します。 |
| 6 | プログラムのデバッ グを行います。 | 機器削除によるプログラム修正ミス、動作タ イミングに影響がないことを確認します。 | 機器削除によるプログラム修正ミス、動作タ イミングに影響がないことを確認します。 |
| 7 | 装置の試運転を行い ます。 | 機器構成、制御プログラムの試運転を実施し ます。 | 機器構成、制御プログラムの試運転を実施し ます。 |

## 9-2-3 装置オプションの追加

- 事例）生産規模の拡大などを目的に、装置のコピーラインでは機器を部分的に拡張するとき

| | 目的 | 従来のプログラミング | Sysmac Studio のプログラミング |
|---|---|---|---|
| 1 | 追加する機器の割り 付けアドレスを確認 します。 | 拡張する装置で使用するユニットの割付アド レスを確認にします。 （注）割付アドレスを内部リレーとして使用 していた場合、内部リレーを別の空きア ドレスに割り付けます。 | － |
| 1 | 追加する機器の割り 付けアドレスを確認 します。 | プログラムで使用する内部リレーに変数名、 コメントを追加します。 | 拡張する装置で使用するユニットをユニット 構成に登録します。 |
| 1 | 追加する機器の割り 付けアドレスを確認 します。 | 拡張する装置で使用するユニットをユニット 構成に登録します。 | プログラムで使用する変数を登録します。 |
| 1 | 追加する機器の割り 付けアドレスを確認 します。 | － | 使用する変数とI/O デバイスを割り付けま す。 |
| 2 | 追加する機器の制御 プログラムを作成し ます。 | 制御プログラムを作成します。 （注）割付アドレスを内部リレーとして使用 していた場合、内部リレーを別の空きア ドレスに振り直します。 | 制御プログラムを作成します。 |
| 2 | 追加する機器の制御 プログラムを作成し ます。 | インターロック回路などは動作、影響を理解 して回路を改造します。 | インターロック回路などは動作、影響を理解 して回路を改造します。 |
| 3 | 追加する機器の構築 および配線を実行し ます。 | 装置構成図、配線図をもとに機器を追加し、 配線を行います。 | 装置構成図、配線図をもとに機器を追加し、 配線を行います。 |
| 4 | 配線チェック、アドレ スチェックを行いま す。 | 配線図とおりにカラーコードが接続されてい ることを目視、またはテスタなどにより導通 チェックで確認します。 | 配線図とおりにカラーコードが接続されてい ることを目視、またはテスタなどにより導通 チェックで確認します。 |
| 4 | 配線チェック、アドレ スチェックを行いま す。 | アドレスへの出力結果が機器に出力、また入 力機器からの入力が入力エリアに反映される ことを確認します。 | 変数への出力結果が機器に出力、また入力機 器からの入力が変数に反映されることを確認 します。 |
| 5 | プログラムのデバッ グを行います。 | プログラムデバッグを行います。 | プログラムデバッグを行います。 |
| 6 | 装置の試運転を行い ます。 | 変更後の機器構成、制御プログラムの試運転 を実施します。 | 変更後の機器構成、制御プログラムの試運転 を実施します。 |
