# 8-16 トラブルシュート

- トラブルシュート機能の概要コントローラで発生している異常の確認、異常の対処方法の指示、異常の解除および確認を行います。

8. -16-1 ［トラブルシュート］ダイアログボックスの表示オンライン接続先のコントローラから、発生中の異常またはイベントログを読み出し、［トラブルシュート・イベントログ］ダイアログボックスにて表示します。

1. オンライン接続状態で、［ツール］–［トラブルシュート・イベントログ］を選択します。または、ツールバーの［トラブルシュート、異常履歴］ボタンをクリックします。以下の［トラブルシュート・イベントログ］ダイアログボックスが表示されます。

![図(元マニュアル p.769)](images/sysmac/08-16-01.webp)

2. 必要に応じて、以下のタブをクリックして画面を切り替えます。

| タブ名 | 表示内容 |
|---|---|
| コントローラ異常 | コントローラ異常の異常内容とともに、異常の詳細内容と対処方法を通知し ます。 |
| コントローライベント ログ | コントローラ内のイベントログに記録されている異常および情報が表示され ます。 |
| ユーザ異常 | ユーザ異常発生命令（SetAlarm）、ユーザ情報発生命令（SetInfo）によって 発生したユーザ異常の異常内容と詳細内容（ユーザが定義）を通知します |
| ユーザイベントログ | コントローラ内のイベントログに記録されているユーザ異常およびユーザ情 報が表示されます。 |

- 操作手順

修正画面への表示切り替え手順

1. 発生している異常の内容および要因を、［詳細情報］、［付属情報］で確認します。

2. ［処置・対策］で、ひととおりの処置方法を確認します。

![図(元マニュアル p.773)](images/sysmac/08-16-03.webp)

![図(元マニュアル p.773)](images/sysmac/08-16-04.webp)

4. ［ERR ジャンプ］ボタンをクリックします。設定を修正する画面に切り替わります。

5. Sysmac Studio をオフラインにして修正を行います。

## 8-16-6 イベント設定テーブル

- イベント設定テーブルとは、ユーザ異常発生命令(SetAlarm)やユーザ情報発生命令(SetInfo)により発生するユーザイベントに対して、Sysmac Studio や表示器で表示する内容を登録するテーブルです。

- 各イベントコードに対して以下に説明する項目（イベント名、重要度、グループ、詳細情報）を登録することができます。

- イベント設定テーブルでは5120 個までの登録ができます。

- Excel で作成したデータをコピーし、イベント設定テーブルに貼付けます。

- イベント設定テーブルで直接編集することもできます。

- イベント名

各ユーザイベントの名称を登録します。タイトルを登録していない場合、「User fault Level 1～8」または「User Information」が表示されます。

- 重要度

ユーザイベントの重要度は下表のようにイベントコードの範囲により自動的に決まります。それぞれの重要度に対して、重要度名称を登録します。

| 5001～10000 | ↑ | User fault Level 2 | ユーザ異常発生命令(SetAlarm)で使用します。 |
| 10001～15000 | ↑ | User fault Level 3 | ユーザ異常発生命令(SetAlarm)で使用します。 |
| 15001～20000 | ↑ | User fault Level 4 | ユーザ異常発生命令(SetAlarm)で使用します。 |
| 20001～25000 | ↑ | User fault Level 5 | ユーザ異常発生命令(SetAlarm)で使用します。 |
| 25001～30000 | ↑ | User fault Level 6 | ユーザ異常発生命令(SetAlarm)で使用します。 |
| 30001～35000 | ↑ | User fault Level 7 | ユーザ異常発生命令(SetAlarm)で使用します。 |
| 35001～40000 | ↑ | User fault Level 8 | ユーザ異常発生命令(SetAlarm)で使用します。 |
