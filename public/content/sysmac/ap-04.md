# A-4 シミュレータと実機コントローラとの差異

## A-4-1 各機能の動作

シミュレータは、実機コントローラに比べ、以下のような機能的な差異があります。

| 項目 | | シミュレータと実機コントローラとの機能的な差異 |
|---|---|---|
| タスク | 起動 | シミュレータは、［シミュレーション］メニューまたはシミュレーションウィンドウから 起動します（［コントローラ］メニューからは起動できません）。 |
| タスク | 同期 | シミュレータは、シミュレータ接続時に同期された状態で立ち上がります（起動後に同 期を実行する必要はありません）。 |
| タスク | 終了 | シミュレータは、シミュレーションメニューまたはシミュレーションウィンドウで終了 します（接続が切断されるだけでなく、シミュレータプログラムとデータも消去されま す）。 |
| タスク | スケジューリング | シミュレータでは、タスク実行のスケジューリングはWindows のスケジューリングポ リシーにしたがって実行されます。 タスクごとの優先度やタイミングは無視されるため、必ずしも同じ動きをするとは限り ません。実行時間予測モードで実行した場合は、設定されたタイムアウト時間内に完了 しないプログラムも、タイムアウトせずにそのまま実行されます。完了後、軽度フォー ルト「タスク実行タイムアウト」が発生します。（コントローラではタイムアウト発生 後、全停止フォールト） これにより、タイムアウト時間内に完了しないプログラムの実行時間予測値を確認する ことができます。 タスク全体の動作およびタスク実行時間の違いの詳細については、後述の「A-4-2 タス ク実行動作」および「A-4-3 タスク実行時間」を参照してくだ さい。 形NX701 コントローラでは、プライマリ定周期タスクと定周期タスク（実行優先度5） を並列に実行できますが、シミュレータではコントローラと同様の並列実行はできませ ん。プライマリ定周期タスク、定周期タスク（実行優先度5）が順番に実行されます。 |
| タスク | 時計 | シミュレータ上の時刻は、パソコンの時刻とは異なります。プライマリ定周期タスクを 実行するたびに制御周期分の時間を加算することによって、時刻を進めます。 プログラムの一時停止やステップ実行を行っても、タイマの実行や時刻を参照するプロ グラムの動作には影響を与えません。 |
| タスク | モニタ | シミュレータでは、パソコン上の実行時間ではなく、実機コントローラ上で動作させた 場合の予測実行時間を表示します。 |
| タスク | リセット | シミュレータにはリセット機能はありません。シミュレータを停止し、再起動すること で、保持変数を除く各POU インスタンスの初期化を行います。 保持変数を含めて初期化したい場合は、Sysmac Studio を再起動してください。 |

| 項目 | | シミュレータと実機コントローラとの機能的な差異 |
|---|---|---|
| 変数・メモリ | 現在値モニタ | シミュレータでは、すべて非同期にて読み書きされます。そのため、実機コントローラ に比べ、以下のような動作の違いが見られることがあります。 • I/O マップにて変数の現在値変更を行った場合、実機コントローラでは現在値変更が 反映できない場合がありますが、シミュレータでは現在値変更が反映されます。 • 強制値リフレッシュ指定をした接点をプログラム内で操作した場合、シミュレータで はモニタ画面で接点がちらつくことがあります。 |
| 変数・メモリ | シミュレータ上の データ | 変数データはシミュレータ実行中のみ有効です。シミュレータ上のプログラムおよびデ ータは、シミュレータ終了時にすべてクリアされます。 |
| 変数・メモリ | 電断保持・非保持 属性 | シミュレータでは、シミュレーションを停止後、次の操作を行うと、シミュレーション 再開時に保持変数の値は初期化されます。 • プロジェクトの切り替え • 機種やユニットバージョンの切り替え • 構成・設定、およびプログラムの編集 ただし、構成・設定、およびプログラムを編集することなくシミュレーションを再開し た場合は、再開時の動作モードに関係なく、保持変数の値は保持されます。 |
| 変数・メモリ | メモリオールクリ ア | シミュレータには、本機能がありません（シミュレータをいったん終了し、再度シミュ レータ接続を行うことが必要です）。 |
| デバッグ | デバッガの利用 | シミュレータでは、ステップ実行やブレークポイントによるデバッグを実行する、デバ ッガ機能を使用可能です。 |
| デバッグ | 実行速度変更 | シミュレータでは、見かけ上のプログラム実行時間を変更することができます（パソコ ン上のプログラム実行間隔を変更して、見かけ上の実行速度を変更します）。 なお、シミュレータ上の時間経過は、実行速度変更の影響を受けません。 |
| デバッグ | デバッグプログラ ム | デバッグプログラムとして設定されたプログラムは、シミュレータ上でのみロードし、 実行することができます（一方、実機コントローラ上ではロードも実行もされません）。 |
| デバッグ | プログラムの部分 実行 | シミュレータでは、選択したプログラムのみを実行することが可能です（一方、実機コ ントローラ上では部分実行はできません）。 |
| モーション制御 | 軸制御 | シミュレータでは、サーボ軸が指定された場合は、エミュレート動作を行います。 エンコーダ軸が指定された場合は、シミュレータ命令で軸を動作させることができます。 |
| モーション制御 | 位置・速度制御 | シミュレータでは、各軸の指令値を現在位置または現在速度として、そのまま使用しま す。 |
| モーション制御 | トルク制御 | シミュレータはメカ機構情報を持たず、速度が計算できないため、現在速度は0 のまま となります。 |
| モーション制御 | ラッチ機能 | シミュレータでは、デバッグプログラム内でシミュレータ専用命令を使うことにより、 外部トリガを発生させることができます。 シミュレータでは、Z 相は2,048 パルスを入力するごとに発生します。 |
| モーション制御 | MC 試運転 | シミュレータでは使用できません。 |
| モーション制御 | カムデータ | シミュレータでは、カムデータのファイル転送機能（実機コントローラ→ファイル、フ ァイル→実機コントローラ）はサポートしていません。 |
| ネットワー ク | 通信アドレス設定 | シミュレータでは、ネットワーク通信は行わないため、設定内容は無視されます。 |
| ネットワー ク | ルーチングテーブ ル | シミュレータでは、ネットワーク通信は行わないため、設定内容は無視されます。 |
| CJ ユニッ ト 管理 | I/O テーブル | シミュレータでは、登録I/O をそのまま実I/O として運用されます。 |
| CJ ユニッ ト 管理 | スロット先頭アド レス | シミュレータでは、設定内容は無視されます。 |
| CJ ユニッ ト 管理 | CJ ユニット設定 | シミュレータでは、CJ ユニットのエミュレーションは行わないため、設定内容は無視 されます。 |

| 項目 | | シミュレータと実機コントローラとの機能的な差異 |
|---|---|---|
| 命令 | 通信命令 | シミュレータでは、命令の信号出力は正しく処理されますが、通信自体は実行されませ ん。 シリアル通信命令（ExecPMCR、SendCmd、SerialRcv、SerialSend）は、命令を実 行するとエラー（エラーID：0415）を出力します。 |

| 命令 | SD メモリカード 命令 | シミュレータでは、実機コントローラと同等に、SD メモリカード命令を使用できます。 シミュレータでは、“c:\OMRON\Data\SimulatorData\CARD\Memory001”を仮想 メモリカードとしてアクセスします。 ただし、BackupToMemoryCard 命令については、命令の信号出力処理は正常に終了 しますが、仮想メモリカードにバックアップファイルを作成しません。 |
| 命令 | 整数演算 | シミュレータでは、整数型や持続時間型の演算において、アンダーフローやオーバーフ ロー発生後の演算結果が異なる場合があります。 |
| 命令 | 浮動小数点・変換 命令 | シミュレータでは、実機コントローラと同等に命令を使用できます。 シミュレータでは、演算時の丸め誤差によって、演算結果が異なる場合があります。 シミュレータでは、指数関数(EXP)命令の結果や演算結果が、無限大(INFinity)または非 数(Not A Number)となった場合は、表記が異なる場合があります。 詳細は後述の「浮動小数点→文字列変換命令における無限大や非数の表記方法」を参照してください。 |
| 命令 | 時計命令 | シミュレータでは、時間／時刻命令の信号出力処理は正常に終了しますが、時刻変更処 理は実行されません。 |
| メンテナンス | 識別情報 | シミュレータでは、生産情報（LOT 番号などの識別情報）は保持されません。 |
| メンテナンス | 複数人プログラム | シミュレータは個人がパソコン単体で使用することを想定しています（複数人が1 台の シミュレータに対して同時に接続して使うといった使い方は、想定していません）。 |
| メンテナンス | 時計情報 | シミュレータでは、起動時のパソコンの時刻に、プライマリ定周期タスクが実行するご とに歩進する時刻を加算し、仮想時刻を生成します。 |
| メンテナンス | メモリカード | シミュレータでは、本機能は使用できません。 |
| メンテナンス | 変数／メモリのバ ックアップ | シミュレータでは、本機能は使用できません。 |
| メンテナンス | 周辺機器 | シミュレータには、外部周辺機器を接続できません、 |

## A-4-2 タスク実行動作

実機コントローラとシミュレータでは、タスク実行の制御方法に次のような違いがあります。

タスク実行動作の差異制御方法の違いにより、以下のような動作の違いが発生することがあります。

(1)データ交換のタイミングが異なる定周期タスクの終了タイミングや、各タスクの実行順番までは正確に再現されません。このため、タスク間でデータの受け渡しを行う場合やデータトレースの取得タイミングは、コントローラと異なる場合があります。

(2)時刻更新タイミングが異なるシミュレータ上の時刻は、その周期に実行するタスクが全て実行完了した時点で、プライマリ周期分の時間を加算することで更新されます。プログラムから時刻を参照する場合は、動作が異なることがあります。また、タイマ命令も、このタイミングで評価されるため、タイマがタイムアップするタイミングが異なることがあります。

(3)タスク実行タイムアウト時の動作が異なる実行時間予測モードで、実行時間が長いタスクを実行した場合、該当タスクが終了するまで他のタスクは実行されません。また、タスク処理の完了後に軽度フォールト「タスク実行タイムアウト」が発生します。コントローラの場合は、タイムアウト発生後に全停止フォールトとなり、動作が異なります。

## A-4-3 タスク実行時間

シミュレータでは、コントローラ上で実行した時の実行時間を予測*1 して表示します。『NJ/NX シリーズ CPU ユニット ユーザーズマニュアル ソフトウェア編（SBCA-467）』 の机上計算式の結果をシミュレーション結果として示します。*1. NY シリーズコントローラでは予測できません。

- I/O リフレッシュ処理時間NJ/NX シリーズコントローラの場合は、『NJ/NX シリーズ CPU ユニット ユーザーズマニュアルソフトウェア編（SBCA-467）』 の「A-2 NX シリーズでのタスク実処理時間の目安の計算」または「A-3 NJ シリーズでのタスク実処理時間の目安の計算」の「共通処理時間」と、「I/O リフレッシュ処理時間」を基に計算しています。バージョンごとのパラメータは、上記マニュアルに記載された通りです。

- ユーザプログラム（UPG）とモーションプログラム（MCE）の処理時間“較正式実行時間測定”方式（特許第4748286 号）にて予測しています。

タスク実行時間の予測値の分布は正規分布に従っているものとし、95%の信頼区間にて予測しています。

参考

パソコン上の他のアプリケーションにより処理時間が延びることがあります。シミュレーション実行中はできるだけ他のアプリケーションを動作させないようにしてください。

## A-4-4 浮動小数点データの表記

浮動小数点→文字列変換命令における無限大や非数の表記方法浮動小数点データを文字列に変換表記する命令において、コントローラとシミュレータで出力結果が異なる場合があります。

| | コントローラ上での表現 | シミュレータ上での表現 |
|---|---|---|
| ∞ | inf | 1.#INF00e+000 1.#INF0000000000e+000 |
| -∞ | -inf | -1.#INF00e+000-1.#INF0000000000e+000 |
| 非数 | nan | 1.#QNAN0e+000 1.#QNAN000000000e+000 |
| -非数 | -nan | -1.#IND00e+000-1.#IND0000000000e+000 |

（注）INF、QNAN、IND の後の“0”の数は増減することがあります。

指数の表記

指数表記の桁数は、実機コントローラでは2 桁または3 桁ですが、シミュレータでは必ず3 桁になります。文字列の比較を行う場合は、比較するデータの桁数が同じであることを確認してください。
