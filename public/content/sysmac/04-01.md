# 4-1 変数の登録

- 変数の登録とはNJ/NX/NY シリーズコントローラの制御アルゴリズムで使用するデータを格納する入れ物を宣言し、作成することです。ユーザが定義して登録可能な変数は、以下のとおりです。

| 変数種別 | 概要 |
|---|---|
| グローバル変数 | すべてのPOU（プログラム、FB、FUN）から読み書き可能な変数です。グローバル変数テーブルで定義（宣言）します。複数のPOU 間のインタフェースとなる変数として使われます。グローバル変数が宣言されたCPU ユニットで動作するPOU なら、どのPOU からでも読み書き可能です。しかし、他のCPU ユニットで動作するPOU からの読み書きはできません。スレーブ／ユニット構成から生成した「デバイス変数」、および軸／軸グループから自動生成される「軸／軸グループ変数」は、自動的にグローバル変数に登録されます。 |
| デバイス変数 | デバイス（スレーブ／ユニット）内のデータにアクセスする変数です。「I/O ポート」と呼ぶ外部とのインタフェース用の論理ポートを介してアクセスします。Sysmac Studio の「I/O マップ」にて、EtherCAT スレーブ構成情報およびユニット構成情報を作成すると自動生成される「I/O ポート」に対して、割り付けます。 |
| ローカル変数 | POU の内部でのみ使用する変数です。POU のローカル変数テーブルで定義（宣言）します。POU の外部から、ローカル変数の値を参照することはできません。異なるPOU で、同じ名前のローカル変数を宣言しても、それらのメモリ上の実体は異なります。 |
| 軸変数 | 「軸」が持つ情報（物理量、状態、異常情報など）を構造体で定義した変数です。「軸」に割り付けているEtherCAT スレーブにアクセスするためのシステム定義変数です。 |
| 軸グループ変数 | 補間関係にある「軸」をまとめた「軸グループ」を、構造体で定義した変数です。プログラムの中で補間動作をするときに使用します。 |
| カムデータ変数 | カムデータにアクセスするための変数です。Sysmac Studio のカムデータ設定によって作成します。位相と変位データからなる、構造体型の配列指定の変数です。 |
| FB のインスタンス変数 | FB を個別に管理するために、そのFB に付ける変数（名前）です。名前を付け、個別に内部変数を保持できる状態になったFB を「FB のインスタンス」と呼びます。インスタンス化されたFB は、自動的にローカル変数に登録されます。他のPOU からFB を呼び出す際、インスタンス変数名を指定します。 |
| 派生データ型変数（構造体型、共用体型、列挙型） | 異なるデータ型を1 つにまとめてグループ化したユーザ定義のデータ型です。データ型に任意の名称を付けることができます。作成したユーザ定義のデータ型を使用する変数名と、そのグループ内の1 つの変数（「メンバ」と呼ぶ）名を指定します。 |
| システム定義変数 | あらかじめNJ/NX/NY システムとして、名称などのすべての属性が定義されており、かつ特定機能が割り付けられている変数です。機能モジュールごとに用意されています。ユーザが、名称などの属性を変更することはできません。 |

## 4-1-1 グローバル変数の登録

- グローバル変数の登録とは複数のPOU（プログラム、ファンクション、ファンクションブロック）からアクセスが可能な変数を登録することです。EtherCAT スレーブデバイス変数やCJ ユニットデバイス変数などスレーブやユニットのデータにアクセスするためのデバイス変数の登録も含まれます。

- グローバル変数の登録画面の起動マルチビューエクスプローラで［プログラミング］–［データ］–［グローバル変数］をダブルクリックします。または、［プログラミング］–［データ］–［グローバル変数］を右クリックして、メニューから［編集］を選択します。

![図(元マニュアル p.180)](images/sysmac/04-01-02.webp)

- グローバル変数の新規登録

1. グローバル変数テーブル内で[Insert]キーを押すか、または右クリックしてメニューから［新規作成］を選択します。

![図(元マニュアル p.180)](images/sysmac/04-01-03.webp)

変数が登録されます。変数名（名称）、データ型は必ず設定してください。各属性の入力方法は、「4-1-9 変数の属性項目の入力方法」を参照してください。

## 4-1-2 デバイス変数の登録

- デバイス変数とはデバイス（スレーブ／ユニット）内のデータにアクセスする変数です。「I/O ポート」と呼ぶ外部とのインタフェース用のポートを介してアクセスします。

新規登録する方法

I/O マップから、デバイス変数を新規登録します。デバイス変数名を新規で入力する方法と、自動生成する2 つの方法があります。

- デバイス変数名を新規で入力する方法デバイス変数名をユーザが入力します。例えば以下のときに、この方法でデバイス変数を登録します。• スレーブのI/O ターミナルや基本I/O ユニットのI/O 名を任意につけるとき

1. スレーブ構成情報またはユニット構成情報を作成します。2 I/O マップにて、I/O ポートを選択して、「変数」の列に変数名を入力します。登録先の変数テーブルを指定する場合は、事前に［変数種別］で指定しておきます。

![図(元マニュアル p.186)](images/sysmac/04-01-22.webp)

各スレーブ／ユニットのI/O ポートに対して、デバイス変数が自動的に割り付けられ、［変数種別］で指定した変数テーブルに登録されます。

- デバイス変数名を自動生成する方法デバイス変数名を、「デバイス名」または任意の文字列と、「I/O ポート名」の組み合わせで自動生成します。

| メニュー名 | 説明 |
|---|---|
| デバイス変数生成 | 「デバイス名」と「I/O ポート名」の組み合わせで変数を自動生成します。 |

- 例えば以下のときに、この方法でデバイス変数を登録します。

- ローカル変数をグローバル変数へ移動

1. 変数一覧リストで対象の変数を選択し、右クリックメニューから［変数をグローバルに移動］を選択します。

選択した変数が、グローバル変数テーブルへ移動し、ローカル変数から削除されます。このとき対象変数がローカル変数テーブルの外部変数に、自動的に登録されます。

使用上の注意

- グローバル変数をローカル変数へ移動する場合、POU が存在しないか、または2 つ以上の場合はエラーとなり実行できません。

- 移動結果を変数マネージャの表示に反映するには、変数テーブル選択コンボボックスで、対象変数テーブルを選択するか、［表示の更新］ボタンをクリックして、表示の更新を行ってください。
