# 6-2 コントローラとのオンライン接続

Sysmac Studioからプロジェクト内の複数のコントローラに、同時にオンライン接続することができます。本節で説明する操作は、現在選択されているコントローラが対象となります。プロジェクト内に複数のコントローラが登録されている場合は、接続する前に必ず操作対象のコントローラを確認してください。

## 6-2-1 コントローラとのオンライン接続の概要

コントローラとのオンライン接続には、接続形態により、次に示す4種類の接続方法があります。

| 接続方法 | 接続方法の説明 |
|---|---|
| USB-直接接続 *1 | パソコンのUSBポートとコントローラのUSBポートを直接接続します。 |
| Ethernet-直接接続 | パソコンのEthernetポートとコントローラの内蔵EtherNet/IPポートを直接接続します。 |
| USB-リモート接続 | パソコンのUSBポートからコントローラのUSBポートに直接接続し、Ethernetネットワークを経由して他のコントローラの内蔵EtherNet/IPポートへ接続します。*2 |
| Ethernet-HUB接続 | パソコンのEthernetポートからEthernetネットワークを経由してコントローラの内蔵EtherNet/IPポートへ接続します。*2 |

*1.NYシリーズコントローラ、および形NX502、形NX102、形NX1P2コントローラの場合は表示されません。*2.CJシリーズEtherNet/IPユニットのポート経由でコントローラへオンライン接続することはできません。

**使用上の注意**

- 以下のコントローラの場合は、Windows 10バージョン1803以降のパソコンを使用してください。Windows 10バージョン1709以前ではコントローラにオンライン接続できません。

- 形NX701のユニットバージョンVer.1.29以降

- 形NX502のユニットバージョンVer.1.60以降

- 形NX102、形NX1P2、NJシリーズのユニットバージョンVer.1.49以降CPUユニットのユニットバージョンに対応するWindowsのバージョンの詳細については、『NJ/NXシリーズCPUユニットユーザーズマニュアルソフトウェア編（SBCA-467）』を参照してください。また、オンライン接続できない場合の対処方法については『NJ/NXシリーズトラブルシューティングマニュアル（SBCA-469）』を参照してください。

## 6-2-2 接続方法の設定

パソコンとコントローラとの接続方法の選択や接続先のIPアドレスなどの通信設定を行います。

1. メインメニューから［コントローラ］–［通信設定］を選択します。［通信設定］ダイアログボックスが表示されます。

3. ［OK］ボタンをクリックします。設定が完了します。

## 6-2-3 オンライン接続

オンライン接続の操作方法をまとめます。

1. ［コントローラ］–［オンライン］を選択します。または、ツールバーからオンライン接続ボタン（：）をクリックします

初めてオンライン接続する場合は、次のメッセージが表示されます。一度名称を書き込むと、以降は表示されません。

![図(元マニュアル p.506)](images/sysmac/06-02-02.webp)

2. ［はい］ボタンをクリックします。オンライン接続状態になり、エディットウィンドウの上部が黄色に表示されます。

![図(元マニュアル p.506)](images/sysmac/06-02-03.webp)

**使用上の注意**

USBケーブルを抜いた直後10秒程度は、再度USBケーブルを接続しないでください。また、USBケーブルを接続後、Windowsが接続を認識するまではケーブルを抜かないでください。以降、Windowsがケーブルの抜き差しを認識できず、USBポートが使用可能にならない場合があります。この現象が発生した場合、パソコンの再起動、もしくはサスペンド／休止状態から復帰するまでは、USBデバイスは使用可能な状態になりません。

**参考**

Windowsのファイアウォールが有効な場合、Sysmac Studioをインストール後初めてコントローラにオンライン接続したときに、接続を確認するダイアログボックスが表示されることがあります。このとき［パブリックネットワーク］にチェックを入れて［アクセスを許可する］ボタンをクリックしてください。ブロックを解除(またはアクセスを許可)しなかった場合、EtherNet/IPコネクションの設定転送やモニタができなくなることがあります。詳細は『NJ/NXシリーズCPUユニット内蔵EtherNet/IPポートユーザーズマニュアル（SBCD-377）』または『NYシリーズ産業用パネル型PC／産業用ボックス型PCユーザーズマニュアル内蔵EtherNet/IPポート編（SBCD-369）』を参照してください。

## 6-2-7 強制状態の確認

［通信設定］ダイアログボックスのオプション設定で、［オンライン終了後に、強制値リフレッシュの確認を行う。］にチェックを入れていると、オンライン接続終了時に強制値リフレッシュが設定された変数が存在する場合、確認のダイアログボックスが表示されます。

オンライン接続を終了時に次のダイアログボックスが表示されます。

![図(元マニュアル p.509)](images/sysmac/06-02-06.webp)

1. ［一括解除］ボタンをクリックします。強制状態を一括解除されます。

2. ［オフラインにする］ボタンをクリックします。オンライン接続が終了します。

**使用上の注意**

- USB接続（［USB-直接接続］および［USB-リモート接続］）では、内部的にEthernet IPアドレス「192.168.255.＊」を利用して通信します。USB接続を行うときは、パソコンのEthernetインタフェースカードでIPアドレス「192.168.255.＊」を使用したり、IPアドレス「192.168.255.＊」のコントローラにリモート接続したりしないでください。

- EthernetのUDP/IP通信ではソケットのポート「9600」および「2224」を利用します。EthernetのUDP/IP通信を行う場合は、他のアプリケーションでこのポートを使用しないでください。

- 「Ethernet-直接接続」を利用する場合、パソコンに複数のEthernetインタフェースカードが装着されている場合は、どのEthernetインタフェースカードを利用するかを選択する必要があります。選択方法は、付録の「A-2複数のEthernetインタフェースカードを使用している場合の指定方法」を参照してください。

- NJ/NXシリーズコントローラで以下の設定をしている場合、［Ethernet-直接接続］、［USB-リモート接続］、［Ethernet-HUB接続］でオンライン接続できなくなることがあります。a)内蔵EtherNet/IPポート設定の［CIPメッセージサーバ］を［使用しない］に設定している。b)内蔵EtherNet/IPポート設定の［Packet Filter］を［する］に設定している。設定の詳細は、『NJ/NXシリーズCPUユニット内蔵EtherNet/IPポートユーザーズマニュアル（SBCD-377）』を参照してください。

- 形NX701、形NX502、形NX102コントローラ、または形NX-CSG320通信コントロールユニットへ「Ethernet-直接接続」で接続する場合は、コントローラのPORT1へケーブルを接続してください。PORT2へケーブルを接続した場合はオンライン接続できません。

**参考**

- オンライン接続時のエラーと対処方法については、「11-4 Sysmac Studio操作のエラーメッセージ一覧」を参照してください。

- オンライン接続時、コントローラに操作権限が設定されている場合は、操作権限の認証ダイアログボックスが表示されます。操作権限認証の詳細については、「8-3-2操作権限認証」を参照してください。

- オンライン接続時、コントローラにユーザ認証が設定されている場合は、ユーザ認証のダイアログボックスが表示されます。ユーザ認証の詳細については、「8-3-3ユーザ認証」を参照してください。

## 6-2-8 プロジェクトを開いていない状態でのオンライン接続

プロジェクトを新規作成したり開いたりせずに、すばやくコントローラにオンライン接続することができます。オンライン接続後に、プロジェクトデータをアップロードしたり、トラブルシュート機能を起動したりすることができます。

1. Sysmac Studioを起動し、スタートページで［デバイスに接続］をクリックします。設定項目が表示されます。

![図(元マニュアル p.510)](images/sysmac/06-02-07.webp)

デバイス：接続するデバイスの種類を選択します。

接続方法：コントローラとの接続方法を選択します。

［ネットワークブラ：［ネットワークブラウザ］ダイアログボックスを表示し、コ

ウザ］：ントローラとの接続状態を確認します。

接続設定：接続方法にUSB-リモート接続またはEthernet-HUB接続を選択したときの接続先IPアドレスを設定します。

| 項目 | 説明 | | |
|---|---|---|---|
| 接続後の動作 | オンライン接続後の動作を選択します。 | | |
| 接続後の動作 | / 転送［デバイス →パソコン］ | コントローラにプロジェクトが存在する場合にデータをアップロードします。チェックを入れると、アップロード対象を選択する項目が表示されます。 | |
| 接続後の動作 | / 転送［デバイス →パソコン］ | / CPUユニット | プログラムや構成・設定データなどCPUユニットに含まれるデータをアップロードします。 これにはEtherCATコネクション設定（タグデータリンク設定）も含まれますが、CJユニットやEtherCATスレーブのデータは含まれません。 常にアップロード対象となります。 |
| 接続後の動作 | / 転送［デバイス →パソコン］ | / CJユニット （NJシリーズのみ）／ EtherCATスレーブ／ NXユニット | チェックを入れると、CJユニット、 EtherCATスレーブ、およびNXユニットのデータをアップロードします。 |

**参考**

作成されたプロジェクトの「作成者」には、オプション設定で設定した作成者名が自動的に設定されます。オプション設定の詳細については、「8-26 Sysmac Studioオプション設定」を参照してください。
