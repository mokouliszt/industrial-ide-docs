# 6-2 コントローラとのオンライン接続

Sysmac Studio からプロジェクト内の複数のコントローラに、同時にオンライン接続することができます。本節で説明する操作は、現在選択されているコントローラが対象となります。プロジェクト内に複数のコントローラが登録されている場合は、接続する前に必ず操作対象のコントローラを確認してください。

## 6-2-1 コントローラとのオンライン接続の概要

コントローラとのオンライン接続には、接続形態により、次に示す4 種類の接続方法があります。

| 接続方法 | 接続方法の説明 |
|---|---|
| USB-直接接続 *1 | パソコンのUSB ポートとコントローラの USB ポートを直接接続します。 |
| Ethernet-直接 接続 | パソコンのEthernet ポートとコントローラ の内蔵EtherNet/IP ポートを直接接続しま す。 |
| USB-リモート 接続 | パソコンのUSB ポートからコントローラの USB ポートに直接接続し、Ethernet ネット ワークを経由して他のコントローラの内蔵 EtherNet/IP ポートへ接続します。*2 |
| Ethernet-HUB 接続 | パソコンのEthernet ポートからEthernet ネットワークを経由してコントローラの内 蔵EtherNet/IP ポートへ接続します。*2 |

*1.NY シリーズコントローラ、および形NX502、形NX102、形NX1P2 コントローラの場合は表示されません。*2.CJ シリーズEtherNet/IP ユニットのポート経由でコントローラへオンライン接続することはできません。

使用上の注意

- 以下のコントローラの場合は、Windows 10 バージョン1803 以降のパソコンを使用してください。Windows 10 バージョン1709 以前ではコントローラにオンライン接続できません。

- 形NX701 のユニットバージョンVer.1.29 以降

- 形NX502 のユニットバージョンVer.1.60 以降

- 形NX102、形NX1P2、NJ シリーズのユニットバージョンVer.1.49 以降CPU ユニットのユニットバージョンに対応するWindows のバージョンの詳細については、『NJ/NX シリーズ CPU ユニット ユーザーズマニュアル ソフトウェア編（SBCA-467）』を参照してください。また、オンライン接続できない場合の対処方法については『NJ/NX シリーズ　トラブルシューティングマニュアル（SBCA-469）』を参照してください。

## 6-2-3 オンライン接続

オンライン接続の操作方法について説明します。

1. ［コントローラ］–［オンライン］を選択します。または、ツールバーからオンライン接続ボタン（：）をクリックします

初めてオンライン接続する場合は、次のメッセージが表示されます。一度名称を書き込むと、以降は表示されません。

![図(元マニュアル p.506)](images/sysmac/06-02-02.webp)

2. ［はい］ボタンをクリックします。オンライン接続状態になり、エディットウィンドウの上部が黄色に表示されます。

![図(元マニュアル p.506)](images/sysmac/06-02-03.webp)

使用上の注意

USB ケーブルを抜いた直後10 秒程度は、再度USB ケーブルを接続しないでください。また、USB ケーブルを接続後、Windows が接続を認識するまではケーブルを抜かないでください。以降、Windows がケーブルの抜き差しを認識できず、USB ポートが使用可能にならない場合があります。この現象が発生した場合、パソコンの再起動、もしくはサスペンド／休止状態から復帰するまでは、USB デバイスは使用可能な状態になりません。

参考

Windows のファイアウォールが有効な場合、Sysmac Studio をインストール後初めてコントローラにオンライン接続したときに、接続を確認するダイアログボックスが表示されることがあります。このとき［パブリックネットワーク］にチェックを入れて［アクセスを許可する］ボタンをクリックしてください。ブロックを解除(またはアクセスを許可)しなかった場合、EtherNet/IP コネクションの設定転送やモニタができなくなることがあります。詳細は『NJ/NX シリーズ CPU ユニット 内蔵EtherNet/IP ポート ユーザーズマニュアル（SBCD-377）』または『NY シリーズ 産業用パネル型PC／産業用ボックス型PC ユーザーズマニュアル 内蔵EtherNet/IP ポート編（SBCD-369）』を参照してください。

## 6-2-7 強制状態の確認

［通信設定］ダイアログボックスのオプション設定で、［オンライン終了後に、強制値リフレッシュの確認を行う。］にチェックを入れていると、オンライン接続終了時に強制値リフレッシュが設定された変数が存在する場合、確認のダイアログボックスが表示されます。

オンライン接続を終了時に次のダイアログボックスが表示されます。

![図(元マニュアル p.509)](images/sysmac/06-02-06.webp)

1. ［一括解除］ボタンをクリックします。強制状態を一括解除されます。2 ［オフラインにする］ボタンをクリックします。オンライン接続が終了します。

使用上の注意

- USB 接続（［USB-直接接続］および［USB-リモート接続］）では、内部的にEthernet IP アドレス「192.168.255.＊」を利用して通信します。USB 接続を行うときは、パソコンのEthernet インタフェースカードでIP アドレス「192.168.255.＊」を使用したり、IP アドレス「192.168.255.＊」のコントローラにリモート接続したりしないでください。

- Ethernet のUDP/IP 通信ではソケットのポート「9600」および「2224」を利用します。Ethernet のUDP/IP 通信を行う場合は、他のアプリケーションでこのポートを使用しないでください。

- 「Ethernet-直接接続」を利用する場合、パソコンに複数のEthernet インタフェースカードが装着されている場合は、どのEthernet インタフェースカードを利用するかを選択する必要があります。選択方法は、付録の「A-2 複数のEthernet インタフェースカードを使用している場合の指定方法」を参照してください。

- NJ/NX シリーズコントローラで以下の設定をしている場合、［Ethernet-直接接続］、［USB-リモート接続］、［Ethernet-HUB 接続］でオンライン接続できなくなることがあります。a) 内蔵EtherNet/IP ポート設定の［CIP メッセージサーバ］を［使用しない］に設定している。b) 内蔵EtherNet/IP ポート設定の［Packet Filter］を［する］に設定している。設定の詳細は、『NJ/NX シリーズ CPU ユニット 内蔵EtherNet/IP ポート ユーザーズマニュアル（SBCD-377）』を参照してください。• 形NX701、形NX502、形NX102 コントローラ、または形NX-CSG320 通信コントロールユニットへ「Ethernet-直接接続」で接続する場合は、コントローラのPORT1 へケーブルを接続してください。PORT2 へケーブルを接続した場合はオンライン接続できません。

参考

- オンライン接続時のエラーと対処方法については、「11-4 Sysmac Studio 操作のエラーメッセージ一覧」を参照してください。

- オンライン接続時、コントローラに操作権限が設定されている場合は、操作権限の認証ダイアログボックスが表示されます。操作権限認証の詳細については、「8-3-2 操作権限認証」を参照してください。
